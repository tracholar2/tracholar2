用于PHP函数的Big-O列表

By zhizi

Published 2018-04-18

现在使用PHP一段时间后，我注意到并不是所有的PHP都像预期的那样快速地构建函数。考虑以下两种可能的函数实现，使用缓存的素数阵列来查找数字是否为素数。

//very slow for large $prime_array
$prime_array = array( 2, 3, 5, 7, 11, 13, .... 104729, ... );
$result_array = array();
foreach( $prime_array => $number ) {
    $result_array[$number] = in_array( $number, $large_prime_array );
}

//speed is much less dependent on size of $prime_array, and runs much faster.
$prime_array => array( 2 => NULL, 3 => NULL, 5 => NULL, 7 => NULL,
                       11 => NULL, 13 => NULL, .... 104729 => NULL, ... );
foreach( $prime_array => $number ) {
    $result_array[$number] = array_key_exists( $number, $large_prime_array );
}

这是因为in_array是通过线性搜索O（n）来实现的，它会随着$ prime_array的增长而线性减速。其中array_key_exists函数是使用散列查找O（1）实现的，除非散列表被极度填充（在这种情况下，它只是O（n）），否则它不会变慢。

到目前为止，我不得不通过反复试验来发现这个大O，偶尔也会发现这个大O的实现。 -on-the-c-level“>查看源代码。现在的问题…

独特（）与lambda？

By zhizi

Published 2018-04-16

正确的，所以我有一个枚举，并希望从它得到不同的值。

使用System.Linq，当然有一个叫做Distinct的扩展方法。在简单的情况下，它可以使用没有参数，如：

var distinctValues = myStringList.Distinct();

好的，但如果我有一个对象，我需要指定相等的唯一可用的重载：

var distinctValues = myCustomerList.Distinct(someEqualityComparer);

相等比较参数必须是IEqualityComparer＆lt; T＆gt;的实例。当然，我可以做到这一点，但是有些冗长，而且很滑稽。

我所期望的是一个需要lambda的重载，比如Func＆lt; T，T，bool＆gt;：

var distinctValues
    = myCustomerList.Distinct((c1, c2) => c1.CustomerId == c2.CustomerId);

任何人都知道，如果有一些这样的扩展存在，或一些相当的解决方法？或者我错过了什么？

另外，有没有一种方法来指定一个IEqualityComparer inline（不理我）呢？

<强>更新

我发现了Anders Hejlsberg的一个回复张贴在这个主题的MSDN论坛。他说：

The problem you’re going to run into is that when two objects compare equal
they must have the same GetHashCode return value (or else the hash table used
internally by Distinct will not function correctly). We use IEqualityComparer
because it packages compatible implementations of Equals and GetHashCode into
a single interface.

我想这是有道理的。
