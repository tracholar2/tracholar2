AngularJS ngClass条件

By zhizi

Published 2018-01-15

有什么办法可以让一个像ngClass这样的表达式成为一个条件表达式。例如，我已经尝试了以下内容：

<span ng-class="{test: 'obj.value1 == \'someothervalue\''}">test</span>

这个代码的问题是，不管obj.value1是什么，类测试总是被应用到元素。这样做：

<span ng-class="{test: obj.value2}">test</span>

只要obj.value2不等于真值，这个类就不会被应用。现在我可以在第一个例子中解决这个问题了：

<span ng-class="{test: checkValue1()}">test</span>

其中checkValue1函数如下所示：

$scope.checkValue1 = function() {
  return $scope.obj.value === 'somevalue';
}

我只是想知道这是不是ngClass应该如何工作。我也正在建立一个自定义指令，我想要做类似的事情。然而，我找不到一种方法来观看表达式（也许这是不可能的，也是这样的原因）。

为什么不在HTML中使用表格进行布局？ [关闭]

By zhizi

Published 2018-01-15

这似乎是[一般意见

a>表格不应该用于HTML中的布局。

为什么？

我从来没有（或很少说实话）看到这个好的论点。通常的答案是：

  • 这是很好的单独的内容布局
    但这是一个错误的论点; Cliche Thinking 。我想这是真的，使用表格元素的布局与表格数据很少。所以呢？我的老板照顾吗？我的用户关心吗？

也许我或我的开发人员必须维护一个网页关心…是不是一个表维护？我认为使用表比更容易使用div和CSS。顺便说一句…为什么要使用div或跨度很好的分离内容从布局和表格不是？只有div才能获得良好的布局，往往需要大量的嵌套div。

  • 代码的可读性 - 我认为这是相反的方式。大多数人理解HTML，很少理解CSS。

  • SEO最好不要使用表格
    为什么？有人可以证明这是一个证据吗？或者谷歌的声明表示，从SEO的角度来看是不鼓励的？

  • 表格较慢。
    必须插入一个额外的tbody元素。这是现代网页浏览器的花生。给我看一些使用表格显着减慢页面速度的基准。

  • 如果没有表格，布局检查更容易，请参阅 css Zen Garden 。
    大多数网站需要升级需要新的内容（HTML）。新版本的网站只需要一个新的CSS文件的情况不太可能。禅园是一个不错的网站，但有点理论。更不用说它的[滥用CSS了。

我对使用div + CSS而不是表格的好理由感兴趣。
