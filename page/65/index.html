将ISO 8601兼容的字符串转换为java util Date

By zhizi

发表于 2018-01-14

我正尝试将 ISO 8601格式化的字符串转换为java.util.Date。

如果与Locale（比较示例）一起使用，我发现模式“yyyy-MM-dd’T’HH：mm：ssZ”符合ISO8601标准。
但是，使用java.text.SimpleDateFormat，我不能转换格式正确的字符串“2010-01-01T12：00：00 +
01：00”。我必须先将它转换为“2010-01-01T12：00：00 + 0100”，不用冒号。 所以，目前的解决方案是

SimpleDateFormat ISO8601DATEFORMAT = new SimpleDateFormat("yyyy-MM-dd'T'HH:mm:ssZ", Locale.GERMANY);
String date = "2010-01-01T12:00:00+01:00".replaceAll("\\+0([0-9]){1}\\:00", "+0$100");
System.out.println(ISO8601DATEFORMAT.parse(date));

这显然不是那么好。我错过了什么或有更好的解决方案吗？

<强>答案

感谢JuanZe的评论，我发现了 Joda-Time 的魔法，它也是<这里描述了一个href =“http://johannburkard.de/blog/programming/java
/date-time-parsing-formatting-joda-time.html”rel =“noreferrer”>。 所以，解决方案是

DateTimeFormatter parser2 = ISODateTimeFormat.dateTimeNoMillis();
String jtdate = "2010-01-01T12:00:00+01:00";
System.out.println(parser2.parseDateTime(jtdate));

或者更简单的说，通过构造函数使用默认的解析器：

DateTime dt = new DateTime( "2010-01-01T12:00:00+01:00" ) ;

对我来说，这很好。

property装饰器是如何工作的？

By zhizi

发表于 2018-01-14

我想了解内置函数property是如何工作的。我感到困惑的是，property也可以作为装饰器使用，但是只有作为内置函数使用时才需要参数，而不能用作装饰器。

这个例子来自文档：

class C(object):
    def __init__(self):
        self._x = None

    def getx(self):
        return self._x
    def setx(self, value):
        self._x = value
    def delx(self):
        del self._x
    x = property(getx, setx, delx, "I'm the 'x' property.")

property的参数是getx，setx，delx和一个doc字符串。

在下面的代码中，property用作装饰器。它的对象是x函数，但是在上面的代码中，参数中没有对象函数的地方。

class C(object):
    def __init__(self):
        self._x = None

    @property
    def x(self):
        """I'm the 'x' property."""
        return self._x

    @x.setter
    def x(self, value):
        self._x = value

    @x.deleter
    def x(self):
        del self._x
