推导出来！我不得不深入研究，发现“魔法”发生的地方：

Hash(params Func<object, TValue>[] hash)
{
  foreach (var func in hash)
  {
    Add(func.Method.GetParameters()[0].Name, func(null));
  }
}

值对的字典。由此产生的语法结构的确非常具有表现力，但同时又非常危险。一般使用lambda表达式可以替换没有副作用的 名字 。我在一本书中看到一个例子：collection.ForEach（book =＆gt; Fire.Burn（book））我知道我可以在我的代码中编写collection.ForEach（log =＆gt; Fire.Burn（log））和 它意味着同样的事情 。但是在这里使用MvcContrib

那么C＃3.5 / 4.0社区和lambda表达式爱好者的这种常见做法是什么？或者是一个流氓特技，我不应该担心？
