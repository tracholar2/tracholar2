在C＃中使用var关键字

By zhizi

Published 2018-07-19

在与同事讨论了在C＃3中使用“var”关键字之后，我想知道人们对var类型推断的适当用法有何看法？

例如，我比较懒惰地在可疑的情况下使用var，例如： -

foreach(var item in someList) { // ... } // Type of 'item' not clear.
var something = someObject.SomeProperty; // Type of 'something' not clear.
var something = someMethod(); // Type of 'something' not clear.

var更合理的用法如下：

var l = new List<string>(); // Obvious what l will be.
var s = new SomeClass(); // Obvious what s will be.

有趣的是，LINQ似乎有点灰色地带，例如： -

var results = from r in dataContext.SomeTable
              select r; // Not *entirely clear* what results will be here.

很明显，结果将会是一个实现了IEnumerable的类型，但是它和声明一个新对象的var不一样。

LINQ到对象更糟，例如： -

var results = from item in someList
              where item != 3
              select item;

这并不比等同的foreach（在someList中的var项）// // …}等同。

这里有一个关于类型安全性的真正问题 - 例如，如果我们将查询的结果放入一个接受IEnumerable＆lt;
int＆gt;的重载方法中，和IEnumerable＆lt; double＆gt;调用者可能会无意中传入错误的类型。

var ` does
保持强类型，但问题是这样做的危险在于定义时不能立即显示类型，当重载意味着编译器错误当您无意中将错误的类型传递给某个方法时，可能不会被发布。
