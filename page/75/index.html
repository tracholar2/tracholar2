如何在Bash中的分隔符上分割字符串？

By zhizi

Published 2018-01-22

我把这个字符串保存在一个变量中：

IN="bla@some.com;john@home.com"

现在我想通过;分隔符来分割字符串，这样我就可以：

ADDR1="bla@some.com"
ADDR2="john@home.com"

我不一定需要ADDR1和ADDR2变量。如果它们是一个更好的数组元素。

在回答下面的建议后，我得出了以下的结论：

#!/usr/bin/env bash

IN="bla@some.com;john@home.com"

mails=$(echo $IN | tr ";" "\n")

for addr in $mails
do
    echo "> [$addr]"
done

输出：

> [bla@some.com]
> [john@home.com]

有一个解决方案，将 Internal_field_separator（IFS）设置为;。我不确定那个答案是怎么回事，你怎么把IFS重置为默认？

RE：IFS解决方案，我试过这个，它工作，我保留旧的IFS，然后恢复：

IN="bla@some.com;john@home.com"

OIFS=$IFS
IFS=';'
mails2=$IN
for x in $mails2
do
    echo "> [$x]"
done

IFS=$OIFS

顺便说一句，当我尝试

mails2=($IN)

在循环中打印时，只有第一个字符串，$ IN的括号没有括起来。

使用保存实例状态保存Android活动状态

By zhizi

Published 2018-01-21

我一直在Android SDK平台上工作，如何保存应用程序的状态还有点不清楚。所以，在“Hello，Android”的例子中进行了一些小的改动：

package com.android.hello;

import android.app.Activity;
import android.os.Bundle;
import android.widget.TextView;

public class HelloAndroid extends Activity {

  private TextView mTextView = null;

  /** Called when the activity is first created. */
  @Override
  public void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);

    mTextView = new TextView(this);

    if (savedInstanceState == null) {
       mTextView.setText("Welcome to HelloAndroid!");
    } else {
       mTextView.setText("Welcome back.");
    }

    setContentView(mTextView);
  }
}

我认为最简单的情况就足够了，但无论我如何离开应用程序，它总是以第一条消息作为响应。
