好奇的空合并运算符自定义隐式转换行为

By zhizi

Published 2018-05-27

注意：这似乎已在 Roslyn 中解决

这个问题出现在我写这个的答案时，它讲述了 null-
coalescing运算符。

提醒一下，空合并运算符的思想是形式表达式

x ?? y

首先评估x，然后：

  • 如果x的值为null，则评估y，这是表达式的最终结果
  • 如果x的值是非null，则y是 not ，并且x是表达式的最终结果，必要时转换为y的编译时类型

现在 通常 不需要转换，或者它只是从一个可以为空的类型转换为一个不可空的类型 - 通常类型是相同的，或者仅仅来自（说）int ？到int。但是，您可以创建您自己的隐式转换运算符，并在必要时使用它们。

对于x的简单情况y，我还没有看到任何奇怪的行为。但是，用（x ?? y）?? z我看到一些令人困惑的行为。

这是一个简短但完整的测试程序 - 结果在评论中：

using System;

public struct A
{
    public static implicit operator B(A input)
    {
        Console.WriteLine("A to B");
        return new B();
    }

    public static implicit operator C(A input)
    {
        Console.WriteLine("A to C");
        return new C();
    }
}

public struct B
{
    public static implicit operator C(B input)
    {
        Console.WriteLine("B to C");
        return new C();
    }
}

public struct C {}

class Test
{
    static void Main()
    {
        A? x = new A();
        B? y = new B();
        C? z = new C();
        C zNotNull = new C();

        Console.WriteLine("First case");
        // This prints
        // A to B
        // A to B
        // B to C
        C? first = (x ?? y) ?? z;

        Console.WriteLine("Second case");
        // This prints
        // A to B
        // B to C
        var tmp = x ?? y;
        C? second = tmp ?? z;

        Console.WriteLine("Third case");
        // This prints
        // A to B
        // B to C
        C? third = (x ?? y) ?? zNotNull;
    }
}

所以我们有三个自定义的值类型：A，B和C，转换从A到B，A到C， B到C。

我可以理解第二种情况和第三种情况…但是为什么在第一种情况下会有额外的A到B的转换？特别是，我真的很希望第一种情况和第二种情况是相同的 -
毕竟只是将一个表达式提取到一个局部变量中。

有什么事情发生？在C＃编译器中，我非常犹豫要哭“bug”，但是我很难理解发生了什么…

编辑：好吧，这是一个很糟糕的例子，感谢配置器的答案，这给了我更多的理由认为这是一个错误。编辑：示例甚至不需要两个空合并操作符…

using System;

public struct A
{
    public static implicit operator int(A input)
    {
        Console.WriteLine("A to int");
        return 10;
    }
}

class Test
{
    static A? Foo()
    {
        Console.WriteLine("Foo() called");
        return new A();
    }

    static void Main()
    {
        int? y = 10;

        int? result = Foo() ?? y;
    }
}

这个输出是：

Foo() called
Foo() called
A to int

Foo（）在这里被调用两次的事实对我来说是非常令人惊讶的 - 我看不出有两次表达式的原因。_ p>

Edit

见下面的哑巴解决方案; - ）

Scope

主要活动有一个带有片段的ViewPager。这些片段可以为其他（子域）活动实现一些不同的逻辑，所以片段的数据通过活动内的回调接口填充。一切运行正常，但！！

Problem

当活动被重新创建（例如在方向改变时），所以ViewPager的片段也是如此。代码（你会发现下面）说，每当创建活动，我尝试创建一个新的ViewPager碎片适配器相同的碎片（也许这是问题），但FragmentManager已经拥有了所有这些片段存储在某处（在哪里？），并启动这些娱乐机制。因此，娱乐机制调用onAttach，onCreateView等“旧”片段与我的回调接口调用启动数据通过活动的实施方法。但是这个方法指向通过Activity的onCreate方法创建的新创建的片段。

Issue

也许我正在使用错误的模式，但即使是Android 3 Pro书也没有太多的介绍。所以， **请给我一个两拳，并指出如何以正确的方式做到这一点。非常感谢！

public class BasePagerAdapter extends FragmentPagerAdapter implements TitleProvider {

private Map<String, Fragment> mScreens;

public BasePagerAdapter(Map<String, Fragment> screenMap, FragmentManager fm) {

    super(fm);
    this.mScreens = screenMap;
}

@Override
public Fragment getItem(int position) {

    return mScreens.values().toArray(new Fragment[mScreens.size()])[position];
}

@Override
public int getCount() {

    return mScreens.size();
}

@Override
public String getTitle(int position) {

    return mScreens.keySet().toArray(new String[mScreens.size()])[position];
}

// hack. we don't want to destroy our fragments and re-initiate them after

By zhizi

Published 2018-05-26

ExpandoObject .aspx)类被添加到.NET
4允许您在运行时任意设置属性到一个对象。

这对于使用 Dictionary＆lt; string，对象＆gt; ，或甚至是一个哈希表？据我所知，这只是一个哈希表，你可以用更简洁的语法来访问。

dynamic obj = new ExpandoObject();
