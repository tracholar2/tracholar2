shell脚本可以设置调用shell的环境变量吗？

By zhizi

Published 2018-05-20

Contents

我正在编写一个shell脚本，在运行的时候会设置一些环境变量，这些变量将保持在调用者的shell中。

setenv FOO foo

在csh / tcsh或

中

export FOO=foo

sh / bash中的

只在脚本执行期间设置它。

我已经知道了

source myscript

将运行脚本的命令，而不是启动一个新的shell，这可能导致设置“调用者”环境。

但是，这是一个难题：

我希望这个脚本可以从bash或者csh中调用。换句话说，我希望任何一个shell的用户都能够运行我的脚本并改变shell的环境。所以’源’不适用于我，因为运行csh的用户无法获取bash脚本，而运行bash的用户无法获取csh脚本。

是否有任何合理的解决方案，不需要在脚本上编写和维护两个版本？

你的shell进程拥有父进程环境的副本，不能访问父进程的环境。当你的shell进程终止你对其环境所做的任何更改时，都会丢失。采购脚本文件是配置shell环境最常用的方法，您可能只想咬一口，为两种shell的每一种维护一个。

使用“点空间脚本”调用语法。例如，以下是如何使用脚本的完整路径：

. /path/to/set_env_vars.sh

以下是如何与脚本位于同一目录中的方法：

. set_env_vars.sh

这些脚本在当前shell下执行脚本，而不是加载另一个脚本（如果执行了./set_env_vars.sh，会发生什么情况）。因为它运行在同一个shell中，所设置的环境变量将在退出时可用。

这与调用source set_env_vars.sh是一样的，但是它的类型更短，并且可能在source没有的地方工作。
