我应该如何将字符串转换为C＃中的枚举？

By zhizi

Published 2018-05-24

Contents
  1. 1. Original Answer

将字符串转换为C＃中的枚举值的最佳方法是什么？

我有一个包含枚举值的HTML选择标记。当页面发布时，我想要获取值（将以字符串的形式）并将其转换为枚举值。

在理想的世界里，我可以做这样的事情：

StatusEnum MyStatus = StatusEnum.Parse("Active");

但这不是有效的代码。

在.NET核心和.NET＆gt; 4中有一个通用的解析方法：

Enum.TryParse("Active", out StatusEnum myStatus);

这也包括C＃7的新内联out变量，所以它会执行try-parse，转换为显式枚举类型并初始化+填充myStatus变量。

如果您有权访问C＃7和最新的.NET，这是最好的方法。

Original Answer

在.NET中，它相当难看（直到4或以上）：

StatusEnum MyStatus = (StatusEnum) Enum.Parse(typeof(StatusEnum), "Active", true);

我倾向于简化：

public static T ParseEnum<T>(string value)
{
    return (T) Enum.Parse(typeof(T), value, true);
}

那么我可以这样做：

StatusEnum MyStatus = EnumUtil.ParseEnum<StatusEnum>("Active");

在注释中建议的一个选项是添加一个扩展，这很简单：

public static T ToEnum<T>(this string value)
{
    return (T) Enum.Parse(typeof(T), value, true);
}

StatusEnum MyStatus = "Active".ToEnum<StatusEnum>();

最后，如果字符串不能被解析，你可能想要使用默认的枚举：

public static T ToEnum<T>(this string value, T defaultValue) 
{
    if (string.IsNullOrEmpty(value))
    {
        return defaultValue;
    }

    T result;
    return Enum.TryParse<T>(value, true, out result) ? result : defaultValue;
}

这就是调用：

StatusEnum MyStatus = "Active".ToEnum(StatusEnum.None);

不过，我会小心的添加一个这样的扩展方法到string（没有命名空间控制），它会出现在string的所有实例上，不管它们是否持有枚举（所以1234.ToString（）.ToEnum（StatusEnum.None）是有效的，但是没有意义）。除非您的整个开发团队对这些扩展的功能有非常好的理解，否则通常最好避免使用只适用于特定环境的额外方法混淆微软的核心类。

使用 Enum.TryParse＆lt; T＆gt;（字符串，T）（.NET 4.0）：

StatusEnum myStatus;
Enum.TryParse("Active", out myStatus);

使用C＃7.0可以进一步简化参数类型内联：

Enum.TryParse("Active", out StatusEnum myStatus);
