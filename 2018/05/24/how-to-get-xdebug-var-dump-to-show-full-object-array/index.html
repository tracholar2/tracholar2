如何让xdebug var_dump显示完整的对象/数组

By zhizi

Published 2018-05-24

Contents

我在 xdebug （php_xdebug-2.1.2-5.3-vc9.dll） .wampserver.com
/ EN /“> WAMP 。在大对象或变量上使用var_dump时，它不会显示完整的变量。

array
'node' => 
  array
    'my_form' => 
      array
        'form' => 
          array
            ...

没有xdebug它显示应该是预期的。我看了文档，但没有看到解决方案。有谁知道我可以如何解决这个问题，所以xdebug var_dump显示了
完整的 对象？

这些是php.ini中的可配置变量：

; with sane limits
xdebug.var_display_max_depth = 5
xdebug.var_display_max_children = 256
xdebug.var_display_max_data = 1024 


; with no limits
; (maximum nesting is 1023)
xdebug.var_display_max_depth = -1 
xdebug.var_display_max_children = -1
xdebug.var_display_max_data = -1

当然，这些也可以在运行时通过ini_set（）来设置，如果你不想修改php.ini并重启你的web服务器，但是需要更快地检查某些东西

ini_set('xdebug.var_display_max_depth', 5);
ini_set('xdebug.var_display_max_children', 256);
ini_set('xdebug.var_display_max_data', 1024);

Xdebug设置在官方文档中有解释。

我知道这是一个超级老的帖子，但我想这可能还是有帮助的。

如果你对阅读json格式感到满意，你可以用你的var_dump代替：

return json_encode($myvar);

我一直在使用这个来帮助解决我一直在构建的服务，它有一些深度嵌套的数组。这将返回你的数组的每个级别而不截断任何东西或要求你改变你的php.ini文件。

另外，因为json_encoded数据是一个字符串，这意味着你可以很容易地将它写入错误日志。

error_log(json_encode($myvar));

对于任何情况，这可能不是最好的选择，但这是一个选择！
