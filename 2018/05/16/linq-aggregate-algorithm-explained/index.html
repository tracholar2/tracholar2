LINQ聚合算法解释

By zhizi

发表于 2018-05-16

文章目录

这可能听起来很蹩脚，但我一直无法找到Aggregate的真正解释。

好的意思是简短的，描述性的，全面的，一个小而明确的例子。

Aggregate最容易理解的定义是它考虑到之前的操作，对列表中的每个元素执行一个操作。也就是说，它对第一个和第二个元素执行操作，并将结果向前传送。然后它对前面的结果和第三个元素进行操作并继续。等

示例1.总结数字

var nums = new[]{1,2,3,4};
var sum = nums.Aggregate( (a,b) => a + b);
Console.WriteLine(sum); // output: 10 (1+2+3+4)

这添加1和2来制作3。然后添加3（前面的结果）和3（序列中的下一个元素）来制作6。然后添加6和4来制作10。

示例2.从一个字符串数组中创建一个csv

var chars = new []{"a","b","c", "d"};
var csv = chars.Aggregate( (a,b) => a + ',' + b);
Console.WriteLine(csv); // Output a,b,c,d

这种工作方式大致相同。连接a逗号和b来制作a，b。然后使用逗号和c连接a，b来制作a，b，c。等等。

示例3.使用种子乘以数字

为了完整起见，Aggregate有一个[重载](http://msdn.microsoft.com/en- us/library/bb549218.aspx)需要一个种子值。

var multipliers = new []{10,20,30,40};
var multiplied = multipliers.Aggregate(5, (a,b) => a * b);
Console.WriteLine(multiplied); //Output 1200000 ((((5*10)*20)*30)*40)

类似上面的例子，它以5的值开始，并将其乘以序列10的第一个元素，给出50 < /代码>。这个结果被继续前进，并乘以序列 20
中的下一个数字，给出 1000 `的结果。这继续通过序列的剩余2个元素。

现场示例： http://rextester.com/ZXZ64749
文档： http://msdn.microsoft.com/zh-cn/library/bb548651的.aspx

<强>附录

上面的例子2使用字符串连接来创建一个由逗号分隔的值列表。这是解释使用Aggregate的简单方法，这就是这个答案的意图。但是，如果使用这种技术来实际创建大量逗号分隔的数据，则使用 StringBuilder 会更合适，而且这完全与Aggregate 兼容。种子重载来启动 StringBuilder `。

var chars = new []{"a","b","c", "d"};
var csv = chars.Aggregate(new StringBuilder(), (a,b) => {
    if(a.Length>0)
        a.Append(",");
    a.Append(b);
    return a;
});
Console.WriteLine(csv);

更新示例： http://rextester.com/YZCVXV6464

这部分取决于你所说的超载，但基本的想法是：

  • 以种子作为“当前值”开始
  • 迭代序列。对于序列中的每个值：
    • 应用用户指定的函数将（currentValue，sequenceValue）转换为（nextValue） 设定currentValue = nextValue 返回最终的currentValue
      您可以找到 在我的Edulinq系列中的 Aggregate ` 很有用 -
      它包含更详细的描述（包括各种重载）和实现。

一个简单的例子是使用Aggregate作为Count的替代：

// 0 is the seed, and for each item, we effectively increment the current value.
// In this case we can ignore "item" itself.
int count = sequence.Aggregate(0, (current, item) => current + 1);

或者，也许总结字符串序列中所有长度的字符串：

int total = sequence.Aggregate(0, (current, item) => current + item.Length);

我个人很少找到Aggregate有用 - “量身定制”的聚合方法通常对我来说足够好。
