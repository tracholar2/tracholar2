如何将流保存到C＃文件中？

By zhizi

发表于 2018-05-01

文章目录

我有一个StreamReader对象，我初始化了一个流，现在我想保存这个流到磁盘（流可能是一个.gif或.jpg或.pdf）。

现有的代码：

StreamReader sr = new StreamReader(myOtherObject.InputStream);
  • 我需要将其保存到磁盘（我有文件名）。
  • 将来我可能希望将其存储到SQL Server中。

我也有编码类型，如果将它存储到SQL Server中，我将需要它，正确吗？

正如Jon Skeet的回答中的Tilendor所强调的那样，自.NET 4以来流有一个CopyTo方法。

var fileStream = File.Create("C:\\Path\\To\\File");
myOtherObject.InputStream.Seek(0, SeekOrigin.Begin);
myOtherObject.InputStream.CopyTo(fileStream);
fileStream.Close();

或者使用使用语法：

using (var fileStream = File.Create("C:\\Path\\To\\File"))
{
    myOtherObject.InputStream.Seek(0, SeekOrigin.Begin);
    myOtherObject.InputStream.CopyTo(fileStream);
}

对于二进制文件（如gif或jpgs），您不得_使用StreamReader 。 StreamReader `用于 text
数据。如果您将它用于任意的二进制数据，您几乎肯定会丢失数据。
（如果你使用Encoding.GetEncoding（28591），你可能会好起来，但是有什么意义？）

为什么你需要使用一个StreamReader？为什么不把二进制数据保存为二进制数据，并将其作为二进制数据写回磁盘（或SQL）呢？

编辑：因为这似乎是人们想要看到的东西…如果你只是想复制一个流到另一个（例如到一个文件）使用这样的东西：

/// <summary>
/// Copies the contents of input to output. Doesn't close either stream.
/// </summary>
public static void CopyStream(Stream input, Stream output)
{
    byte[] buffer = new byte[8 * 1024];
    int len;
    while ( (len = input.Read(buffer, 0, buffer.Length)) > 0)
    {
        output.Write(buffer, 0, len);
    }    
}

使用它来将流转储到文件，例如：

using (Stream file = File.Create(filename))
{
    CopyStream(input, file);
}

请注意，在.NET 4中引入了Stream.CopyTo ` ，它们的用途基本相同。
