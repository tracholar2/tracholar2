退出基于进程退出代码的Shell脚本

By zhizi

Published 2018-05-08

Contents

我有一个执行许多命令的shell脚本。如果任何命令以非零退出代码退出，如何退出shell脚本？

在每个命令之后，可以在$？变量中找到退出代码，这样你就可以得到如下所示的结果：

ls -al file.ext
rc=$?; if [[ $rc != 0 ]]; then exit $rc; fi

由于$ <仅给出管道中最后一个元素的返回代码，所以在代码中

ls -al file.ext | sed 's/^/xx: /"

如果文件不存在，

将不会返回错误代码（因为管道的sed部分实际工作，返回0）。

** shell实际上提供了一个可以帮助解决这个问题的数组，即PIPESTATUS。这个数组对于每个管道组件都有一个元素，您可以像$ {PIPESTATUS [0]}一样单独访问：

pax> false | true ; echo ${PIPESTATUS[0]}
1

请注意，这是获取false命令的结果，而不是整个管道。您也可以根据需要处理整个列表：

pax> false | true | false; echo ${PIPESTATUS[*]}
1 0 1

如果您想从流水线中获取最大的错误代码，可以使用如下所示：

true | true | false | true | false
rcs=${PIPESTATUS[*]}; rc=0; for i in ${rcs}; do rc=$(($i > $rc ? $i : $rc)); done
echo $rc

如果它大于前面的rc值，则依次遍历每个PIPESTATUS元素，并将其存储在rc中。 / p>

如果你想使用$？，你需要在每个命令后检查它，因为$？在每个命令退出后更新。这意味着如果你执行一个管道，你将只能得到管道中最后一个进程的退出代码。

另一种方法是做到这一点：

set -e
set -o pipefail

如果你把它放在shell脚本的顶部，它看起来像bash将为你照顾这个。正如前面的海报所指出的那样，“set
-e”会导致bash在任何简单的命令中出错并退出。 “set -o pipefail”会导致bash在管道中的任何命令上出错并退出。
