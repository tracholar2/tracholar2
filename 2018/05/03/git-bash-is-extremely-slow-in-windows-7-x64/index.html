Git / Bash在Windows 7 x64中速度非常慢

By zhizi

Published 2018-05-03

Contents

在一个小型项目的开发过程中，我一直在Windows和Ubuntu上使用Git，经常在两者之间来回翻转。我遇到的问题是Git /
Bash一直变慢。当我说慢时，我的意思是运行cd需要8-25秒，运行git命令需要5-20秒，而` ls
对我而言，暂时性的解决方案之一就是禁用我的网络连接（如这个答案），启动git，然后重新连接。有时在这样做后的几天里，它会继续快速运行，但是性能总是最终降低。我已经通过msysgit讨论组，SO，msysgit问题列表等拖延了几个星期，但我一直无法打开解决方案的工作。

到目前为止，我已经尝试过：

  • 加入git＆amp;项目文件夹到病毒扫描程序的排除列表
  • 彻底禁用我的病毒扫描程序（Kaspersky IS 2011）
  • 确保Outlook未运行（Outlook 2007）
  • 关闭所有其他应用程序
  • 以管理员身份运行git
  • 禁用网络连接，启动git并禁用连接
  • 禁用网络连接，启动git，重新启用连接（仅偶尔使用）
  • 运行git gc
  • 以上的组合

我读过一些人已经成功地禁止了bash的完成，但理想情况下，我想保持这种活跃。
msysgit的版本是1.7.3.1-preview20101002＆amp;操作系统是Windows 7
x64。在Linux上运行相同的东西，可以预见，闪电般快。我会专门使用Linux，但是我也需要在Windows中运行一些东西（某些应用程序，测试等）。

有没有人遇到类似的问题？如果是这样，底层的问题是什么，解决方案是什么（如果有的话）？

编辑：这超出了只是git仓库，但仅供参考，我已经使用git的回购已经很小：最多4-50个文件

通过运行三个命令来设置一些配置选项，您可以显着加快Windows上的git：

$ git config --global core.preloadindex true
$ git config --global core.fscache true
$ git config --global gc.auto 256

注意：

core.preloadindex `并行执行文件系统操作以隐藏延迟（update：在默认情况下在git 2.1中启用）

core.fscache `修复了UAC问题，所以你不需要以admin身份运行git（更新：在Git for Windows 2.8中默认启用）

  • gc.auto将.git /

中的文件数量减至最少

您的Bash提示中是否显示Git信息？如果是这样，也许你无意中在每一个命令上做了太多的工作。要测试这个理论，请在Bash中尝试以下临时更改：

export PS1='$'
