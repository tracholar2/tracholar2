直接施放vs as 操作符？

By zhizi

Published 2018-05-10

Contents

考虑下面的代码：

void Handler(object o, EventArgs e)
{
   // I swear o is a string
   string s = (string)o; // 1
   //-OR-
   string s = o as string; // 2
   // -OR-
   string s = o.ToString(); // 3
}

这三种类型的铸件之间有什么区别（好的，第三件不是铸件，但你有意图）。哪一个应该是首选？

string s = (string)o; // 1

如果o是 InvalidCastException 不是字符串。否则，即使o是null，也会将o赋给s。

string s = o as string; // 2

如果o不是string，或者o ， >是 null
。出于这个原因，你不能使用它的值类型（在这种情况下，运算符永远不能返回 null ）。否则，将 o 分配给 s `。

string s = o.ToString(); // 3

如果o导致 NullReferenceException 是null。无论什么类型的o是什么，都将o.ToString（）返回给s。

使用1进行大多数转换 -
这很简单，直接。我倾向于几乎从不使用2，因为如果事情不是正确的类型，我通常期望发生异常。我只看到需要这种返回null类型的功能与糟糕的设计库使用错误代码（例如返回null
=错误，而不是使用例外）。

3不是强制转换，只是一个方法调用。在需要非字符串对象的字符串表示时使用它。

  • 什么时候应该使用 绝对是另一回事。
  • 当 可能是 时使用另一个 事情。
  • 当你不在乎什么的时候使用 这是，但你只是想使用的 可用的字符串表示形式。
