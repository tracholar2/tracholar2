Bash：如何包含其他脚本？

By zhizi

Published 2018-05-27

Contents

通常包含脚本的方式是使用“source”

例如：

main.sh：

#!/bin/bash

source incl.sh

echo "The main script"

incl.sh：

echo "The included script"

执行“./main.sh”的输出是：

The included script
The main script

…现在，如果您尝试从另一个位置执行该shell脚本，除非它位于您的路径中，否则它将无法找到包含。

确保脚本可以找到包含脚本的好方法是什么，特别是脚本需要是可移植的？ 我倾向于使我的脚本都是相对于彼此。 这样我可以使用dirname：

#!/bin/sh

my_dir="$(dirname "$0")"

"$my_dir/other_script.sh"

我知道我迟到了，但是无论你如何开始脚本并独占使用内建脚本，这都是可行的。

DIR="${BASH_SOURCE%/*}"
if [[ ! -d "$DIR" ]]; then DIR="$PWD"; fi
. "$DIR/incl.sh"
. "$DIR/main.sh"

命令是源的别名，$ PWD是工作目录的路径，BASH_SOURCE是一个数组变量，成员是源文件名，$ {string％substring}从$ string后面剥离$ substring的最短匹配
