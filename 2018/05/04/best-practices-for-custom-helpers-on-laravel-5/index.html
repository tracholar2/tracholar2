在Laravel 5上定制帮助程序的最佳实践

By zhizi

Published 2018-05-04

Contents

我想创建一些帮助器（函数）来避免在一些视图之间重复使用L5风格的代码：

view.blade.php

<p>Foo Formated text: {{ fooFormatText($text) }}</p>

他们基本上是文本格式功能。在哪里以及如何将这些功能放在一个文件？

在您的应用程序文件夹中创建一个helpers.php文件，并使用composer加载：

"autoload": {
    "classmap": [            ...
    ],
    "psr-4": {
        "App\\": "app/"
    },
    "files": [            "app/helpers.php" // <---- ADD THIS
    ]
},

将composer.json文件添加到`文件后，运行以下命令：

composer dump-autoload

我最初的想法也是作曲家自动加载，但是对我来说，并不觉得Laravel就是这样。 L5大量使用服务提供商，它们是引导您的应用程序的原因。

开始之前，我在app目录中创建了一个名为Helpers的文件夹。然后在Helpers文件夹中添加我想要添加的函数的文件。拥有一个包含多个文件的文件夹，可以避免一个大文件变得太长而难以管理。

接下来，我通过运行artisan命令创建了一个HelperServiceProvider.php：

artisan make:provider HelperServiceProvider

在register方法中，我添加了这个片段

public function register()
{
    foreach (glob(app_path().'/Helpers/*.php') as $filename){
        require_once($filename);
    }
}

最后将服务提供者注册到providers数组中的config / app.php中

'providers' => [        'App\Providers\HelperServiceProvider',
]

现在，您的Helpers目录中的任何文件都已加载，并可供使用。

更新2016-02-22

这里有很多不错的选择，但是如果我的答案适合你，我就继续前进，并以这种方式包括助手。您可以使用该软件包获取灵感，也可以随时使用Composer进行下载。它有一些内置的帮助器，我经常使用（但默认情况下都是非活动的），并允许您使用简单的Artisan生成器来创建自己的定制帮助器。它还解决了一个响应者使用映射器的建议，并允许您明确定义要加载的自定义助手，或者默认情况下会自动将所有PHP文件加载到助手目录中。反馈和PR非常感谢！
