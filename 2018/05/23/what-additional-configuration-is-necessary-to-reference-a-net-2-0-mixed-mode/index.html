在 NET 4 0项目中引用 NET 2 0混合模式程序集时需要什么“附加配置”？

By zhizi

发表于 2018-05-23

文章目录

我有一个项目，我想使用一些.NET
4.0的功能，但是一个核心要求是我可以使用针对2.X编译的System.Data.SQLite框架。我看到提到这是可能的，如这里接受的答案，但我不明白如何实际实现这一目标。

当我试着在引用2.X程序集的时候运行我的4.0项目时：

Mixed mode assembly is built against version ‘v2.0.50727’ of the runtime
and cannot be loaded in the 4.0 runtime without additional configuration
information.

什么“额外配置”是必要的？

为了使用CLR 2.0 混合模式程序集 ，你需要修改你的App.Config文件来包含：

<?xml version="1.0"?>  
<configuration>  
  <startup **useLegacyV2RuntimeActivationPolicy="true"** >  
    <supportedRuntime version="v4.0" sku=".NETFramework,Version=v4.0"/>  
  </startup>  
</configuration>

关键是useLegacyV2RuntimeActivationPolicy标志。这会导致CLR使用最新版本（4.0）加载混合模式程序集。没有这个，它将无法正常工作。

注意这只对混合模式（C ++ / CLI）程序集很重要。您可以加载所有托管的CLR 2程序集，而无需在app.config中指定。

这论坛帖子在.NET Framework开发人员中心上。这可能会提供一些见解。

（添加到应用程序的配置文件。）

<configuration>
  <startup useLegacyV2RuntimeActivationPolicy="true">
    <supportedRuntime version="v4.0"/>
  </startup>
</configuration>
