BASH函数中的return和exit语句在退出代码方面有什么区别？

return从函数返回一个值。 exit放弃当前的shell。

引用exit的最后一条语句引起了一些注释。为了理解OP，事实上，在程序/ shell脚本的 任意 给定点上，区分return和exit ，exit是以退出代码结束脚本到调用进程的唯一方法。

在shell中执行的每个命令都会产生一个本地的“退出代码”：它将$？变量设置为该代码，并且可以与一起使用if，＆amp;＆amp;和其他运算符来有条件地执行其他命令。

这些退出代码（以及$？变量的值）由每个命令执行重置。

顺便说一句，由脚本执行的最后一个命令的退出代码被用作调用进程看到的脚本本身的退出代码。

最后，函数在调用时作为关于退出代码的shell命令。函数的退出代码（ __ ）是通过使用return来设置的。因此，当函数return 0运行时，函数执行终止，给出0的退出码。

return 会导致当前函数超出范围，而 exit `会导致脚本在被调用的地方结束。这里有一个示例程序来帮助解释这个：

#!/bin/bash

retfunc()
{
    echo "this is retfunc()"
    return 1
}

exitfunc()
{
    echo "this is exitfunc()"
    exit 1
}

retfunc
echo "We are still here"
exitfunc
echo "We will never see this"

Output

$ ./test.sh
this is retfunc()
We are still here
this is exitfunc()
