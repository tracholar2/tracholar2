在PHP中从空值创建默认对象？

By zhizi

Published 2018-05-21

Contents

只有在将我的PHP环境升级到PHP 5.4及更高版本后，才会看到此错误。错误指向这行代码：

错误：

Creating default object from empty value

代码：

$res->success = false;

我首先需要声明我的$ res对象吗？

您的新环境可能有E_STRICT警告 error_reporting 设置为至少 E_WARNING ，则在error_reporting `中启用“noreferrer” >与PHP 5.4 +。 。当$ res为NULL或尚未初始化时触发该错误：

$res = NULL;
$res->success = false; // Warning: Creating default object from empty value

如果$ res已经被初始化为某个值但不是一个对象，那么PHP会报告一个不同的错误信息：

$res = 33;
$res->success = false; // Warning: Attempt to assign property of non-object

为了符合PHP 5.4之前的E_STRICT标准，或者PHP> 5.4中的正常E_WARNING错误级别，假设您正在尝试创建通用对象并指定属性success，则需要在全局名称空间中声明$ res作为stdClass的对象：

$res = new \stdClass();
$res->success = false;

这个消息对于PHP <= 5.3是E_STRICT。从PHP 5.4开始，它被改为E_WARNING。由于E_WARNING消息很有用，所以您不希望完全禁用它们。

为了摆脱这个警告，你必须使用下面的代码：

if (!isset($res)) 
    $res = new stdClass();

$res->success = false;

这是 完全等同替换 。它保证了完全一样的东西，PHP是默默地做的 - 不幸的是现在警告 -
隐式对象的创建。你应该经常检查对象是否已经存在，除非你完全确定它不存在。
Michael提供的代码通常并不好，因为在某些情况下，对象有时可能已经在代码中的相同位置定义了，具体情况取决于环境。
