管道到/从剪贴板

By zhizi

Published 2018-02-13

Contents

是否可以在Bash中通过剪贴板进行传输？

无论是从设备手柄进行管道连接还是使用辅助应用程序，我都找不到任何东西。

例如，如果/ dev / clip是连接剪贴板的设备，我们可以这样做：

cat /dev/clip        # Dump the contents of the clipboard
cat foo > /dev/clip  # Dump the contents of "foo" into the clipboard

你有点模棱两可我希望你可能是X内部的一个Linux用户，他们想把东西放在X PRIMARY剪贴板中。

理解bash没有剪贴板是很重要的。没有“剪贴板”这样的东西，因为bash可以在Windows，Mac OS
X，很多其他操作系统上运行，在X内部，在X外部运行……更不用说X本身三个不同的剪贴板。有很多你可以处理的剪贴板。通常，您要与之交谈的剪贴板上有一个实用工具，可以让您与之交谈。

在X的情况下，是的，有xclip（和其他）。 xclip -selection c会将数据发送到在大多数应用程序中与Ctrl-C，Ctrl-
V一起使用的剪贴板。

如果您正试图与Mac OS X剪贴板进行交谈，那么pbcopy。

如果你在Linux终端模式下（不是X），那么你可能需要查看gpm。

还有 GNU screen ，它有一个剪贴板。把东西放在那里，看看screen命令“readreg”。

在Windows / cygwin下，使用/ dev / clipboard或者clip更新版本的Windows（至少是Windows
10）。

确保你使用的是别名xclip =“xclip -selection c” 否则你不能只用Ctrl + v粘贴到另一个地方。

echo test | xclip
