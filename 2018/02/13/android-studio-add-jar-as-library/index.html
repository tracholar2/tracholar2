Android Studio：添加jar作为库？

By zhizi

发表于 2018-02-13

文章目录

我试图使用新的Android Studio，但似乎无法正常工作。

我使用 Gson 库来序列化/反序列化JSON对象。但该库不知何故不包含在构建中。

我只用 MainActivity 创建了一个新项目。 在/ libs文件夹中复制 gson-2.2.3.jar
，并将其添加为库依赖（右键单击＆gt;添加为库）。这包括android studio中的jar，所以可以从源文件中引用它。

当我尝试运行这个项目时，它无法编译，所以我添加了：

compile files('libs/gson-2.2.3.jar')

在.gradle文件中的依赖关系。之后，它编译正确，但在运行应用程序时，我得到一个ClassDefNotFoundException。

有人知道我在做什么错了吗？

我一直在为同样的事情奋斗了好几个小时，试图让Gson jar工作不下。我终于破解了它 - 这是我采取的步骤：

  • 将Gson jar（在我的例子中是gson-2.2.4.jar）放到libs文件夹中
  • 右键点击它并点击“添加为库”
  • 确保编译文件（'libs / gson-2.2.4.jar'）在您的build.gradle文件（或compile fileTree（dir ：'libs'，include：'* .jar'）如果你使用的是很多jar文件）
  • 做一个干净的构建（你可以在Android Studio中做到这一点，但要确保我在终端中导航到我的应用程序的根文件夹并输入gradlew clean。 Mac OS X，该命令可能与您的系统不同

我做了以上四个，它开始工作正常。我认为“添加为图书馆”的步骤是我以前错过的步骤，直到我清理它才能工作。

[编辑 - 添加了build.gradle步骤，这也是别人指出的必要步骤]

以下是将本地jar文件作为库添加到模块的说明：

  • 在模块目录（包含’src’目录的同一目录）顶层创建一个’libs’文件夹

在build.gradle文件中添加以下内容，以便您的依赖关闭具有：

     依赖关系{
    // ...其他依赖关系
    编译文件（'libs /＆lt;你的jar的名字在这里＆gt;'）
}

Android Studio应该已经设置了一个gradlew包装器。从命令行导航到项目的顶层（具有gradlew文件的目录）。

运行./ gradlew assemble。这应该与图书馆编译项目。您可能需要根据需要修复build.gradle文件中的错误。

为了使Android Studio在IDE中进行编码时将本地jar文件识别为支持库，您需要执行以下几个步骤：

4.1。右键单击左侧面板上的模块，然后选择打开库设置。

4.2。在对话框的左侧面板上，选择Libraries。

4.3。点击面板上方的+标志，从左侧第二个 - ＆gt; <代码>爪哇

4.4。选择您的本地jar并将其添加到项目中。

  • 您可能需要再次运行上面的./ gradlew命令
