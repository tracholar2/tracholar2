sh和bash之间的区别

编写shell程序时，经常使用/ bin / sh和/ bin / bash。我通常使用bash，但我不知道它们之间有什么区别。

和sh之间的主要区别是什么？

在bash和sh中进行编程时，我们需要注意什么？

What is sh

sh （或者Shell命令语言）是由[ POSIX 标准](“http://pubs.opengroup.org/onlinepubs/009695399/utilities/xcu_chap02.html”描述的编程语言，的rel)。 它有许多实现（ ksh88 ， dash ，...）。也可以是 考虑了sh（见下文）的实现。

因为sh是一个规范，而不是一个实现，所以/ bin / sh是一个符号链接 （或硬链接）在大多数POSIX系统上的实际实现。

What is bash

bash 作为一个 sh
`兼容的实现来启动（虽然它早于POSIX标准几年），但是随着时间的推移，它已经获得了许多扩展。许多这样的扩展可能会改变有效的POSIX
shell脚本的行为，所以它本身并不是一个有效的POSIX shell。相反，它是POSIX shell语言的一种方言。

bash 支持一个 - posix 开关，这使得它更符合POSIX标准。如果以 sh `的形式调用，它也试图模仿POSIX。

sh = bash?

长期以来，在大多数GNU / Linux系统上，/ bin / sh用于指向/ bin / bash。结果，忽略两者之间的差异几乎变得安全了。但最近开始发生变化。

/ bin / sh不指向/ bin / bash（以及其中一些/ bin / bash代码>可能不存在）：

  • 现代Debian和Ubuntu系统，默认情况下将sh链接到dash;
  • Busybox ，通常在Linux系统启动时运行，作为initramfs的。它使用 ash ` shell实现。
  • BSD，以及一般的非Linux系统。 OpenBSD使用pdksh，它是Korn shell的后代。 FreeBSD的sh是最初的UNIX Bourne shell的后代。 Solaris有自己的sh，很长一段时间不符合POSIX标准;免费实施可从传家宝项目获得。

你怎么知道你的系统上的/ bin / sh指向了什么？

复杂的是/ bin / sh可能是一个符号链接或一个硬链接。
如果是符号链接，则可以使用便携式解决方法：<
/ p>

% file -h /bin/sh
/bin/sh: symbolic link to bash

如果是硬链接，请尝试

% find -L /bin -samefile /bin/sh
/bin/sh
/bin/bash

实际上，-L标志覆盖了符号链接和硬链接， 但这种方法的缺点是它不是便携式的 - POSIX
不要求 find来支持-samefile选项， 虽然GNU查找和 FreeBSD发现支持它。

Shebang line

最终，由你来决定使用哪一个，通过编写“shebang”这一行。

E.g。

#!/bin/sh

将使用sh（以及发生的任何指向），

#!/bin/bash

如果可用，

将使用/ bin / bash（如果不是，则会失败并显示错误消息）。当然，你也可以指定另一个实现，例如

#!/bin/dash

Which one to use

对于我自己的脚本，我更喜欢sh，原因如下：

  • 标准化
  • 学习起来要简单得多
  • 它可以跨POSIX系统移植 - 即使它们碰巧没有bash，它们也必须有sh
    使用bash也有好处。其特点使得编程更加方便和类似于其他现代编程语言的编程。这些包括像范围局部变量和数组。 Plain sh是一种非常简约的编程语言。

sh： http://man.cx/sh bash： http://man.cx/bash

TL; DR **：bash是sh的超集，具有更优雅的语法和更多的功能。几乎在所有情况下使用bash
shebang线是安全的，因为它在现代平台上是无处不在的。

注意：在某些环境中，sh _是 **bash。检查sh --version。
