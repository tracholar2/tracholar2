define（）与const

By zhizi

Published 2018-02-02

Contents

很直接的问题：在PHP中，你什么时候使用

define('FOO', 1);

你什么时候使用

const FOO = 1;

这两者之间的主要区别是什么？

从PHP 5.3开始，有两种方法可以定义常量：使用const关键字或使用 define（） 函数：

const FOO = 'BAR';
define('FOO', 'BAR');

这两种方法之间的根本区别在于const在编译时定义了常量，而define在运行时定义了它们。这会导致大部分const的缺点。 const的一些缺点是：

  • const不能用来有条件地定义常量。要定义一个全局常量，必须在最外层的范围中使用：

      if（...）{
const FOO ='BAR'; //无效

    }
    //但是
    如果（…）{

    define（'FOO'，'BAR'）; //有效

    }

你为什么要这么做呢？一个常见的应用是检查常量是否已经定义：

if（！defined（’FOO’））{ define（’FOO’，’BAR’）; }

const 接受静态标量（数字，字符串或其他常量，如 true ， false ， null code>， FILE
），而 define（）取任何表达式。由于PHP 5.6常量表达式在 const `中也是允许的：

      const BIT_5 = 1 <＆lt; 5; //自PHP 5.6起有效，以前无效
define（'BIT_5'，1 <5）; //总是有效的

const 接受一个普通的常量名，而 define（）`接受任何表达式作为名字。这允许做这样的事情：

（$ i = 0; $ i <32; ++ $ i）{

    define（'BIT_'。$ i，1 <＆lt; $ i）;
}

const 常常区分大小写，而 define（）允许你通过传递 true `来定义不区分大小写的常量。第三个参数：

define（’FOO’，’BAR’，true）; 回声FOO; // BAR 回声foo; // BAR

所以，这是事情的坏处。现在让我们来看看为什么我个人总是使用const，除非出现以上情况之一：

  • const简单地读取更好。它是一种语言结构，而不是一个函数，并且与你在类中定义常量的方式一致。
    const 在当前命名空间中定义一个常量，而 define（）`必须传递完整的命名空间名称：
     命名空间A \ B \ C;
//定义常量A \ B \ C \ FOO：
const FOO ='BAR';
define（'A \ B \ C \ FOO'，'BAR'）;

由于PHP 5.6常量const 常量也可以是数组，而define（）还不支持数组。但是在PHP 7中，这两种情况都支持数组。

      const FOO = [1，2，3]; //在PHP 5.6中有效
define（'FOO'，[1，2，3]）; //在PHP 5.6中无效，在PHP 7.0中有效

由于const是语言结构，在编译时定义，它们比define（）快一些。

众所周知，PHP define（）在使用大量的常量时速度很慢。人们甚至发明了像 apc_load_constants（）和 hidef 来解决这个问题。

const
`使常量的定义大约快一倍（在XDebug开发的机器上打开更多）。查询时间另一方面不会改变（因为两个常量类型共享相同的查找表）：演示

最后，注意const也可以用在类或接口中来定义一个类常量或接口常量。
define不能用于这个目的：

class Foo {
    const BAR = 2; // valid
}
// but
class Baz {
    define('QUX', 2); // invalid
}

<强>摘要

除非您需要任何类型的条件或表达式定义，否则仅仅为了可读性，使用const而不是define（）！

在PHP 5.3之前，const不能在全局范围中使用。你只能在课堂上使用这个。当你想要设置某种常量选项或与该类相关的设置时，应该使用这个选项。或者也许你想创建一些枚举。

define `可以用于相同的目的，但它只能在全局范围内使用。它只能用于影响整个应用程序的全局设置。

一个好的const用法的例子是摆脱幻数。看看PDO的常量。例如，当您需要指定提取类型时，可以键入PDO :: FETCH_ASSOC。如果不使用const，那么最终会输入类似于35（或者其他定义为FETCH_ASSOC）的内容。这对读者没有意义。

一个好的define用法的例子可能是指定你的应用程序的根路径或一个库的版本号。
