我想捕捉 ` var_dump < / code> 转换为字符串。

PHP文档说：

As with anything that outputs its result directly to the browser, the
output-control functions
can be used to capture the output of this function, and save it in a string
(for example).

有人可以给我一个这样的例子吗？

print_r（）`不是一个有效的可能性，因为它不会给我提供我需要的信息。

使用输出缓冲：

<?php
ob_start();
var_dump($someVar);
$result = ob_get_clean();
?>

Try

您可能希望查看 var_export ，而不要提供与var_dump相同的输出，它会提供第二个$ return参数，使其返回输出而不是打印输出：

$debug = var_export($my_var, true);

Why?

我更喜欢使用ob_start和ob_get_clean（）。我也发现输出有点容易阅读，因为它只是PHP代码。

var_dump和var_export之间的区别在于var_export返回一个变量的可解析的字符串表示形式 __ 而var_dump只是转储有关变量的信息。这在实践中意味着var_export为您提供有效的PHP代码（但是可能不会提供相当多的关于变量的信息，尤其是在您使用resources ）。

Demo:

$demo = array(
    "bool" => false,
    "int" => 1,
    "float" => 3.14,
    "string" => "hello world",
    "array" => array(),
    "object" => new stdClass(),
    "resource" => tmpfile(),
    "null" => null,
);

// var_export -- nice, one-liner
$debug_export = var_export($demo, true);

// var_dump
ob_start();
var_dump($demo);
$debug_dump = ob_get_clean();

// print_r -- included for completeness, though not recommended
$debug_printr = print_r($demo, true);

Caveat: var_export does not handle circular references

如果您正尝试转储具有循环引用的变量，则调用var_export将导致PHP警告：

$circular = array();
$circular['self'] =& $circular;
var_export($circular);

结果在：

Warning: var_export does not handle circular references in example.php on line 3
array (
  'self' => 
  array (
    'self' => NULL,
  ),
)

另一方面，当遇到循环引用时，var_dump和print_r将输出字符串* RECURSION *。
