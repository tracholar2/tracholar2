如何检查一个变量是否在Bash中设置？

By zhizi

Published 2018-02-01

Contents
  1. 1. The right way
    1. 1.1. Quotes Digression
  2. 2. The wrong way

我如何知道是否在Bash中设置了变量？

例如，如何检查用户是否给函数第一个参数？

function a {
    # if $1 is set ?
}

The right way

if [ -z ${var+x} ]; then echo "var is unset"; else echo "var is set to '$var'"; fi

其中$ {var + x}是参数扩展，如果var未被设置，那么它的计算结果为空，否则用字符串x替换。

Quotes Digression

引用可以省略（所以我们可以用$ {var + x}来代替“$ {var + x}”），因为这个语法＆amp;使用保证这只会扩展到不需要引号（因为它扩展到x（它不包含任何分词，所以它不需要引号），或者什么都不是（这导致[-z]，它可以方便地计算出和[-z“”]相同的值（true））。
然而，虽然引号可以被安全的省略，但并不是立刻就显而易见的（对于这个引用解释的第一作者也是一个主要的Bash编码器），有时使用引号将解决方案写成[-z“$ {var + x}”]，在O（1）速度惩罚的可能成本非常小。第一作者也在代码旁边添加了这个代码，使用这个解决方案给出了这个答案的URL，现在还包括解释为什么可以安全省略引号。

The wrong way

if [ -z "$var" ]; then echo "var is unset"; else echo "var is set to '$var'"; fi

这是因为它不区分未设置的变量和设置为空字符串的变量。也就是说，如果var =''，那么上面的解决方案将错误地输出该var未设置。

但是，这种区分在用户必须指定扩展名或附加属性列表的情况下是必不可少的，而不是指定它们的默认值为非空值，而指定空字符串则会使脚本使用空扩展名或附加属性列表。

检查非空/非零字符串变量，即如果设置，使用

if [ -n "$1" ]

与-z相反。我发现自己正在使用-n而不是-z。
