如何管道标准错误，而不是标准输出？

By zhizi

Published 2018-02-10

Contents

我有一个将信息写入到stdout和stderr的程序，我需要通过，而忽略 stdout 。

我当然可以分两步来做：

command > /dev/null 2> temp.file
grep 'something' temp.file

但是我宁愿能够在没有临时文件的情况下做到这一点。有没有什么巧妙的管道技巧？

首先将stderr重定向到stdout - 管道;然后将stdout重定向到/ dev / null（不改变stderr的位置）：

command 2>&1 >/dev/null | grep 'something'

有关各种I /
O重定向的详细信息，请参阅重定向在Bash参考手册中。

请注意，I / O重定向的顺序是从左向右解释的，但是在解释I / O重定向之前设置了管道。文件描述符（如1和2）是对打开的文件描述的引用。操作2＆amp; 1＆lt; / code>使得文件描述符2 aka stderr指向与stdout当前所指的文件描述符1（也就是[ dup2（）](“http：//)和[ open（）](”http://pubs.opengroup.org/onlinepubs/)）。然后，操作＆gt;
/ dev / null 更改文件描述符1，以便引用 / dev / null
`的打开文件描述，但这不会改变事实该文件描述符2是指文件描述符1最初指向的打开的文件描述 - 即管道。

或者将stderr和stdout的输出交换使用：

command 3>&1 1>&2 2>&3

创建一个新的文件描述符（3），并将其分配到1（stdout）的相同位置，然后将fd 1（stdout）分配给与fd
2（stderr）相同的位置，最后分配fd 2（stderr）与fd 3（stdout）相同的地方。
Stderr现在可以作为标准输出和旧stdout保存在stderr中。也许是矫枉过正，但希望提供更多的细节上bash文件描述符（每个进程有9个可用）。
