致命错误：允许134217728字节的内存大小用尽（CodeIgniter + XML-RPC）

By zhizi

Published 2018-02-21

Contents

我有一大堆客户销售点（POS）系统，它们定期将新的销售数据发送到一个集中的数据库，这个数据库将数据存储到一个大数据库中生成报告。

客户端POS基于PHPPOS，并且我已经实现了一个使用标准的XML-RPC库来将销售数据发送到服务的模块。服务器系统建立在CodeIgniter之上
，并为Web服务组件使用XML-RPC和XML-
RPCS库。每当我发送大量的销售数据（从销售表中只有50行，销售中每个项目的sales_items中的单个行），我得到以下错误：

Fatal error: Allowed memory size of 134217728 bytes exhausted (tried to allocate 54 bytes)

128M是php.ini中的默认值，但我认为这是一个巨大的数字。事实上，我甚至试图将这个值设置为1024M，而且所做的只是花费更长时间才能出错。

至于我采取的步骤，我已经尝试禁用服务器端的所有处理，并已经操纵它返回一个罐头响应，无论输入。但是，我认为问题在于数据的实际发送。我甚至尝试禁用PHP的最大脚本执行时间，但仍然出错。

人们通过ini_set（'memory_limit'，'-1'）改变memory_limit是 NOT 的解决方案。

请不要这样做。很明显，PHP有一个内存泄漏的地方，你告诉服务器只使用它想要的所有内存。问题一直没有得到解决。如果你监视你的服务器，你会发现它现在可能已经占用了大部分内存，甚至交换到磁盘。

您应该试着追踪代码中的确切错误并修复它。
