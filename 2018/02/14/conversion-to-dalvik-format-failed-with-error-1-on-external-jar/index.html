外部JAR上的“转换为Dalvik格式失败，出现错误1”

By zhizi

发表于 2018-02-14

文章目录

在Eclipse中我的Android应用程序中出现以下错误。

UNEXPECTED TOP-LEVEL EXCEPTION:
java.lang.IllegalArgumentException: already added:
Lorg/xmlpull/v1/XmlPullParser;
….
Conversion to Dalvik format failed with error 1

只有在将特定的外部JAR文件添加到我的项目时，才会出现此错误。我花了很长时间寻找可能的解决方案，但没有一个可行的解决方案。我甚至试图改为Android
1.6而不是1.5（我使用的当前版本）。

转到项目»属性»Java构建路径»库，并删除所有除“Android X.Y”（在我的情况下Android
1.5）。单击确定。转到Project»Clean»下面选择的Clean项目»选择您的项目，然后单击OK。这应该有效。

也可能你的项目文件夹中有一个JAR文件（我已经将Admob JAR文件复制到了我的src文件夹中），然后将它添加为Java Path
Library。它没有显示在Package Explorer下，所以你不会注意到它，但是它会被计算两次，导致可怕的Dalvik错误1。

另一个可能的原因可能是软件包名称冲突。假设在这个包内有一个包com.abc.xyz和一个名为A.java的类，另一个库项目（添加到这个项目的依赖项中）其中包含相同的com.abc.xyz.A.java，那么你将得到完全相同的错误。这意味着，您有多个对同一个文件A.java的引用，并且无法正确地构建它。

在其他方面，这可能会发生，如果你不小心或有意编辑/手动添加类路径文件中的任何东西。在某些情况下，我们可能会手动添加android.jar路径到类路径文件生成java
doc.On删除在javadoc生成的代码将正常工作。请检查这一点，如果任何一个仍然发生。

我解决了这个问题。

这是一个JAR文件冲突。

看起来我的构建路径上有两个JAR文件，它们包含相同的包和类。

smack.jar和android_maps_lib-1.0.2
从一个JAR文件中删除这个包解决了这个问题。
