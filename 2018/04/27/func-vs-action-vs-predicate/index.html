用真实的例子和他们的用法，有人可以帮我理解：

  • 我们什么时候需要Func委托？
  • 我们什么时候需要动作委托？
  • 我们什么时候需要Predicates委托？

Func 和 Action 的不同之处在于您是否希望代理返回值（使用 Func `）或不使用<代码>动作）。

Func在LINQ中可能是最常用的 - 例如在投影中：

list.Select(x => x.SomeProperty)

或过滤：

list.Where(x => x.SomeValue == someOtherValue)

或按键选择：

list.Join(otherList, x => x.FirstKey, y => y.SecondKey, ...)

Action更常用于像List＆lt; T＆gt ;.ForEach之类的东西：对列表中的每个项目执行给定的操作。我使用这个比Func的次数要少，尽管我们有时候会使用无参数的版本，比如Control.BeginInvoke和` Dispatcher.BeginInvoke

Predicate实际上是在所有的Func和大部分之前引入的一个特殊的 Func＆lt; T，bool＆gt;

行动代表来了。我怀疑，如果我们已经有了 Func 和 Action 各种形式， Predicate 就不会被引入...尽管它 __ 给委托的使用赋予了一定的含义，而 Func 和 Action `用于广泛不同的用途。

Predicate主要用于FindAll和RemoveAll等方法的List＆lt; T＆gt;。 > Action
**是一个方法的委托（指针），它接受零个，一个或多个输入参数，但不返回任何内容。

Func **是一个方法的委托（指针），它接受零个，一个或多个输入参数，并返回一个值（或引用）。

Predicate **是一种经常用于比较的特殊类型的Func。

尽管Linq广泛使用，但Action和Func是逻辑上独立于Linq的概念。 C ++已经包含了类型化函数指针形式的基本概念。

下面是一个Action和Func的小例子，不使用Linq：

class Program
{
    static void Main(string[] args)
    {
        Action<int> myAction = new Action<int>(DoSomething);
        myAction(123);           // Prints out "123"
                                 // can be also called as myAction.Invoke(123);

        Func<int, double> myFunc = new Func<int, double>(CalculateSomething);
        Console.WriteLine(myFunc(5));   // Prints out "2.5"
    }

    static void DoSomething(int i)
    {
        Console.WriteLine(i);
    }

    static double CalculateSomething(int i)
    {
        return (double)i/2;
    }
}
