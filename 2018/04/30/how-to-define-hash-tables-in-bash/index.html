Bash 4

Bash 4本身支持这个功能。确保脚本的hashbang是<！code>＃！/ usr / bin / env bash 或者＃！/ bin /
bash 或者任何引用 bash 不是 sh 。确保你正在执行你的脚本，而不是像 sh script 那样做一些愚蠢的事情，这会导致你的bash ` hashbang被忽略。这是基本的东西，但是这么多的东西都在失败，所以重新进行重复。

你通过这样做来声明一个关联数组：

declare -A animals

您可以使用正常的数组赋值操作符来填充元素：

animals=( ["moo"]="cow" ["woof"]="dog")

或者合并它们：

declare -A animals=( ["moo"]="cow" ["woof"]="dog")

然后像普通数组一样使用它们。 “$ {animals [@]}”展开了值，“$ {！animals [@]}”（注意！`）展开按键。不要忘记引用他们：

echo "${animals[moo]}"
for sound in "${!animals[@]}"; do echo "$sound - ${animals[$sound]}"; done

Bash 3

在bash 4之前，你没有关联数组。 请勿使用eval来模拟它们
。你必须避免像鼠疫这样的eval，因为它是shell脚本的瘟疫。最重要的原因是你不想把你的数据当作可执行代码（还有很多其他原因）。

首先要考虑的是：考虑升级到bash 4.认真的。现在的未来就是停止生活在过去，并且迫使你的代码被愚蠢的破坏和丑陋的黑客所束缚，每一个可怜的灵魂都会坚持下去。

如果你为什么“ 无法升级 ”有一些愚蠢的借口，那么declare是一个更安全的选择。它并不像bash代码那样评估数据，就像eval那样，所以它不允许任意的代码注入非常容易。

让我们通过引入这些概念来准备答案：

首先，间接的（认真的;除非你是精神病患者，或者有其他不好的借口来写这些黑客），否则不要使用它。

$ animals_moo=cow; sound=moo; i="animals_$sound"; echo "${!i}"
cow

其次，declare：

$ sound=moo; animal=cow; declare "animals_$sound=$animal"; echo "$animals_moo"
cow

把它们放在一起：

# Set a value:
declare "array_$index=$value"

# Get a value:
arrayGet() { 
    local array=$1 index=$2
    local i="${array}_$index"
    printf '%s' "${!i}"
}

让我们使用它：

$ sound=moo
$ animal=cow
$ declare "animals_$sound=$animal"
$ arrayGet animals "$sound"
cow

注意：declare不能放在函数中。任何在bash函数中使用declare都会把它创建的变量 local
变成该函数的作用域，这意味着我们不能用它来访问或修改全局数组。 （在bash 4中，你可以使用declare -g声明全局变量 - 但是在bash
4中，你应该首先使用关联数组，而不是这个黑客。）

Summary

升级到bash 4并使用declare -A。如果你不能，那么在做如上所述的丑陋的黑客之前，请考虑完全切换到awk。而且肯定会远离eval hackery。

有参数替换，尽管也可能是un-PC，就像间接方式一样。

#!/bin/bash

# Array pretending to be a Pythonic dictionary
ARRAY=( "cow:moo"
        "dinosaur:roar"
        "bird:chirp"
        "bash:rock" )

for animal in "${ARRAY[@]}" ; do
    KEY="${animal%%:*}"
    VALUE="${animal##*:}"
    printf "%s likes to %s.\n" "$KEY" "$VALUE"
done

printf "%s is an extinct animal which likes to %s\n" "${ARRAY[1]%%:*}" "${ARRAY[1]##*:}"

BASH 4的方式当然更好，但是如果你需要黑客，只有黑客才能做到。 你可以用类似的技术搜索数组/散列。
