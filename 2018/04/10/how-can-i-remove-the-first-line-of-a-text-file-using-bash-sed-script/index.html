如何使用bash / sed脚本删除文本文件的第一行？

By zhizi

Published 2018-04-10

Contents

我需要使用bash脚本从一个巨大的文本文件中反复删除第一行。

现在我正在使用sed -i -e“1d”$ FILE - 但删除操作大概需要一分钟时间。

有没有更有效的方法来完成这个？

试试 GNU tail ：

tail -n +2 "$FILE"

-n x：只需打印最后一个x行。 tail -n 5 会给你输入的最后5行。 + 符号类型反转参数，并使 tail
输出第一个 x-1 行。 tail -n + 1 会打印整个文件， -n + 2所有内容除了第一行外，等等。

GNU tail比sed快得多。 tail在BSD上也是可用的，-n +2标志在两个工具中都是一致的。检查 FreeBSD或 OS X 手册页。

不过，BSD版本可能比sed慢得多。我想知道他们是如何管理的。 tail应该只是逐行读取一个文件，而sed执行相当复杂的操作，包括解释脚本，应用正则表达式等等。

注意：您可能会使用

# THIS WILL GIVE YOU AN EMPTY FILE!
tail -n +2 "$FILE" > "$FILE"

但是这会给你一个 空文件 。原因是重定向（＆gt;）发生在shell调用tail之前：

  • Shell截断文件$ FILE * Shell为tail创建一个新进程

  • Shell将tail进程的stdout重定向到$ FILE * tail从现在空的$ FILE读取

如果你想删除文件中的第一行，你应该使用：

tail -n +2 "$FILE" > "$FILE.tmp" && mv "$FILE.tmp" "$FILE"

＆lt; code＆amp;＆amp; `将确保文件在出现问题时不会被覆盖。

对于那些非GNU的SunOS来说，下面的代码会有帮助：

sed '1d' test.dat > tmp.dat
