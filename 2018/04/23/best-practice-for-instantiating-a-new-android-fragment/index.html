实例化新的Android Fragment的最佳做法

By zhizi

Published 2018-04-23

Contents

我已经看到了两种在应用程序中实例化新片段的一般做法：

Fragment newFragment = new MyFragment();

和

Fragment newFragment = MyFragment.newInstance();

第二个选项使用静态方法newInstance（）和 通常 包含以下方法。

public static Fragment newInstance() 
{
    MyFragment myFragment = new MyFragment();
    return myFragment;
}

首先，我认为主要的好处是我可以重载newInstance（）方法来创建Fragment的新实例时具有灵活性 -
但是我也可以通过为Fragment创建一个重载的构造函数来实现这一点。 / p> 我错过了什么？

一种方法比另一种方法有什么好处？或者这只是一个好习惯吗？

如果以后Android决定重新创建你的Fragment，它将会调用你片段的无参数构造函数。所以重载构造函数不是一个解决方案。

就这么说，把东西传递给你的Fragment的方式，以便在Android重新创建Fragment后可用，就是把一个bundle传递给setArguments方法。

因此，例如，如果我们想要传递一个整数给片段，我们可以使用类似于：

public static MyFragment newInstance(int someInt) {
    MyFragment myFragment = new MyFragment();

    Bundle args = new Bundle();
    args.putInt("someInt", someInt);
    myFragment.setArguments(args);

    return myFragment;
}

稍后在Fragment onCreate（）中，您可以使用以下命令访问该整数：

getArguments().getInt("someInt", 0);

即使片段被Android重新创建，这个Bundle也是可用的。

另请注意：setArguments只能在Fragment附加到Activity之前调用。

在android开发人员参考中也记录了这种方法： https：// developer .android.com /参考/机器人/应用/
Fragment.html

使用我看到的newInstance（）的唯一好处是：

  • 您将拥有一个地方，可以将碎片使用的所有参数捆绑起来，而且每次实例化碎片时都不必写下面的代码。

    Bundle args = new Bundle（）;

    args.putInt（“someInt”，someInt）;
    args.putString（“someString”，someString）;
    //放置任何其他参数
    myFragment.setArguments（参数）;

p>

因此，我认为使用静态newInstance（）来实例化一个片段是一个很好的做法。
