NoSQL 解决方案在那里用于.NET？

您也应该考虑使用 Redis。这是一个先进的NoSQL数据库，支持丰富的服务器端数据结构，如列表，集合，排序集合和哈希。它在内存中运行，但同时支持“快照”持久性模式以及仅追加文件日志以实现快照之间的完整性。

Simple and Elegant

像大多数NoSQL数据存储一样，Redis无模式允许您直接使用它，而无需事先定义任何模式，从而大大提高生产力。
Redis的美妙之处就在于它像访问C＃中习惯的.NET泛型集合类，所以只需要很少的投影，因为它的API已经与List和内存中的数据结构紧密匹配程序已经使用。

正是这种提供基本的comp-
sci数据结构的优点，使得Redis具有了独特的优势，它使其具有极强的通用性，同时仅支持有限的操作集。即Redis通常被看作是分布式缓存，消息队列，语言/进程之间的IPC代理，Pub
/
Sub，推/基于事件的通信，分布式锁定，CQRS事件源，唯一ID生成器等。不要使用它作为你的主要数据存储，它的速度和简单性使你能够访问一个多功能的瑞士军队工具箱，可以满足大量的使用情况。

另一个完全使用Redis构建的实际应用程序的另一个示例是 Redis StackOverflow。

示例代码显示完整的CRUD应用程序：

public class Todo
{
    public long Id { get; set; }
    public string Content { get; set; }
    public int Order { get; set; }
    public bool Done { get; set; }
}

//Thread-safe client factory
var redisManager = new PooledRedisClientManager("localhost:6379");

redisManager.ExecAs<Todo>(redisTodos => {
    var todo = new Todo {
        Id = redisTodos.GetNextSequence(),
        Content = "Learn Redis",
        Order = 1,
    };

    redisTodos.Store(todo);

    Todo savedTodo = redisTodos.GetById(todo.Id);
    savedTodo.Done = true;

    redisTodos.Store(savedTodo);

    redisTodos.DeleteById(savedTodo.Id);

    var allTodos = redisTodos.GetAll();

    Assert.That(allTodos.Count, Is.EqualTo(0));
});

Runs on all languages and platforms

由于它的简单性，它几乎适用于今天使用的所有语言的语言绑定： http://redis.io/clients

在动态语言中它更简洁，创建一个非阻塞的Web服务器托管命名计数器是多么容易在node.js
/ CoffeeScript中：

app.get '/counters/:id', (request, response) ->
    id = request.params.id
    redis.incr id, (err, val) ->
        response.send "#{id}: #{val}", 'Content-Type': 'text/plain', 201

NoSQL的好处之一是您可以与来自不同的开发者语言背景，共同研究如何最好地利用redis来满足我们的要求。
