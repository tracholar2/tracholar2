我们什么时候需要在shell变量上花括号？

By zhizi

发表于 2018-04-07

文章目录

在shell脚本中，扩展变量时什么时候使用{}？

例如，我看到以下内容：

var=10        # Declare variable

echo "${var}" # One use of the variable
echo "$var"   # Another use of the variable

有一个显着的差异，还是只是风格？比其他人更喜欢吗？

在这个特定的例子中，它没有区别。但是，如果要在字符串中展开变量foo，{}中的{}

"${foo}bar"

因为“$ foobar”会改为扩展foobar。

在下列情况下，大括号也是无条件要求的：

  • 展开数组元素，如$ {array [42]}所示

  • 使用参数扩展操作，如$ {filename％。*}（删除扩展名）

  • 扩展位置参数超过9：“$ 8 $ 9 $ {10} $ {11}”
    做到这一点，而不是仅仅在可能含糊的情况下，可以被认为是良好的编程实践。这是为了保持一致性，并避免像$ foo_ $ bar.jpg这样的惊喜，在这种情况下，下划线变成变量名的一部分并不明显。

变量是声明和赋值的，没有$，也没有{}。你必须使用

var=10

分配。为了读取变量（换句话说，“展开”变量），您必须使用$。

$var      # use the variable
${var}    # same as above
${var}bar # expand var, and append "bar" too
$varbar   # same as ${varbar}, i.e expand a variable called varbar, if it exists.

这有时使我感到困惑 - 在其他语言中，我们用相同的方式引用变量，而不管它是在作业的左边还是右边。但是shell脚本是不同的，$ var = 10不会做你可能会认为它的做法！
