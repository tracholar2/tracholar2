如何在Magento中创建一个简单的“Hello World”模块？

文章目录

如何在Magento中完成以下工作？

  • 使用控制器/视图/模型方法显示“Hello World”消息。所以，如果我去http://example.com/myController，它会显示字符串“Hello World”。能够在我的网站模板中显示这个字符串（例如，页眉，页脚等）将是一个奖励。

  • 如何添加一个方法到这个控制器（或一个新的控制器，如果有必要的话），它与模型进行交互，并执行查询选择* FROM文章，其中id = '10'并将该行（包含列id，title，content）返回给控制器？然后使用控制器来包含一个视图，这将显示此行。所以去http://example.com/myController/show_row（或类似的东西）会显示视图中的行。 （不需要很花哨，只需要一个echo $ row-＆gt; id;或类似的东西就行了）。

## Create a module

所有Magento的添加和定制都是通过模块完成的。所以，你需要做的第一件事是创建一个新的模块。在app / modules中创建一个名为

的XML文件



cd /path/to/store/app
touch etc/modules/MyCompanyName_HelloWorld.xml



<?xml version=”1.0”?>



true
local





MyCompanyName是你修改的唯一名称空间，它不一定是你公司的名字，而是推荐的公约my magento。 HelloWorld是你的模块名称。

## Clear the application cache

现在模块文件已经存在，我们需要让Magento知道它（并检查我们的工作）。在管理员应用程序中 转到系统 - ＆gt;高速缓存管理
从“全部高速缓存”菜单中选择刷新 点击保存缓存设置

现在，我们确保Magento知道模块

转至系统 - ＆gt;配置 点击高级
* 在“禁用模块输出”设置框中，查找名为“MyCompanyName_HelloWorld”的新模块

如果你能忍受性能下降，你可能想在开发/学习的时候关闭应用程序缓存。没有什么比这更令人沮丧的了，然后忘记清除缓存，并想知道为什么你的更改没有显示出来。

## Setup the directory structure

接下来，我们需要为模块设置一个目录结构。你不需要所有的这些目录，但现在把它们全部设置好就没有什么害处了。



mkdir -p app/code/local/MyCompanyName/HelloWorld/Block
mkdir -p app/code/local/MyCompanyName/HelloWorld/controllers
mkdir -p app/code/local/MyCompanyName/HelloWorld/Model
mkdir -p app/code/local/MyCompanyName/HelloWorld/Helper
mkdir -p app/code/local/MyCompanyName/HelloWorld/etc
mkdir -p app/code/local/MyCompanyName/HelloWorld/sql


然后添加一个配置文件



touch app/code/local/MyCompanyName/HelloWorld/etc/config.xml


并在配置文件中添加以下内容，这本质上是一个“空白”配置。



<?xml version=”1.0”?>



0.1.0





简单地说，这个配置文件会让你告诉Magento你想要运行的代码。

## Setting up the router

接下来，我们需要设置模块的路由器。这会让系统知道我们正在以

的形式处理任何网址



http://example.com/magento/index.php/helloworld


所以，在您的配置文件中，添加以下部分。




    <frontend>
        <routers>
            <!-- the <helloworld> tagname appears to be arbitrary, but by
            convention is should match the frontName tag below-->
            <helloworld>
                <use>standard</use>
                <args>
                    <module>MyCompanyName_HelloWorld</module>
                    <frontName>helloworld</frontName>
                </args>
            </helloworld>
        </routers>
    </frontend>
<!-- ... -->
</config>

你在这里说的是“任何具有helloworld的正面名称的URL …

http://example.com/magento/index.php/helloworld

应该使用frontName控制器MyCompanyName_HelloWorld“。

所以，通过上面的配置，当你加载上面的helloworld页面，你会得到一个404页面。那是因为我们还没有为我们的控制器创建一个文件。我们现在就做。

touch app/code/local/MyCompanyName/HelloWorld/controllers/IndexController.php

现在尝试加载页面。进展！而不是404，你会得到一个PHP / Magento异常

Controller file was loaded but class does not exist

所以，打开我们刚创建的文件，并粘贴下面的代码。该类的名称需要基于您在路由器中提供的名称。

<?php
class MyCompanyName_HelloWorld_IndexController extends Mage_Core_Controller_Front_Action{
    public function indexAction(){
        echo "We're echoing just to show that this is what's called, normally you'd have some kind of redirect going on here";
    }
}

我们刚刚设置的是模块/ frontName控制器。 这是模块的默认控制器和默认操作。 如果你想添加控制器或动作，你必须记住，一个Magento
URL的树第一部分是不可变的，他们总是会这样`http://example.com/magento/index.php/frontName/controllerName / actionName

所以如果你想匹配这个网址

http://example.com/magento/index.php/helloworld/foo

你必须有一个FooController，你可以这样做：

touch app/code/local/MyCompanyName/HelloWorld/controllers/FooController.php



<?php
class MyCompanyName_HelloWorld_FooController extends Mage_Core_Controller_Front_Action{
    public function indexAction(){
        echo 'Foo Index Action';
    }

    public function addAction(){
        echo 'Foo add Action';
    }

    public function deleteAction(){
        echo 'Foo delete Action';
    }
}

请注意，默认控制器IndexController和默认操作indexAction可以通过隐式，但必须是明确的，如果之后有什么。 因此，http://example.com/magento/index.php/helloworld/foo将匹配控制器FooController和动作indexAction，而不是IndexController的动作fooAction。如果你想有一个fooAction，在控制器IndexController中，你必须像这样明确地调用这个控制器：
http://example.com/magento/index.php/helloworld/index/foo，因为URL的第二部分是并且将始终是controllerName。 这种行为是Magento捆绑的Zend Framework的继承。

您现在应该可以点击以下网址并查看回声声明的结果

http://example.com/magento/index.php/helloworld/foo
http://example.com/magento/index.php/helloworld/foo/add
http://example.com/magento/index.php/helloworld/foo/delete

远离核心文件。不要修改它们，而是编写自己的模块并覆盖你需要的内容。

Magento使用由XML组成的配置文件来决定它需要做什么。为了得到它运行自己的东西，而不是核心功能，你需要正确的XML。不幸的是，没有关于如何构建XML的指南。你需要看例子，做一些严肃的测试。使这些文件的内容复杂化很大程度上是区分大小写的。但是，如果您掌握了这些功能，您可以覆盖基本功能的任何部分，从而形成非常强大的系统。

Magento使用像Mage :: getModel（’mymodel’）， Mage ::
getSingleton（’mysingleton’）， Mage :: helper（’
myhelper’）来返回某些类的对象。它在默认的核心命名空间中找到这些。如果你想使用你自己的，你需要在你的 config.xml `文件中覆盖它们。

您的课程名称必须与其所在的文件夹相符。

Magento中的很多对象最终扩展了一个名为Varien_Object的东西。这是一个通用的课程（有点像瑞士军刀），它的目的是让你在飞行中定义自己的方法/变量。例如，你会看到它被用作一个荣耀的数组来传递数据从一个方法到另一个。

在开发过程中，请确保您的缓存已禁用。它会使得极慢的极端，但它会为您节省很多的头部创伤（从你的桌子上撞到它）。

您会看到$ this被大量使用。这意味着一个不同的类取决于你看到的文件。 get_class（$ this）是你的朋友，特别是与FirePHP结合使用。
