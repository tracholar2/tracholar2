array_map，array_walk和array_filter之间的区别

By zhizi

发表于 2018-04-03

文章目录

array_map，array_walk和array_filter之间的区别究竟是什么。从文档中我可以看到，你可以传递一个回调函数来对提供的数组执行操作。但我似乎没有发现他们之间的任何特别的区别。

他们执行同样的事情吗？ 它们可以互换使用吗？

如果他们完全不同，我将非常感激您的帮助。

  • array_map 没有附带效果，而 array_walk 可以;特别是 array_map 永远不会改变它的参数。
  • array_map 不能使用数组键， array_walk 可以。
  • array_map 返回一个数组， array_walk 仅返回true / false。因此，如果你不想因为遍历一个数组而创建一个数组，你应该使用 array_walk代码> 。
  • array_map 也可以接收任意数量的数组，而 array_walk 只能在一个上运行。
  • array_walk 可以接收一个额外的任意参数传递给回调函数。从PHP 5.3开始，这种做法基本上是不相干的（当时引入了匿名函数） / LI>
  • array_map / array_walk 的结果数组与参数的元素数目相同。 [ array_filter ` ](http://php.net/array_filter)根据过滤函数仅挑选数组元素的一个子集。它确实保存了密钥。

示例：

<pre>
<?php

$origarray1 = array(2.4, 2.6, 3.5);
$origarray2 = array(2.4, 2.6, 3.5);

print_r(array_map('floor', $origarray1)); // $origarray1 stays the same

// changes $origarray2
array_walk($origarray2, function (&$v, $k) { $v = floor($v); }); 
print_r($origarray2);

// this is a more proper use of array_walk
array_walk($origarray1, function ($v, $k) { echo "$k => $v", "\n"; });

// array_map accepts several arrays
print_r(
    array_map(function ($a, $b) { return $a * $b; }, $origarray1, $origarray2)
);

// select only elements that are > 2.5
print_r(
    array_filter($origarray1, function ($a) { return $a > 2.5; })
);

?>
</pre>

[将函数映射到数据数组的功能来自功能节目。你不应该把array_map想象成一个在数组的每个元素上调用一个函数的foreach循环（即使它是如何实现的）。它应该被认为是独立地将函数应用于数组中的每个元素。

在理论上，像函数映射这样的事情可以并行完成，因为应用于数据的函数只应该影响数据而不影响全局状态。这是因为array_map可以选择将函数应用于项目的任何顺序（即使在PHP中它不会）。

另一方面，arraywalk 与处理数据数组完全相反。它不是分别处理每个项目，而是使用一个状态（＆amp; $ userdata
），并且可以编辑项目（很像foreach循环）。由于每次有一个项目应用了 $ funcname
`，它就可以改变程序的全局状态，因此需要一个正确的处理项目的方法 p>

在PHP领域，array_map和array_walk除了array_walk以外，几乎都是相同的，可以更好地控制数据的迭代，在原地“更改”数据，并返回一个新的“更改”数组。

array_filter 实际上是 array_walk （或 array_reduce `）的一个应用程序，只是为了方便而提供的。 >
