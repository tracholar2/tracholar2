如何避免反向工程的APK文件？

By zhizi

Published 2018-04-02

Contents

我正在为Android开发 支付处理应用程序 ，我想阻止黑客访问 APK档案。

如果有人将.apk扩展名更改为.zip，则可以将其解压缩并轻松访问所有应用程序的资源和资源，并使用 dex2jar和一个Java反编译器，他们也可以访问源代码。这是一个非常容易反向工程Android
APK文件 - 更多详细信息请参阅堆栈溢出问题
从APK文件反向工程到项目 。

我使用了Android
SDK提供的Proguard工具。当我使用签名的密钥库和Proguard对生成的APK文件进行反向工程时，我得到了混淆的代码。但是，Android组件的名称保持不变，某些代码（如应用程序中使用的键值）保持不变。根据Proguard文档，该工具不能混淆清单文件中提到的组件。

现在我的问题是：

  • 如何 彻底避免 Android APK的逆向工程？这可能吗？
  • 如何保护所有应用程序的资源，资产和源代码，使黑客无法以任何方式破解APK文件？
  • 有没有办法使黑客更加困难甚至不可能？ 我还能做些什么来保护APK文件中的源代码？

1. How can I completely avoid reverse engineering of an Android APK? Is
this possible?

AFAIK，完全避免逆向工程是没有任何窍门的。

而且@inazaruk也说得很对：无论你对代码做什么，潜在的攻击者都能够以任何方式改变它，或者他认为它是可行的_。你基本上不能保护你的应用程序不被修改。并且您放入的任何保护都可以被禁用/删除。

2. How can I protect all the app’s resources, assets and source code so
that hackers can’t hack the APK file in any way?

尽管你可以做不同的技巧来使黑客更难。例如，使用混淆（如果它是Java代码）。这通常会显着减缓逆向工程。

3. Is there a way to make hacking more tough or even impossible? What more
can I do to protect the source code in my APK file?

正如大家所说，你可能知道，没有100％的安全性。但是，Google已经建立起来的Android平台是ProGuard。如果您可以选择包含 共享库
，则可以在C ++中包含所需的代码来验证文件大小，集成， 等等。如果您需要将外部本机库添加到每个版本的APK库文件夹中， 那么你可以通过下面的建议来使用它。

将库放在默认为“libs”的本机库路径中 你的项目文件夹。如果您为 ‘armeabi’ 目标构建了本机代码，则将其放入 在 libs /
armeabi 下。如果它是用 armeabi-v7a 构建的，那就把它放在下面 的 **库/ armeabi-V7A。

<project>/libs/armeabi/libstuff.so

AFAIK，你不能保护/ res目录中的文件，而不是保护它们。

然而，有些步骤可以保护您的源代码，或者至少是什么，如果不是所有的东西。

  • 使用ProGuard之类的工具。这些会混淆你的代码，反编译时难以阅读，甚至是不可能的
  • 将服务中最关键的部分移出应用程序，然后放到一个web服务中，隐藏在PHP之类的服务器端语言之后。例如，如果你有一个算法，你花了一百万美元来写。你显然不希望有人偷你的应用程序。移动算法，让它在远程服务器上处理数据，并使用应用程序简单地提供数据。或者使用NDK将它们本地写入到.so文件中，这些文件比反编译的可能性要小得多。我不认为.so文件的反编译器现在甚至还存在（即使这样做，也不会像Java反编译器那样好）。另外，正如评论中提到的@nikolay，你应该在服务器和设备之间进行交互时使用SSL。
  • 在设备上存储值时，请勿将其存储为原始格式。例如，如果您有一个游戏，并且您正在SharedPreferences中存储用户所拥有的游戏货币数量。我们假设它是10000硬币。直接保存10000，而不是使用（（currency * 2）+1）/ 13等算法进行保存。因此，而不是10000，您将1538.53846154保存到SharedPreferences中。然而，上面的例子并不完美，你得努力想出一个方程式，不会因为错误而损失货币。
  • 您可以为服务器端任务做类似的事情。现在举一个例子，让我们来看看您的付款处理应用程序。假设用户必须支付$ 200。而不是发送一个原始的$ 200值给服务器，发送一系列小的，预定义的值，加起来到$ 200。例如，在您的服务器上有一个文件或表格，将文字与值相等。所以我们假设Charlie对应于$ 47和John到$ 3。因此，不要发送$ 200，您可以四次发送Charlie和John四次。在服务器上，解释他们的意思，并把它加起来。这可以防止黑客发送任意值到你的服务器，因为他们不知道什么字对应什么值。作为安全措施的一个附加措施，您也可以使用类似于第3点的公式，并在每个n天数内更改关键字。
    最后，你可以在应用程序中插入随机无用的源代码，以便黑客在干草堆中寻找针。插入随机类，包含来自互联网的片段，或者只是用于计算随机事件的函数，如斐波那契数列。确保这些类被编译，但不被应用程序的实际功能使用。添加足够的这些错误的类，黑客将难以找到真正的代码。

总而言之，没有办法保护你的应用程序100％。你可以让它更难，但不是不可能的。您的网络服务器可能会受到攻击，黑客可以通过监控多个交易金额和您为其发送的关键字来找出您的关键字，黑客可以通过源代码仔细查看哪些代码是虚拟的。

你只能反击，而不能赢。
