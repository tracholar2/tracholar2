我想杀死整个进程树。使用任何通用脚本语言来做这件事的最好方法是什么？我正在寻找一个简单的解决方案。

你不会说你想杀的树是不是一个进程组。 （如果树是从服务器启动或shell命令行分叉的结果，则通常是这种情况。）您可以使用GNU
ps发现进程组，如下所示：

ps x -o  "%p %r %y %x %c "

如果它是一个你想杀的进程组，只需使用kill（1）命令，而不是给它一个进程号，给它 否定 组号。例如，要杀死组5112中的每个进程，请使用kill -TERM--5112。

  • PGID = $（ps opgid =“$ PID”） =＆gt;从树的任何 Process-ID 检索 Process-Group-ID ，而不仅仅是 Process-Parent-ID 。其中 ps opgid = $ PID 的变体是 ps -o pgid --no-headers $ PID strong> pgid 可以被 pgrp ** 替换。但是： 当PID小于5位数字并右对齐时，

  • grep -o [0-9] * 仅打印连续的数字（不打印空格或按字母顺序的标题）。

Further command lines

PGID=$(ps -o pgid= $PID | grep -o [0-9]*)
kill -TERM -"$PGID"  # kill -15
kill -INT  -"$PGID"  # correspond to [CRTL+C] from keyboard
kill -QUIT -"$PGID"  # correspond to [CRTL+\] from keyboard
kill -CONT -"$PGID"  # restart a stopped process (above signals do not kill it)
sleep 2              # wait terminate process (more time if required)
kill -KILL -"$PGID"  # kill -9 if it does not intercept signals (or buggy)

Limitation

  • 正如 davide 和 Hubert Kario ，当 kill 由属于同一个树的进程调用时， kill 本身在结束整个树木杀害之前。
    因此，请务必使用具有不同 Process-Group-ID 的进程运行该命令。
