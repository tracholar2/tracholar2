如何在PHP中定义一个空对象

By zhizi

Published 2018-04-26

Contents

用一个新的数组我可以这样做：

$aVal = array();

$aVal[key1][var1] = "something";
$aVal[key1][var2] = "something else";

有没有类似的对象语法

(object)$oVal = "";

$oVal->key1->var1 = "something";
$oVal->key1->var2 = "something else";

$x = new stdClass();

手册中的评论总结最好：

stdClass is the default PHP object. stdClass has no properties, methods or
parent. It does not support magic methods, and implements no interfaces.

>

When you cast a scalar or array as Object, you get an instance of stdClass.
You can use stdClass whenever you need a generic object instance.

创建“空”对象的标准方式是：

$oVal = new stdClass();

但是，如果PHP＆gt; = 5.4，我个人更喜欢使用：

$oVal = (object)[];

它更短，我个人认为它更清晰，因为stdClass _可能会误导新手程序员（即“嘿，我想要一个对象，而不是一个类！”）…

与PHP相同＆lt; 5.4是：

$oVal = (object) array();

[] 等价于 new stdClass（）`。**

请参阅PHP手册（此处）：

stdClass : Created by typecasting to object.

和（此处）
：

If an object is converted to an object, it is not modified. If a value of
any other type is converted to an object, a new instance of the stdClass
built-in class is created.

然而请记住， 空（$ oVal）返回false ，因为@PaulP说：

Objects with no properties are no longer considered empty.

关于你的例子，如果你写：

$oVal = new stdClass();
$oVal->key1->var1 = "something"; // PHP creates  a Warning here
$oVal->key1->var2 = "something else";

PHP创建下面的Warning，隐式创​​建属性key1（一个对象本身）

Warning: Creating default object from empty value

如果您的配置可能会出现问题（请参阅错误报告级别）向浏览器显示此警告。这是另一个完整的主题，但是一个快速和肮脏的方法可以使用错误控制运算符（@）忽略警告：

$oVal = new stdClass();
@$oVal->key1->var1 = "something"; // the warning is ignored thanks to @
$oVal->key1->var2 = "something else";
