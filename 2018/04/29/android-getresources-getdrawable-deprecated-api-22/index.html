Android getResources（）。getDrawable（）不推荐使用API​​ 22

使用新的android API 22 getResources（）。getDrawable（）现在已经被弃用了。 现在最好的方法是仅使用getDrawable（）。

什么改变？

您有一些选项可以根据您正在加载的是哪种类型的drawable来处理这种弃权（ future proof ）的方式：

用_主题属性

绘制 __

ContextCompat.getDrawable(getActivity(), R.drawable.name);

您的活动主题将指示您将获得一个样式的Drawable。 这可能是你需要的。

B）** 无需_主题属性

即可绘制 __

ResourcesCompat.getDrawable(getResources(), R.drawable.name, null);

你会用旧的方式得到你的无风格的drawable。请注意：ResourcesCompat.getDrawable（）不是 **不推荐使用！

另外一个 主题的主题属性

ResourcesCompat.getDrawable(getResources(), R.drawable.name, anotherTheme);

Edit: see [my blog post](

/contextcompat-getcolor-getdrawable.html) on the subject for a more complete
explanation

您应该使用支持库中的以下代码：

ContextCompat.getDrawable(context, R.drawable.***)

使用此方法相当于调用：

if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.LOLLIPOP) {
    return resources.getDrawable(id, context.getTheme());
} else {
    return resources.getDrawable(id);
}

从API 21起，您应该使用getDrawable（int，Theme）方法而不是getDrawable（int），因为它允许您获取关联的可绘制对象给定屏幕密度/主题的特定资源ID。调用已弃用的getDrawable（int）方法相当于调用getDrawable（int，null）。
