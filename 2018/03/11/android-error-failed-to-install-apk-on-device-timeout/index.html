Android错误：无法在设备上安装* apk *：超时

By zhizi

Published 2018-03-11

Contents

我不时得到这个错误，不知道是什么原因造成的：
当试图运行/调试一个真正的设备上的Android应用程序（在我的情况下银河三星S）我在控制台中收到以下错误：

Failed to install .apk on device :

>

timeout Launch canceled!

这是所有的控制台告诉我。 LogCat不提供任何信息。 Eclipse Problems视图没有显示任何问题。

我尝试了以下步骤，但没有成功：
1.清理项目（项目 - >清洁）
2.重启设备，Eclipse，笔记本电脑，以上所有… …
3.根据失败将项目移动到不含空格的位置在设备“emulator-5554”上安装apk：timeout

该应用程序已经在该设备上多次调试（应用程序在市场上运行），但是这个问题每隔一段时间就会发生一次，而且非常麻烦……

任何帮助将不胜感激！感谢。

尝试更改ADB连接超时。我认为它默认为 5000ms ，我将其更改为 10000ms 以摆脱此问题。

如果你在Eclipse中，可以通过

来完成

窗口 - ＆gt;首选项 - ＆gt; Android - ＆gt; DDMS - ＆gt; ADB连接超时（ms）

我得到这个错误时通常会这样做：通过输入cmd来重新启动adb服务器：

adb kill-server
adb start-server
使用平台工具的一些从不版本，您可以在<代码>设备标签菜单中的 DDMS透视图捕获按钮），点击 Reset adb `。

EDIT2：**另外，我发现在PC的背面最好使用USB端口，因为大部分的前端USB端口< / code>是低功耗的，而且在你的设备上上传apks `的时候真的好慢。
