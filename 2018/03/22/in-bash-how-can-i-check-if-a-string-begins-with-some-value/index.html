在bash中，如何检查一个字符串是否以某个值开始？

By zhizi

Published 2018-03-22

Contents

我想检查一个字符串是否以“node”开头，例如“node001”。就像

if [ $HOST == user* ]  
  then  
  echo yes  
fi

我怎样才能正确地做到这一点？

我还需要结合表达式来检查HOST是“user1”还是以“node”开始

if [ [[ $HOST == user1 ]] -o [[ $HOST == node* ]] ];  
then  
echo yes 
fi

> > > -bash: [: too many arguments

如何正确执行？

高级Bash脚本指南上的这段代码说： p>

# The == comparison operator behaves differently within a double-brackets
# test than within single brackets.

[[ $a == z* ]]   # True if $a starts with a "z" (wildcard matching).
[[ $a == "z*" ]] # True if $a is equal to z* (literal matching).

所以你几乎是正确的;你需要 double 括号，而不是单个括号。

关于你的第二个问题，你可以这样写：

HOST=user1
if  [[ $HOST == user1 ]] || [[ $HOST == node* ]] ;
then
    echo yes1
fi

HOST=node001
if [[ $HOST == user1 ]] || [[ $HOST == node* ]] ;
then
    echo yes2
fi

哪个会回显

yes1
yes2

如果语法很难习惯（IMO），那么Bash的
如果你使用最近的bash（v3 +），建议bash正则表达式比较运算符=〜，即

if [[ "$HOST" =~ ^user.* ]]; then
    echo "yes"
fi

要在正则表达式中匹配这个或那个，使用|，即

if [[ "$HOST" =~ ^user.*|^host1 ]]; then
    echo "yes"
fi

注意 - 这是“正确”的正则表达式语法。

user *表示使用和零次或多次出现r，所以使用和userrrr将匹配。

user。*表示user，任何字符出现零次或多次，所以user1，userX将匹配。

  • ^ user。*表示匹配$ HOST开始处的模式user。*。
