PHPDoc类型提示对象的数组？

By zhizi

Published 2018-03-13

Contents

因此，在PHPDoc中，可以在成员变量声明之上指定@var来提示其类型。然后IDE，为前。
PHPEd，将知道它正在使用什么类型的对象，并能够提供一个代码洞察该变量。

<?php
  class Test
  {
    /** @var SomeObj */
    private $someObjInstance;
  }
?>

这很好，直到我需要对一组对象做同样的事情，以便稍后迭代这些对象时能够得到适当的提示。

那么，有没有办法声明一个PHPDoc标签来指定成员变量是一个SomeObj的数组？ @var数组是不够的，例如@var
array（SomeObj）`似乎不是有效的。

你可以做的最好的就是说，

foreach ($Objs as $Obj)
{
    /* @var $Obj Test */
    // You should be able to get hinting after the preceding line if you type $Obj->
}

我在Zend Studio中做了很多。不知道其他编辑，但它应该工作。

在JetBrains的PhpStorm IDE中，您可以使用/ ** @var SomeObj [] * /，例如：

/**
 * @return SomeObj[]
 */
function getSomeObjects() {...}

phpdoc文档推荐了这种方法：

specified containing a single type, the Type definition informs the reader
of the type of each array element. Only one Type is then expected as element
for a given array.

>

Example: @return int[]
