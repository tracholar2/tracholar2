正确使用“收益回报”

By zhizi

Published 2018-03-07

Contents

yield 关键字是C＃中的那些关键字继续让我神秘，我从来没有信心，我正确使用它。

以下两段代码，哪个是首选，为什么？

版本1： 使用收益率回报

public static IEnumerable<Product> GetAllProducts()
{
    using (AdventureWorksEntities db = new AdventureWorksEntities())
    {
        var products = from product in db.Product
                       select product;

        foreach (Product product in products)
        {
            yield return product;
        }
    }
}

第2版： 返回列表

public static IEnumerable<Product> GetAllProducts()
{
    using (AdventureWorksEntities db = new AdventureWorksEntities())
    {
        var products = from product in db.Product
                       select product;

        return products.ToList<Product>();
    }
}

当我计算列表中的下一个项目（甚至是下一组项目）时，我倾向于使用yield-return。

使用你的版本2，在返回之前你必须有完整的列表。 通过使用yield-return，你真的只需要返回之前有下一个项目。

除此之外，这有助于在更大的时间范围内传播复杂计算的计算成本。例如，如果列表连接到GUI，并且用户永远不会到最后一页，则永远不会计算列表中的最终项目。

如果IEnumerable表示无限集合，另一种情况是yield-
return更可取。考虑素数列表，或随机数的无限列表。你永远不能一次返回完整的IEnumerable，所以你使用yield-return来递增返回列表。

在您的特定示例中，您有完整的产品列表，因此我将使用版本2。

填充一个临时列表就像是下载整个视频，而使用yield则类似于流式传输视频。
