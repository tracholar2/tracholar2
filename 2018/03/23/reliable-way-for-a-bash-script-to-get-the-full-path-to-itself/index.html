一个bash脚本可靠的方式来获得自己的完整路径？

By zhizi

发表于 2018-03-23

文章目录

This question already has an answer here:

  • Getting the source directory of a Bash script from within 48 answers

我有一个bash脚本，需要知道它的完整路径。我试图找到一个广泛兼容的方式来做到这一点，而不是结束相对或时髦的路径。我只需要支持bash，而不是sh，csh等。

我到目前为止发现的：

  • 接受的答案是“获取一个Bash脚本的源代码目录，通过dirname $ 0获取脚本路径，这很好，但是可能会返回一个 相对路径 （如。），如果要更改脚本中的目录并使路径仍指向脚本目录，则会出现问题。尽管如此，dirname将成为难题的一部分。

  • “ Bash脚本绝对路径与OSX “ （特定于OS X，但答案不管） 给出了一个函数，用于测试$ 0是否相对，如果是的话将pre-pend $ PWD。但是结果仍然可能有相对的位（尽管总体上它是绝对的） - 例如，如果脚本是t在目录/ usr / bin中，在/ usr中输入bin /../bin/t来运行它（是的，这很麻烦），最后用/ usr / bin / ../ bin `作为脚本的目录路径。哪个 有效 ，但…

  • readlink解决方案[，看起来像这样：

    ＃这个脚本的绝对路径。 /home/user/bin/foo.sh

    SCRIPT = $（readlink -f $ 0）
    ＃脚本的绝对路径。/ home / user / bin
    SCRIPTPATH =dirname $ SCRIPT

但是readlink不是POSIX，显然这个解决方案依赖于GNU的readlink，因为某些原因BSD不能工作（我没有访问BSD像系统检查）。

所以，采取不同的方式，但都有警告。

什么是更好的方法？ “更好”是指：

  • 给我绝对路径。
  • 即使以错综复杂的方式调用，也能取得时髦的特征（参见上面＃2的评论）。 （例如，至少适度规范化路径）。
  • 仅依赖于bash-isms或者几乎可以确定是* nix系统（GNU / Linux，BSD和BSD-like系统，比如OS X等）的最流行版本的东西。
  • 如果可能的话，避免调用外部程序（例如，更喜欢内置的bash）。
  • （ 更新 ，感谢您的支持，至）不必解决符号链接（事实上，我宁愿留下他们一个人，但这不是要求）。

下面是我想到的（编辑：加上 sfstewman提供的一些调整，levigroker ，Kyle Strand ，和罗布·肯尼迪），这似乎大多符合我的“更好”的标准：

SCRIPTPATH="$( cd "$(dirname "$0")" ; pwd -P )"

为了正确处理空格和符号链接，SCRIPTPATH行显得特别迂回，但我们需要它而不是SCRIPTPATH =pwd。

还要注意的是，深奥的情况，例如执行一个脚本，而不是来自一个可访问的文件系统中的文件（这是完全可能的），不是迎合那里（或任何其他答案I见过）。

我很惊讶这里没有提到realpath命令。我的理解是，它被广泛地移植/移植。

你的初始解决方案变成：

SCRIPT=`realpath $0`    SCRIPTPATH=`dirname $SCRIPT`

如果没有符号链接，请根据您的偏好设置：

SCRIPT=`realpath -s $0`    SCRIPTPATH=`dirname $SCRIPT`
