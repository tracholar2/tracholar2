从流中创建一个字节数组

By zhizi

发表于 2018-03-26

文章目录

从输入流创建字节数组的首选方法是什么？

这是我现在使用.NET 3.5的解决方案。

Stream s;
byte[] b;

using (BinaryReader br = new BinaryReader(s))
{
    b = br.ReadBytes((int)s.Length);
}

读取和写入数据流块是否更好？

这取决于你是否可以信任s.Length。对于许多流，你只是不知道会有多少数据。在这样的情况下 - 和.NET 4之前 - 我会使用这样的代码：

public static byte[] ReadFully(Stream input)
{
    byte[] buffer = new byte[16*1024];
    using (MemoryStream ms = new MemoryStream())
    {
        int read;
        while ((read = input.Read(buffer, 0, buffer.Length)) > 0)
        {
            ms.Write(buffer, 0, read);
        }
        return ms.ToArray();
    }
}

使用.NET 4及以上版本，我会使用 <代码> Stream.CopyTo ` ，它基本上等于我的代码中的循环 - 创建MemoryStream，调用stream.CopyTo（ms）然后返回ms.ToArray（）。工作完成。

我也许应该解释为什么我的答案比其他答案长。 Stream.Read 不能保证它会读取所有要求的东西。例如，如果您正在从网络流中读取数据，则可能会读取一个数据包的值，然后返回，即使即将有更多的数据。
BinaryReader.Read 将继续下去，直到流的结束或您指定的大小，但您仍然必须知道大小开始。

上面的方法将继续读取（并复制到MemoryStream中）直到数据用完。然后它要求MemoryStream返回一个数组的数据副本。如果您知道大小开始 -
或者 认为 您知道大小，不用确定 - 您可以构造MemoryStream是开始的大小。同样，你可以在最后进行检查，如果流的长度与缓冲区的大小相同（由MemoryStream.GetBuffer ），那么你可以只返回缓冲区。所以上面的代码并没有被优化，但至少是正确的。它不承担关闭流的责任 -
调用者应该这样做。

/ p> Jon的答案是正确的，他正在重写CopyTo中已经存在的代码。所以对于.Net
4使用Sandip的解决方案，但对于以前版本的.Net使用乔恩的答案。在很多情况下，CopyTo中的异常使用“using”可以改善Sandip的代码，并且不会丢弃MemoryStream。

public static byte[] ReadFully(Stream input)
{
    using (MemoryStream ms = new MemoryStream())
    {
        input.CopyTo(ms);
        return ms.ToArray();
    }
}
