如何将 Android Studio添加库项目（如Sherlock ABS）？

（不是旧的ADT基于Eclipse的软件包，而是新的 Android Studio）。

Android Studio 1.0更新

自从Android Studio 1.0发布之后（v1.0和我之前的回答中的第一个版本之间的很多版本），有些事情已经发生了变化。

我的描述主要是通过Gradle文件手动添加外部库项目（为了更好地理解过程）。如果您想要通过Android
Studio创建者添加库，请使用视觉指南（下面的答案）检查[ Studio 1.0和那些截图，但过程非常相似）。

在您开始手动添加库到您的项目之前，请考虑添加外部依赖项。它不会混乱你的项目结构。几乎每个知名的Android库都可以在 Maven资料库中找到，其安装只需要一行app / build.gradle文件中的代码：

dependencies {
     compile 'com.jakewharton:butterknife:6.0.0'
}

添加库

以下是将外部Android库添加到我们项目的完整过程：

  • 通过Android Studio创建者创建一个新项目。我将它命名为 HelloWorld 。
  • 以下是由Android Studio创建的原始项目结构：

>

HelloWorld/

      app/

           - build.gradle  // local Gradle configuration (for app only)

           ...

      - build.gradle // Global Gradle configuration (for whole project)

      - settings.gradle

      - gradle.properties

      ...
  1. 在根目录（HelloWorld /）中，创建一个新的文件夹：/ libs，我们将在其中放置我们的外部库（这一步不是必需的 - 只有 - 保持一个更清洁的项目结构）
  2. 将你的库粘贴到新创建的/ libs文件夹中。在这个例子中，我使用了 PagerSlidingTabStrip库（只需从GitHub下载ZIP，将库目录重命名为“PagerSlidingTabStrip”复制它）。这是我们项目的新结构：

<预> <代码>的HelloWorld / 应用程序/ - build.gradle
//本地Gradle配置（仅适用于应用程序） … 库/ PagerSlidingTabStrip
/ - build.gradle //本地Gradle配置（仅用于库） - build.gradle
//全局Gradle配置（整个项目） - settings.gradle - gradle.properties
…

1. 通过将库添加到` include `来编辑settings.gradle。如果您像我一样使用自定义路径，则还需要为我们的库定义项目目录。整个settings.gradle应该如下所示：

include <’app’，’：PagerSlidingTabStrip’
project（’：PagerSlidingTabStrip’）。projectDir = new File（’libs /
PagerSlidingTabStrip’）

5.1如果你面对“默认配置”错误，那就试试这个，而不是第5步，

      include'：app'
包括'：libs：PagerSlidingTabStrip'

在app / build.gradle中添加我们的库项目作为依赖：

             依赖关系{
    编译fileTree（dir：'libs'，include：['* .jar']）
    编译“com.android.support:appcompat-v7:21.0.3”
    编译项目（“：PagerSlidingTabStrip”）
}

6.1。如果你按照步骤5.1，那么按照这个而不是6，

                 依赖关系{
            编译fileTree（dir：'libs'，include：['* .jar']）
            编译“com.android.support:appcompat-v7:21.0.3”

            编译项目（“：libs：PagerSlidingTabStrip”）
        }


1. 如果你的库项目没有` build.gradle `文件，你必须手动创建它。这是这个文件的例子：

                          apply plugin：'com.android.library'

            依赖{
                编译'com.android.support:support-v4:21.0.3'
            }

            android {
                compileSdkVersion 21
                buildToolsVersion“21.1.2”

                defaultConfig {
                    minSdkVersion 14
                    targetSdkVersion 21
                }

                sourceSets {
                    主{
                        manifest.srcFile'AndroidManifest.xml'
                        java.srcDirs = ['src']
                        res.srcDirs = ['res']
                    }
                }
            }

此外，您可以为您的项目创建一个全局配置，其中将包含SDK版本和为每个模块构建工具版本以保持一致性。只需编辑gradle.properties文件并添加行：

<预> <代码> ANDROID_BUILD_MIN_SDK_VERSION = 14 ANDROID_BUILD_TARGET_SDK_VERSION =
21 ANDROID_BUILD_TOOLS_VERSION = 21.1.3 ANDROID_BUILD_SDK_VERSION = 21

现在，您可以在build.gradle文件（在应用程序和库模块）中使用它，如下所示：

<预> <代码> // … android { compileSdkVersion
Integer.parseInt（project.ANDROID_BUILD_SDK_VERSION）
buildToolsVersion项目.ANDROID_BUILD_TOOLS_VERSION defaultConfig {
minSdkVersion Integer.parseInt（project.ANDROID_BUILD_MIN_SDK_VERSION）
targetSdkVersion Integer.parseInt（project.ANDROID_BUILD_TARGET_SDK_VERSION）
} } // …

那是所有的。只需点击“使用Gradle”图标同步项目。您的图书馆应该在您的项目中可用。
