无法更新EntitySet - 因为它有一个DefiningQuery并且不存在UpdateFunction元素

By zhizi

Published 2018-07-25

Contents

我正在使用实体框架1与.net 3.5。

我正在做一些简单的事情：

var RoomDetails = context.Rooms.ToList();

foreach (var Room in Rooms)
{        
   Room.LastUpdated = DateTime.Now;
}

我试着去做这个错误：

context.SaveChanges();

我得到错误：

**Unable to update the EntitySet - because it has a DefiningQuery and no

element exists in the element
to support the current operation.**

我在上下文上做了很多更新，没有任何问题，只是当我试图更新这个特定的实体时。

我所有的搜索都显示了同样的情况，在我试图更新的实体上没有声明主键。但是，唉，我确实有一个主键声明…

这通常是因为以下原因之一：

  • 实体集从数据库视图映射
  • 自定义数据库查询
  • 数据库表没有主键

这样做之后，您可能仍然需要在实体框架设计器中进行更新（或者删除实体，然后添加它），然后再停止出现错误。

只需将一个主键添加到表中。而已。问题解决了。

ALTER TABLE <TABLE_NAME>
ADD CONSTRAINT <CONSTRAINT_NAME> PRIMARY KEY(<COLUMN_NAME>)
