转义sed替换模式的字符串

By zhizi

Published 2018-07-02

Contents

在我的bash脚本中，我有一个外部（从用户接收）字符串，我应该在sed模式中使用。

REPLACE="<funny characters here>"
sed "s/KEYWORD/$REPLACE/g"

如何将$ REPLACE字符串转义出来，以便被sed安全地接受为字面替换？

注意：** KEYWORD是一个哑巴的子字符串，没有匹配等。它不是由用户提供的。

警告 ： not 不考虑换行符。有关更深入的答案，请参见这个SO-
问题可靠地转义正则表达式元字符。 （谢谢，Ed Morton＆amp;
Niklas Peter）

注意，逃避一切是一个坏主意。 sed需要很多字符才能被转义到 _其 __ 的特殊含义。例如，如果您在替换字符串中转义一个数字，它将转入反向引用。

正如Ben Blank所说，在替换字符串中只有三个字符需要被转义（转义本身，用于语句结束的正斜杠和用于替换的字符）：

sed -e 's/[\/&]/\\&/g'

如果您需要转义KEYWORD字符串，以下是您需要的：

sed -e 's/[]\/$*.^[]/\\&/g'

请记住，如果您使用除/之外的其他字符作为分隔符，则需要用上述表达式中的斜杠来替换正在使用的字符。请参阅PeterJCLaw的解释评论。

编辑： 由于以前没有考虑到某些角落案例，上述命令已经多次更改。查看编辑历史了解详细信息。

sed命令允许您使用其他字符而不是/作为分隔符：

sed 's#"http://www\.fubar\.com"#URL_FUBAR#g'

双引号不是问题。
