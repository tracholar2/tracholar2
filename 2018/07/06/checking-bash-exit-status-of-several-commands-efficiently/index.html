有效地检查几个命令的Bash退出状态

By zhizi

发表于 2018-07-06

文章目录

有没有类似pipefail多个命令，像一个’尝试’的声明，但在bash内。我想要做这样的事情：

echo "trying stuff"
try {
    command1
    command2
    command3
}

任何时候，如果有任何命令失败，则退出并回显该命令的错误。我不想做这样的事情：

command1
if [ $? -ne 0 ]; then
    echo "command1 borked it"
fi

command2
if [ $? -ne 0 ]; then
    echo "command2 borked it"
fi

等等…或类似的东西：

pipefail -o
command1 "arg1" "arg2" | command2 "arg1" "arg2" | command3

因为每个命令的论点我相信（纠正我，如果我错了）会互相干扰。这两种方法对我来说似乎非常啰嗦和讨厌，所以我在这里呼吁一个更有效的方法。

你可以编写一个函数来启动并测试你的命令。假设command1和command2是已设置为命令的环境变量。

function mytest {
    "$@"
    local status=$?
    if [ $status -ne 0 ]; then
        echo "error with $1" >&2
    fi
    return $status
}

mytest $command1
mytest $command2

你是什么意思“退出并回应错误”？如果你的意思是你希望脚本在任何命令失败时立即终止，那么就执行

set -e

在脚本的开始。不要打扰回应错误消息：让失败的命令处理。换句话说，如果你这样做：

#!/bin/sh

set -e
command1
command2
command3

并且command2失败，同时向stderr打印一条错误信息，那么看起来你已经达到了你想要的。 （除非我误解你想要的东西！）

作为一个推论，你写的任何命令都必须表现得很好：它必须向stderr报告错误而不是stdout（问题中的示例代码向stdout输出错误），并且在失败时必须以非零状态退出。
