无法执行dex：多个dex文件定义Lcom / myapp / R $数组;

By zhizi

Published 2018-07-14

Contents

自从更新到ADT 14我不能再构建我的项目了。更新之前，它正在建设好。

错误：

[2011-10-23 16:23:29 - Dex Loader] Unable to execute dex: Multiple dex files define Lcom/myapp/R$array;
[2011-10-23 16:23:29 - myProj] Conversion to Dalvik format failed: Unable to execute dex: Multiple dex files define Lcom/myapp/R$array;

有类似的问题已经被报道，我已经尝试了其中的建议，包括

  • 重新启动Eclipse。
  • 清理项目并重建 - 禁用“Project-＆gt;自动构建”选项，然后“清理”和“构建”项目，然后尝试运行。重置“自动生成”选项打开
  • 重新安装Android开发者工具
  • 重新安装Eclipse（已更新至最新版本3.7.1）
  • 创建了一个从文件系统导入的新项目
  • 从subversion创建了一个新项目。

我有同样的问题，很奇怪，因为它只在使用Eclipse时发生（但对于Ant来说是可以的）。 这是我解决它的方法：

  • 右键点击Project Name * 选择Build Path - ＆gt; 配置构建路径在Java Build Path中，转到Order and Export
  • 取消选中.jar库

有时只是： 在顺序和导出选项卡，我没有任何jar库，所以我有未经检查的Android私人库项目。现在我的项目正在运行。

清理了以前版本的工具中剩下的一些目录和文件后，我的问题得到了解决。 ADT第14版更改存储二进制文件的位置。我删除了整个bin目录，重新启动了Eclipse并清除了构建并强制重建。这似乎是最初的伎俩，但问题在下一次运行后又回来了。

我终于发现我的bin目录包含在项目构建路径中。我从构建路径中排除bin，并重复上述步骤。这解决了我的问题。
