为什么是可变的结构？邪恶？

By zhizi

Published 2018-07-21

Contents

在这里讨论之后，我已经多次阅读了可变结构是“恶魔”的评论（就像这个问题）。

C＃中的可变性和结构的实际问题是什么？

结构是值类型，意味着它们在被传递时被复制。

因此，如果您更改副本，则仅更改该副本，而不是更改原始副本，而不是任何其他副本。

如果你的结构是不可变的，那么通过值传递的所有自动拷贝将是相同的。

如果你想改变它，你必须有意识地通过修改数据创建一个新的结构实例。 （不是副本）

从哪里开始; -p

Eric Lippert的博客是总是很好的报价：

This is yet another reason why mutable value types are evil. Try to always
make value types immutable.

首先，你很容易失去改变…例如，从列表中取出东西：

Foo foo = list[0];
foo.Name = "abc";

那是什么改变的？没有用…

与属性相同：

myObj.SomeProperty.Size = 22; // the compiler spots this one

强迫你做：

Bar bar = myObj.SomeProperty;
bar.Size = 22;
myObj.SomeProperty = bar;

少批评，有一个规模问题;可变对象 倾向 具有多个属性;但是如果你有一个带有两个int，string，DateTime和一个bool的结构，你可以很快就烧穿了很多的记忆。使用一个类，多个调用者可以共享对同一个实例的引用（引用很小）。
