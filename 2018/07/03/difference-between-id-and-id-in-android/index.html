Android中“@ id /”和“@ + id /”的区别

By zhizi

Published 2018-07-03

Contents

What is the diffirence between the @id/ and @+id/?

在 @ + id / 中，加号指示创建一个新的资源名称并将其添加到R.java文件中那么 @id / 呢？从ID ：引用Android资源ID时，不需要加号，但必须添加android包名称空间，如下所示：

android:id="@android:id/list"

但是在Eclipse下面的图片中并没有提供任何类型的@android：id /。

Are @id/ and @android:id/ the same?

您可以在@android：id /..中引用已经在Android系统中定义的Android资源，同时访问您已经定义/创建的资源你的项目，你使用@id /..

根据你在聊天中的说明，你说你有这样的问题：

If we use android:id="@id/layout_item_id" it doesn’t work. Instead @+id/works so what’s the difference here? And that was my original question.

那么，这取决于上下文，当你使用android：id的XML属性时，你指定了一个新的id，并指示解析器生成器）在 R.java
中创建一个新条目，因此您必须包含一个 + `符号。

在另一种情况下，像android：layout_below =“@ id / myTextView”，你指的是一个已经创建的id，所以解析器把它链接到已经创建的id在R.java。

中 再次**

正如您在聊天中所说的，如果在后面写入android：layout_below =“@ id / myTextView”，将无法识别id为myTextView的元素你正在使用它的元素。

+符号是将id添加到资源ID列表的捷径。否则，你需要把它们放在一个像这样的xml文件中

<?xml version="1.0" encoding="utf-8"?>
<resources>
    <item name="my_logo" type="id"/>
</resources>
