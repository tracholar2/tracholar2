-bash： /my_script：/ bin / bash ^ M：错误的解释器：没有这样的文件或目录

By zhizi

Published 2018-07-13

Contents

我正在通过本教程学习bash脚本，以便为我自动完成一些任务。我使用putty连接到服务器。

位于... / Documents / LOG中的脚本是：

#!/bin/bash
# My first script
echo "Hello World!"

我执行以下的读/写/执行权限

chmod 755 my_script

然后，当我输入./ my_script时，我得到了标题中给出的错误。

一些类似的问题想要看到这些，所以我认为他们可能会帮助：

which bash

/bin/bash

和

echo $PATH

>
/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/games:/usr/bin/mh

我尝试将当前目录添加到PATH，但这不起作用。

在Windows环境中创建脚本，然后移植到Unix环境中，我已经看到了这个问题。

尝试在脚本上运行：

http://linuxcommand.org/man_pages/dos2unix1.html

更新的链接：

http://dos2unix.sourceforge.net/

或者只是使用vi在Unix环境中重写脚本并测试。

Unix使用不同的换行符和回车符，所以不能读取你在Windows上创建的文件。因此它将^ M视为非法字符。

如果你想在Windows上编写一个文件然后移植，确保你的编辑器设置为以UNIX格式创建文件。

在屏幕右下角的记事本++中，它会告诉你文档格式。默认情况下，它会显示Dos \ Windows。要更改它，请转到

  • 设置 - >首选项
  • 新建文件/默认目录标签
  • 选择格式为unix并关闭
  • 创建一个新文档

在终端

运行以下命令

sed -i -e 's/\r$//' scriptname.sh

然后尝试

./scriptname.sh

它应该工作。
