伙计，我的php ini在哪里？

By zhizi

发表于 2018-01-05

文章目录

几年前，我在我维护的Linux服务器上安装了Apache 2.2x和PHP 5.3.1。我使用了.tar.gz，并按照指示构建它们
（而不是rpms和what-have-you）。一切都很好。

今天我需要安装这看起来像一个PHP库。我经历了所有的步骤来安装，我发现ibm_db2.so在$PHP_HOME/lib/extensions/somecomplicatedname/ibm_db2.so

最大的困难是最后一步是配置php.ini，但是我的系统中没有php.ini。恐怖的恐怖。 PHP的工作正常，当然除了这个新的ibm_db2
thingamagic，我想要使用，所以有人可以使用GUI来修补DB2。 （我尝试了一个小php脚本失败，并表示ibm_db2函数不可用）。

我必须每隔几年处理一次PHP，所以请在最基本的层面告诉我可以做些什么来启用基于Web的GUI访问DB2。

找到这个最好的方法是： 创建一个PHP文件并添加以下代码：

<?php phpinfo(); ?>

并在浏览器中打开，会显示正在读取的文件！

OP的更新：

以前接受的答案可能会更快，更方便，但并不总是正确的。请参阅有关该答案的评论。 还请注意在这个答案中提到的更方便的替代<?php echo php_ini_loaded_file(); ?>。

在命令行上执行：

php --ini

你会得到像这样的东西：

Configuration File (php.ini) Path: /etc/php5/cli
Loaded Configuration File:         /etc/php5/cli/php.ini
Scan for additional .ini files in: /etc/php5/cli/conf.d
Additional .ini files parsed:      /etc/php5/cli/conf.d/curl.ini,
/etc/php5/cli/conf.d/pdo.ini,
/etc/php5/cli/conf.d/pdo_sqlite.ini,
/etc/php5/cli/conf.d/sqlite.ini,
/etc/php5/cli/conf.d/sqlite3.ini,
/etc/php5/cli/conf.d/xdebug.ini,
/etc/php5/cli/conf.d/xsl.ini

这是从我的本地开发机器。但是，第二行是有趣的一个。如果没有提到什么，看看第一个。这是PHP查找php.ini的路径。

您可以在脚本中使用phpinfo() grep相同的信息，并用浏览器调用它。它在输出的第一部分提到。 php -i对于命令行是一样的，但是相当不舒服。
