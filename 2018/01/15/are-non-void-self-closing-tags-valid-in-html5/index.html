（非无效）自闭标签在HTML5中有效吗？

By zhizi

Published 2018-01-15

Contents

W3C验证器不喜欢自闭标签（那些以“代码>
/＆gt; `“）在非-void 元素。 （空元素是那些可能永远不会包含任何内容的元素。）它们在HTML5中仍然有效吗？

一些 接受 void元素的例子：

<br />
<img src="" />
<input type="text" name="username" />

一些 的例子拒绝了 非空元素：

<div id="myDiv" />
<span id="mySpan" />
<textarea id="someTextMessage" />

`）。但是，一般而言，自闭标签在HTML5中并不是100％有效，而且这些答案详细解释了各种HTML风格的自闭标签问题。_

在 HTML 4 中，＆lt; foo /（是，没有＆gt;）意味着＆lt; ;＆lt; br＆gt; （即通往＆lt; br
/＆gt; 的含义＆lt; br＆gt;＆gt; （即＆lt; br＆gt;＆amp; gt ; ）和＆lt; title / hello /
表示＆lt; title＆gt; hello＆lt; / title＆gt;
`）。浏览器在支持这方面做得很差，规范建议作者以避免语法。

  • 在 XHTML 中， ＆lt; foo /＆gt;表示＆lt; foo＆gt;＆lt; / foo＆gt; 。这是适用于所有XML文档的 XML 规则。也就是说，XHT​​ML经常被用作text / html，这些（至少在历史上）被浏览器使用不同的解析器处理，而不是用作application / xhtml + xml的文档。 W3C提供兼容性指导，以遵循XHTML为` text / html的。 （本质上：只有当元素被定义为EMPTY时，才使用自闭标记语法（并且HTML标记中的结束标记是被禁止的））。

HTML5 ，＆lt; foo /＆gt; 取决于元素的类型。

* 在被指定为 _void元素_ 的HTML元素（本质上是“HTML 5之前存在且禁止任何内容的元素”）时，简单地禁止结束标记。开始标签末尾的斜线是允许的，但没有意义。对于那些沉迷于XML的人来说，这只是语法糖（和语法荧光笔）。 
* 在其他HTML元素上，斜杠是错误的，但错误恢复将导致浏览器忽略它，并将标记视为常规的开始标记。这通常会以一个缺少的结束标记结束，导致后续元素成为孩子而不是兄弟姐妹。
* 外部元素（从SVG等XML应用程序导入）将其视为自闭合语法。

正如Nikita Skvortsov所指出的那样，一个自我关闭的div将不会被验证。这是因为div是 普通元素 ，而不是 空元素 。根据HTML5规范，不能有任何内容的标签（称为
空元素 ）可以自闭。这包括以下标签：

area, base, br, col, embed, hr, img, input, 
keygen, link, meta, param, source, track, wbr

在上面的标签中，“/”是完全可选的，所以＆lt; img /＆gt;与＆lt; img＆gt;没有区别，但是＆lt; img＆gt;＆lt; / img＆gt;无效。

注意： *外国元素 也可以自动关闭，但是我不认为这是在这个答案的范围内。
