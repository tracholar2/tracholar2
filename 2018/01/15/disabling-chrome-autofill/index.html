禁用Chrome自动填充

By zhizi

Published 2018-01-15

Contents

我一直在用几种形式的chrome自动填充行为遇到问题。

表单中的字段都有非常普遍和准确的名称，例如“email”，“name”或“password”，还有autocomplete =“off”集。

自动填充标志已成功禁用了自动填充行为。在您开始输入时，下拉的值会出现，但并未更改Chrome自动填充字段的值。

除了chrome填写输入错误，例如用电子邮件地址填写电话输入外，这种行为可以。客户对此抱怨，所以经过验证，在多种情况下都会发生，而不是像我在本地机器上完成的某些结果那样。

我能想到的唯一的当前解决方案是动态生成自定义输入名称，然后提取后端的值，但这似乎是一个相当hacky解决这个问题。是否有任何标签或怪异改变了自动填充行为，可以用来解决这个问题？

对于新的Chrome版本，您只需在密码字段中输入autocomplete =“new-password”就可以了。我检查过了，工作正常。

在本次讨论中得到了Chrome开发者的提示： https://bugs.chromium.org/p/chromium/issues /细节？ID =

P.S。不要忘记使用不同字段的唯一名称来防止自动填充。

我刚刚发现，如果您有一个记住用户名和密码的网站，Chrome的当前版本将自动填充您的用户名/电子邮件地址 之前 任何type =密码字段。它并不关心这个字段被称为什么 - 只是在密码将成为你的用户名之前假设这个字段。

旧解决方案

只要使用＆lt; form autocomplete =“off”＆gt;，它就可以防止预先填写密码，以及基于浏览器可能做出的假设的任何启发式填充字段）。与使用＆lt; input autocomplete =“off”＆gt;相反，它似乎被密码自动填充忽略了（在Chrome中，Firefox确实服从它）。

更新的解决方案

Chrome现在忽略＆lt; form autocomplete =“off”＆gt;。因此，我原来的解决方法（我删除了）现在是所有的愤怒。

只需创建几个字段，并用“display：none”隐藏它们。例如：

<!-- fake fields are a workaround for chrome autofill getting the wrong fields -->
<input style="display:none" type="text" name="fakeusernameremembered"/>
<input style="display:none" type="password" name="fakepasswordremembered"/>

然后把你的真实领域放在下面。

记住要添加评论或其他人在你的团队会想知道你在做什么！

如果您使用javascript来修改表单，那么您将需要额外的技巧。在操作表单时显示假的字段，然后在一毫秒后再隐藏它们。

使用jQuery的示例代码（假设你给你的假字段一个类）：

$(".fake-autofill-fields").show();
// some DOM manipulation/ajax here
window.setTimeout(function () {
    $(".fake-autofill-fields").hide();
},1);
