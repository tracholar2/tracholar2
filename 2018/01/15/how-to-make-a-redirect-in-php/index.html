是否可以通过使用PHP将用户重定向到其他页面？

假设用户转到www.example.com/page.php，我想将它们重定向到www.example.com/index.php我这样做没有使用元刷新？可能的？

这甚至可以保护我的网页免受未经授权的用户的侵害。

现有答案的摘要加上我自己的两分钱：

1. Basic answer

您可以使用header（）函数发送一个新的HTTP标头，但是这必须在任何HTML或文本之前发送到浏览器（所以在＆lt;！DOCTYPE ...＆gt;声明，例如）。

header('Location: '.$newURL);

2. Important details

die（）或 exit（）**

header("Location: http://example.com/myOtherPage.php");
die();

为什么你应该使用die（）或exit（）：每日跆拳道

绝对网址

网址必须是绝对的。请参阅 RFC 2616。但是在大多数情况下，相对URL也会被接受。

状态码

PHP的“位置”标题仍然使用 HTTP 302重定向代码，但这是不是你应该使用的那个。您应该考虑 301（永久重定向）或 303 （other）。

注意：W3C提及，头部与许多HTTP
/ 1.1用户代理不兼容，目前使用的浏览器都是HTTP / 1.1用户代理，对于许多其他用户代理（如蜘蛛和机器人）来说，并不是这样。

3. Documentation

HTTP头文件和PHP中的header（）函数

  • PHP说明书
  • 维基百科所说的
  • W3C所说的

4. Alternatives

您可以使用http_redirect（$ url）的替代方法;需要安装 PECL包pecl。

5. Helper Functions

此功能不包含303状态码：

function Redirect($url, $permanent = false)
{
    header('Location: ' . $url, true, $permanent ? 301 : 302);

    exit();
}

Redirect('http://example.com/', false);

这更灵活：

function redirect($url, $statusCode = 303)
{
   header('Location: ' . $url, true, $statusCode);
   die();
}

6. Workaround

正如前面提到的标题（）重定向只有在写出任何东西之前。如果输入HTML中的内容，他们通常会失败。那么你可以使用一个HTML头解决方法（不是很专业！），如：

<meta http-equiv="Location" content="http://example.com/">

甚至是Javascript重定向。

window.location.replace("http://example.com/");



function Redirect($url, $permanent = false)
{
    if (headers_sent() === false)
    {
        header('Location: ' . $url, true, ($permanent === true) ? 301 : 302);
    }

    exit();
}

Redirect('http://www.google.com/', false);

不要忘了死（）/退出（）！
