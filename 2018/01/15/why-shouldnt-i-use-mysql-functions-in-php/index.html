查看红盒？

Why

离开ext / mysql不仅关乎安全性，还关于访问MySQL数据库的所有功能。

ext / mysql 是为 **MySQL 3.23** 构建的，从那以后只增加了很少的内容，而且保持了与旧版本的兼容性，这使得代码难以维护。缺少ext / mysql `不支持的功能包括：（ _ from

  • 压缩
  • 完整的字符集支持

不使用mysql _ *函数的原因 ：

  • 不在积极开发中
  • 自PHP 7起删除
  • 缺少OO界面
  • 不支持非阻止异步查询

使用mysql _ *函数或者我们可以用旧的方式（在PHP 5.5及以上版本中弃用）

使用PDO：您只需创建一个新的PDO对象。构造函数接受用于指定数据库源的参数。PDO的构造函数大多接受四个参数：DSN（数据源名称）和可选的username <代码>密码

Error Handling

错误处理在PDO中比mysql _ *容易得多。

使用mysql _ *时的常见做法是：

//Connected to MySQL
$result = mysql_query("SELECT * FROM table", $link) or die(mysql_error($link));

由于我们无法处理die中的内容，所以OR（）不是处理错误的好方法。它会突然结束脚本，然后将错误回显到通常不希望显示给最终用户的屏幕上，让血腥的黑客发现您的架构。或者，mysql _ * `函数的返回值通常可以与 mysql_error（）来处理错误。

PDO 提供了一个更好的解决方案：异常。我们对 PDO 所做的任何事情都应该包装在一个 try \- catch
块中。通过设置错误模式属性，我们可以强制 PDO `进入三种错误模式之一。下面是三种错误处理模式。

  • <代码> PDO :: ERRMODESILENT 。它只是设置错误代码，与` mysql * 几乎相同，您必须检查每个结果，然后查看 $ db-＆gt; errorInfo（）; `以获取错误细节。
  • PDO :: ERRMODE_WARNING引发E_WARNING。 （运行时警告（非致命错误）脚本的执行不会停止。）
  • PDO :: ERRMODE_EXCEPTION：抛出异常。它代表了PDO提出的一个错误。你不应该从你自己的代码中抛出一个PDOException。有关PHP中的异常的更多信息，请参阅 Exceptions 。它的行为非常像或die（mysql_error（））;，当它没有被捕获。但是，与或die（）不同，如果您选择这样做，则可以正确地捕获并处理PDOException。

Selecting Data

那么你在mysql _ *中所做的是：

<?php
$result = mysql_query('SELECT * from table') or die(mysql_error());

$num_rows = mysql_num_rows($result);

    echo $row['field1'];
}

现在在PDO中，可以这样做：

<?php
$stmt = $db->query('SELECT * FROM table');

while($row = $stmt->fetch(PDO::FETCH_ASSOC)) {
    echo $row['field1'];
}

或者

<?php
$stmt = $db->query('SELECT * FROM table');
$results = $stmt->fetchAll(PDO::FETCH_ASSOC);

//Use $results

注意：如果你使用下面的方法（query（）），这个方法返回一个PDOStatement对象。所以如果你想获取结果，像上面一样使用它。

<?php
foreach($db->query('SELECT * FROM table') as $row) {
    echo $row['field1'];
}

在PDO数据中，它是通过你的语句句柄的方法- > fetch（）获得的。在调用fetch之前，最好的方法是告诉PDO你如何获取数据。在下面的部分，我正在解释这一点。

Fetch Modes

注意上面的fetch（）和fetchAll（）代码中PDO :: FETCH_ASSOC的使用。这告诉PDO以字段名作为关键字将行作为关联数组返回。还有很多其他的获取模式，我会一一解释。

首先，我解释如何选择获取模式：

$stmt->fetch(PDO::FETCH_ASSOC)

在上面，我一直在使用fetch（）。您也可以使用：

  • PDostatement :: fetchAll（） - 返回包含所有结果集行的数组
  • PDOStatement :: fetchColumn（） - 返回结果集的下一行中的单个列
  • PDOStatement :: fetchObject（） - 获取下一行并将其作为对象返回。
  • PDOStatement :: setFetchMode（） - 为此语句设置默认提取模式

现在我来抓取模式：

  • PDO :: FETCH_ASSOC：返回按列名索引的数组，返回结果集
    PDO :: FETCH_BOTH（默认值）：返回一个由列名和0索引列号索引的数组，返回结果集

还有更多的选择！请阅读 PDOStatement获取文档。

Prepared Statements

Q。 什么是准备好的陈述，为什么我需要他们？ A。 准备好的语句是一个预编译的SQL语句，只需将数据发送到服务器即可多次执行。

使用准备好的语句的典型工作流程如下（引自维基百科三点三分
）：

准备**：语句模板由应用程序创建并发送到数据库管理系统（DBMS）。某些值未指定，称为参数，占位符或绑定变量（在下面标记为？）：

插入产品（name，price）VALUES（？，？）
DBMS解析，编译并在语句模板上执行查询优化，并存储结果而不执行。

执行
：稍后，应用程序为参数提供（或绑定）值，DBMS执行语句（可能返回结果）。应用程序可以根据需要以不同的值执行多次该语句。在这个例子中，它可能为第一个参数提供“Bread”，为第二个参数提供1.00。

您可以通过在SQL中包含占位符来使用准备好的语句。基本上有三个没有占位符（不要试图用上面的变量），一个没有名字的占位符，一个有命名的占位符。

现在，什么是指定的占位符，我如何使用它们？

Q A。 命名的占位符。使用以冒号开头的描述性名称，而不是问号。我们不关心在名称持有者中的价值的位置/顺序：

$stmt->bindParam(':bla', $bla);

bindParam（parameter，variable，data_type，length，driver_options）

你也可以使用一个执行数组来绑定：

<?php

$stmt->execute(array(':name' => $name, ':id' => $id));
$rows = $stmt->fetchAll(PDO::FETCH_ASSOC);

对于OOP朋友来说，另一个很好的功能是，假定属性与命名字段匹配，指定的占位符就可以直接将对象插入到数据库中。例如：
