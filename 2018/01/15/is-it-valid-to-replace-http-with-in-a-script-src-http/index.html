用script src =“http：// ”替换http：//是否有效？

By zhizi

Published 2018-01-15

Contents

我有以下元素：

<script type="text/javascript" src="https://cdn.example.com/js_file.js"></script>

在这种情况下，该网站是HTTPS，但该网站也可能只是HTTP。 （JS文件在另一个域中。）我想知道是否有效，为方便起见，请执行以下操作：

<script type="text/javascript" src="//cdn.example.com/js_file.js"></script>

我想知道是否可以移除http：或https：？

这似乎在我测试过的任何地方都能正常工作，但是在任何情况下，它都不起作用吗？

没有方案的相对网址（http：或https :)是有效的，根据 RFC 3986：
“统一资源标识符（URI）：通用语法”，第4.2节。如果一个客户端扼杀它，那么这是客户端的错，因为它们不符合在RFC中指定的URI语法。

你的例子是有效的，应该工作。我已经在流量很大的网站上使用了这种相对的URL方法，并没有任何投诉。此外，我们在Firefox，Safari，IE6，IE7和Opera测试我们的网站。这些浏览器都了解URL格式。

我们保证可以在任何主流浏览器上工作（我不考虑浏览器的市场份额低于0.05％）。哎呀，它在Internet Explorer 3.0中工作。

RFC 3986 将URI定义为由以下部分组成： p>

  foo://example.com:8042/over/there?name=ferret#nose
  \_/   \______________/\_________/ \_________/ \__/
   |           |            |            |        |
scheme     authority       path        query   fragment

定义相对URI（第5.2节）时，可以省略任何那些部分，总是从左边开始。在伪代码中，它看起来像这样：

result = ""

 if defined(scheme) then
    append scheme to result;
    append ":" to result;
 endif;

 if defined(authority) then
    append "//" to result;
    append authority to result;
 endif;

 append path to result;

 if defined(query) then
    append "?" to result;
    append query to result;
 endif;

 if defined(fragment) then
    append "#" to result;
    append fragment to result;
 endif;

 return result;

您所描述的URI是一个无方案的相对URI。
