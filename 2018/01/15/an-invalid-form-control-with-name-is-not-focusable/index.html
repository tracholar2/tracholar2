名称=“”的无效表单控件不可聚焦

By zhizi

发表于 2018-01-15

文章目录
  1. 1. UPDATE: August 21, 2015

我的网站上有一个严重的问题。 在谷歌浏览器中，有些客户无法进入我的付款页面。 当试图提交表单时，我得到这个错误：

An invalid form control with name='' is not focusable.

这是来自JavaScript控制台。

我知道这个问题可能是由于隐藏字段具有必需的属性。 现在问题是我们正在使用.net webforms所需的字段验证器，而不是HTML5必需的属性。

这个错误似乎是随机的。 有没有人知道这个解决方案？

这是不够的状态

Adding a novalidate attribute to the form will help

这并不能解释问题，OP也许不明白为什么会有帮助，或者真的有帮助。

这里是一个真正解释案例的答案，理解这个问题应该使你得到一个解决方案适合你的发展：

Chrome wants to focus on a control that is required but still empty so that
it can pop up the message ‘Please fill out this field’. However, if the
control is hidden at the point that Chrome wants to pop up the message, that
is at the time of form submission, Chrome can’t focus on the control because
it is hidden, therefore the form won’t submit.

>

So, to get around the problem, when a control is hidden by javascript, we
also must remove the ‘required’ attribute from that control.

为了使警告变得有用，请考虑这一点 - 验证失败时不要隐藏字段 。但这不是一个严格的规定，也不是一个规则。正如我在下面的评论中提到的那样，我将其解释为

In some cases, the issue is not a problem at all and does not cause loss of
functionality in an application. Then the error maybe ignored.

UPDATE: August 21, 2015

有关的错误也可能由于过早的验证而产生。如果您有＆lt;按钮＆gt;输入，而没有设置 类型
属性，则可能会发生过早验证。如果没有将按钮的type属性设置为按钮，则Chrome（或任何其他浏览器）会在每次单击该按钮时执行验证，因为
submit 是默认值按钮类型。为了解决这个问题，如果您的网页上有一个按钮，而不是 提交 或 重置
，一定要记得这样做：＆lt; button type =“button”＆gt;
在表单中添加novalidate属性有助于：

<form name="myform" novalidate>
