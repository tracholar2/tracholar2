我有一些格式不正确的HTML代码，我想重新格式化。是否有命令会自动重新格式化Sublime Text 2中的HTML代码，使其看起来更好，更易于阅读？

你不需要任何插件来做到这一点。 只需选择所有行（Ctrl A），然后从菜单中选择Edit→Line→Reindent。
如果您的文件使用包含HTML（如.html或.php）的扩展名保存，

如果你经常这样做，你可能会发现这个关键映射很有用：

{ "keys": ["ctrl+shift+r"], "command": "reindent" , "args": { "single_line": false } }

如果您的文件没有保存（例如，您只是将代码段粘贴到新窗口中），则可以通过选择菜单View→Syntax→language of choice来手动设置缩进语言选择重新选择。

在Sublime中有六种格式的HTML格式。我已经测试了各种最受欢迎​​的插件（请参阅我在博客上做的写作）细节），但是下面是一些最受欢迎的选项的快速概览：

Reindent command

优点：

  • 使用Sublime发货，所以不需要安装插件

缺点：

  • 不删除多余的空白行
  • 无法处理缩小的HTML，包含多个开放标记的行
  • 格式不正确＆lt; script＆gt;块

优点：

  • 支援ST2 / ST3
  • 删除多余的空白行
  • 没有二进制相关性

缺点：

  • PHP标签上的扼流圈
  • 不正确处理＆lt; script＆gt;块

优点：

  • 处理PHP标记
  • 某些设置可以调整格式

缺点：

  • 需要PHP（退回到Web服务）
  • 仅限ST2
  • 废弃？

优点：

  • 支援ST2 / ST3
  • 简单且没有binaray依赖关系
  • 支援OS X，Win和Linux

缺点：

  • 使用内嵌评论扼流
  • 展开最小化/压缩代码

优点：

  • 支援ST2 / ST3
  • 处理HTML，CSS，JS
  • 与Sublime的菜单很好地结合
  • 高度自定义
  • 每个项目设置
  • 保存选项的格式

缺点：

  • 需要Node.js
  • 不适合嵌入式PHP

Which is best?
