对于SVG文件，我使用img，object还是embed？

By zhizi

Published 2018-01-15

Contents

我应该使用＆lt; img＆gt;，＆lt; object＆gt;或者＆lt; embed＆gt;来加载SVG文件到页面类似于加载jpg，gif或png？

每个代码是什么，以确保它的工作尽可能好？ （我看到引用了包含mimetype或者指向我的研究中的备用SVG渲染器，并没有看到一个好的状态的艺术参考）。

假设我正在使用 Modernizr 检查SVG支持，然后回退（可能会用简单的＆lt; img＆gt; ;标签），用于不支持SVG的浏览器。

我可以推荐SVG入门（由W3C出版），它涵盖了这个主题：http://www.w3.org/Graphics/SVG/IG/resources/svgprimer.html#SVG_in_HTML

如果您使用＆lt; object＆gt;，则可以免费获得栅格后备*：

<object data="your.svg" type="image/svg+xml">
  <img src="yourfallback.jpg" />
</object>

*）呃，不是完全免费的，因为有些浏览器会下载这两个资源，请参阅下面的Larry的建议来解决这个问题。

2014年更新：

  • 如果你想要一个非交互式的svg，使用＆lt; img＆gt;脚本回退 到png版本（对于较老的IE和android＆lt; 3）。一个干净，简单 方法：

＆lt; img src =“your.svg”onerror =“this.src ='your.png'”＆gt;。

这将表现得非常像GIF图像，如果您的浏览器支持声明性动画（SMIL），那么这些将会播放。

  • 如果您需要交互式svg，请使用＆lt; iframe＆gt;或＆lt; object＆gt;。

  • 如果您需要提供旧版浏览器，则可以使用svg插件，然后使用＆lt; embed＆gt;。

  • 对于css background-image中的svg和类似的属性， modernizr 是一个选择切换到备用图片，另一个取决于多个背景自动执行：

      div {
background-image：url（fallback.png）;
background-image：url（your.svg），none;

    }

注意_ **：多重背景策略在Android 2.3上不起作用，因为它支持多种背景，但不支持svg。

这篇博文是关于svg fallback的。

＆lt; object＆gt;和＆lt; embed＆gt;具有一个有趣的属性：它们可以从外部文档获取对SVG文档的引用政策考虑在内）。然后可以使用该引用来为SVG设置动画，更改样式表等。

鉴于

<object id="svg1" data="/static/image.svg" type="image/svg+xml"></object>

然后，你可以做一些事情，比如

document.getElementById("svg1").addEventListener("load", function() {
    var doc = this.getSVGDocument();
    var rect = doc.querySelector("rect"); // suppose our image contains a <rect>
    rect.setAttribute("fill", "green");
});
