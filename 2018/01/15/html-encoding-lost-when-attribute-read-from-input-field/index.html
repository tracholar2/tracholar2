从输入字段读取属性时，HTML编码丢失

By zhizi

Published 2018-01-15

Contents

我使用JavaScript从隐藏字段中提取值并将其显示在文本框中。隐藏字段中的值被编码。

例如，

<input id='hiddenId' type='hidden' value='chalk &amp; cheese' />

被拉进

<input type='text' value='chalk &amp; cheese' />

通过一些jQuery从隐藏字段获取值（在这一点上，我失去了编码）：

$('#hiddenId').attr('value')

问题是，当我阅读chalk＆amp; amp; amp; amp; amp; amp; amp; amp; amp; amp;奶酪从隐藏的领域，JavaScript似乎失去了编码。要转义“和'，我希望编码保持不变。

有没有一个JavaScript库或一个jQuery方法将HTML编码的字符串？

我使用这些功能：

function htmlEncode(value){
  // Create a in-memory div, set its inner text (which jQuery automatically encodes)
  // Then grab the encoded contents back out. The div never exists on the page.
  return $('<div/>').text(value).html();
}

function htmlDecode(value){
  return $('<div/>').html(value).text();
}

基本上div元素是在内存中创建的，但是它永远不会附加到文档中。

在htmlEncode函数中，我设置了元素的innerText，并检索了编码后的innerHTML。在htmlDecode函数中，我设置元素的innerHTML值，并检索innerText。

检查一个正在运行的示例此处。

jQuery技巧不会对引号进行编码，在IE中，它会将您的空格删除。

基于Django中的 escape 模板标签，我猜测这个标签已被大量使用/测试过了，所以我开发了这个功能来完成需要的功能。

与whitespace-stripping问题的任何解决方法相比，它可以说是更简单（也可能更快），并且它会对引号进行编码，例如，如果要使用属性值内的结果，
p>

function htmlEscape(str) {
    return str
        .replace(/&/g, '&amp;')
        .replace(/"/g, '&quot;')
        .replace(/'/g, '&#39;')
        .replace(/</g, '&lt;')
        .replace(/>/g, '&gt;');
}

// I needed the opposite function today, so adding here too:
function htmlUnescape(str){
    return str
        .replace(/&quot;/g, '"')
        .replace(/&#39;/g, "'")
        .replace(/&lt;/g, '<')
        .replace(/&gt;/g, '>')
        .replace(/&amp;/g, '&');
}
