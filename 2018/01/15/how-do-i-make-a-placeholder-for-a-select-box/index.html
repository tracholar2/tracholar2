如何为“选择”框创建一个占位符？

By zhizi

Published 2018-01-15

Contents

我正在使用占位符来处理文本输入。但是我想为我的选择框使用一个占位符。当然，我可以使用这个代码：

<select>
    <option value="">Select your option</option>
    <option value="hurr">Durr</option>
</select>

但是，“选择你的选择”是黑色而不是浅灰色。所以我的解决方案可能是基于CSS的。 jQuery也很好。

这只会使下拉菜单变成灰色（所以点击箭头后）：

option:first {
    color: #999;
}

编辑：问题是：人们如何在选择框中创建占位符？但已经得到回答，欢呼。

使用这个结果，选定的值总是灰色的（即使在选择一个真实的选项后）：

select {
    color:#999;
}

如何一个非CSS - 没有javascript / jQuery的答案？

<select>
    <option value="" disabled selected>Select your option</option>
    <option value="hurr">Durr</option>
</select>

只是偶然发现了这个问题，这里是FireFox＆amp; Chrome（至少）

<style>
    select:invalid { color: gray; }
</style>
<form>
    <select required>
        <option value="" disabled selected hidden>Please Choose...</option>
        <option value="0">Open when powered (most valves do this)</option>
        <option value="1">Closed when powered, auto-opens when power is cut</option>
    </select>
</form>

Disabled选项停止使用鼠标和键盘选择的＆lt; option＆gt;，而使用'display：none'允许用户仍然通过键盘箭头。
'display：none'风格只是使列表框看起来很好。

注意：在“占位符”选项上使用一个空的值属性可以使验证（必需属性）具有“占位符”，所以如果选项没有改变，但是是必需的;浏览器应该提示用户从列表中选择一个选项。

更新（2015年7月）：**

此方法已在下列浏览器中得到确认：

  • Google Chrome - v.43.0.2357.132
  • Mozilla Firefox - v.39.0
  • Safari - v.8.0.7（占位符在下拉列表中可见，但不可选）
  • Microsoft Internet Explorer - v.11（占位符在下拉列表中可见，但不可选）
  • Project Spartan - v.15.10130（占位符在下拉列表中可见，但不可选）

更新（2015年10月）：**

删除了style =“display：none”以支持广泛支持的HTML5属性hidden。 IE浏览器（Project
Spartan需要检查）的hidden元素与display：none具有相似的特征，选项在下拉列表中可见，但不可选。

更新（2016年1月）：**

当select元素是required时，它允许使用：invalid CSS伪类，它允许您设置处于“占位符”状态时选择元素。
：invalid因为占位符选项中的空值而在这里工作。

一旦设置了一个值，：invalid伪类将被删除。如果您愿意的话，您也可以选择使用：valid。
