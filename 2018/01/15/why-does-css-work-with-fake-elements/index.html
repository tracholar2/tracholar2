为什么CSS使用假元素？

By zhizi

Published 2018-01-15

Contents

在我的课堂上，我在玩耍，发现CSS使用了构成元素

示例：

imsocool {
    color:blue;
}


<imsocool>HELLO</imsocool>

当我的教授第一次看到我使用这个，他有点惊讶，化妆元素的工作，并建议我只是改变我所有的元素编号段的ID。

为什么我的教授不想让我使用化妆元素？他们有效地工作。

另外，为什么他不知道构成元素存在和使用CSS。他们是不是很普遍？

Why does CSS work with fake elements?

（大多数）浏览器被设计为（在某种程度上）与将来添加到HTML的前向兼容。无法识别的元素被分析到DOM中，但是没有语义或与之相关的专门的默认渲染。

当一个新的元素被添加到规范中时，有时CSS，JavaScript和ARIA可以被用来在旧版本的浏览器中提供相同的功能（并且元素必须出现在DOM中才能被这些语言操纵添加该功能）。

（虽然应该指出，目前正在开展工作来定义一种方法来使用自定义元素来扩展HTML
，但这项工作目前还处于发展的初期阶段，所以应该避免到成熟为止）。

Why doesn’t my professor want me to use made-up elements?

  • HTML规范不允许
  • 它们可能与将来的标准元素具有相同的名称冲突
  • 可能有一个现有的HTML元素更适合该任务

Also; why didn’t he know that made-up elements existed and worked with CSS.
Are they uncommon?

是。人们不使用他们，因为他们有上述问题。

<强> TL; DR

  • 自定义标记在HTML中无效。这可能会导致呈现问题。
  • 由于代码不可移植，使未来的开发变得更加困难。
  • 有效的HTML提供了许多好处，如搜索引擎优化，速度和专业性。

长答复

一些参数与自定义标签的代码更加可用

然而，它导致无效的HTML。这对您的网站不利。

**有效的CSS / HTML点| StackOverflow的

  • Google prefers it so it is good for SEO.

  • It makes your web page more likely to work in browsers you haven’t
    tested.

  • It makes you look more professional (to some developers at least)

  • Compliant browsers can render [valid HTML faster]

  • It points out a bunch of obscure bugs you’ve probably missed that affect
    things you probably haven’t tested e.g. the codepage or language set of the
    page.

>

  • Validation as a debugging tool

  • Validation as a future-proof quality check

  • Validation eases maintenance

  • Validation helps teach good practices

  • Validation is a sign of professionalism
