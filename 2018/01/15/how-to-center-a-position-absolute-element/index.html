我将一个将position属性设置为absolute的元素居中的问题。 有谁知道为什么图像不居中？

body {
  text-align: center;
}

#slideshowWrapper {
  margin-top: 50px;
  text-align: center;
}

ul#slideshow {
  list-style: none;
  position: relative;
  margin: auto;
}

ul#slideshow li {
  position: absolute;
}

ul#slideshow li img {
  border: 1px solid #ccc;
  padding: 4px;
  height: 450px;
}


<body>
  <div id="slideshowWrapper">
    <ul id="slideshow">
      <li><img src="img/dummy1.JPG" alt="Dummy 1" /></li>
      <li><img src="img/piano_unlicened.JPG" alt="Dummy 2" /></li>
    </ul>
  </div>
</body>

position: absolute;
margin-left: auto;
margin-right: auto;

right: 0;

如果不知道定位的 1 元素的width / height，仍然可以如下对齐它：

示例 

.child {
    position: absolute;
    top: 50%;  /* position the top  edge of the element at the middle of the parent */
    left: 50%; /* position the left edge of the element at the middle of the parent */

    transform: translate(-50%, -50%); /* This is a shorthand of
                                         translateX(-50%) and translateY(-50%) */
}

值得注意的是， CSS转换 [。 （供应商前缀为简洁起见）_

Explanation

添加 top / left 50％移动上/左元素的边缘边缘到父项的中间，并且 translate（） 函数的 （负） 值<-50％`将元素移动一半的大小。因此，该元素将位于中间。

这是因为 top 上的百分比值/ left 属性是相对于height /
width父元素（正在创建一个包含块）。

translate（） ](http://dev.w3.org/csswg/css-transforms/#funcdef-
translate)的百分比值<一个href =“https://developer.mozilla.org/en-
US/docs/Web/CSS/transform”> transform功能是相对于元素本身的宽度/高度而言的（实际上它是指到边界框）。

对于单向比对，请使用translateX（-50％）或translateY（-50％）。

1。 position元素不是static。即relative，absolute，fixed values。
