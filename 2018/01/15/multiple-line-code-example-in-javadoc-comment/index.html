Javadoc注释中的多行代码示例

By zhizi

发表于 2018-01-15

文章目录

我有一个小代码示例，我想在一个方法的Javadoc注释中加入。

/**
 * -- ex: looping through List of Map objects --
 * <code>
 * for (int i = 0; i < list.size(); i++) {
 *      Map map = (Map)list.get(i);
 *      System.out.println(map.get("wordID"));
 *      System.out.println(map.get("word"));
 * }
 * </code>
 * 
 * @param query - select statement
 * @return List of Map objects
 */

问题是Javadoc中显示的代码示例没有换行符，很难阅读。

-- ex: looping through List of Map objects -- for (int i = 0; i list.size(); i++) { Map map = (Map)list.get(i); System.out.println(map.get("wordID")); System.out.println(map.get("word")); } 
Parameters
query - - select statement 
Returns:
List of Map objects

我想我错了，假设代码标签将处理换行符。在Javadoc注释中设置代码示例的最佳方式是什么？

除了已经提到的＆lt; pre＆gt;标签外，还应该使用@code JavaDoc注释，这将使HTML实体问题（特别是泛型），例如：

* <pre>
* {@code
* Set<String> s;
* System.out.println(s);
* }
* </pre>

将提供正确的HTML输出：

Set<String> s;
System.out.println(s);

忽略@code块（或者使用＆lt; code＆gt;标记）会导致HTML如下所示：

Set s;
System.out.println(s);

（仅供参考，可以在这里找到Java SE 8标记描述：Javadoc标签）

在javadoc评论中包含特定的代码示例，我感到非常难过。我想分享这一个。
请注意以下几点：

  • 使用旧的＆lt; code＆gt; - 标记来防止大括号被解析
  • 使用“新”{@ code ...} - 标记以获取输出中包含的泛型
  • 通过“{@ literal @}覆盖”@Override `中的@符号转义，因为javadoc生成器“倾斜”了，因为@直接进入打开大括号后
  • 在{@ code和{@ literal前面删除一个空格，以补偿内部空间并保持对齐状态

javadoc代码：

/** this methods adds a specific translator from one type to another type. `      * i.e.
  * <pre>
  * <code>new BeanTranslator.Builder()
  *   .translate(
  *     new{@code Translator<String, Integer>}(String.class, Integer.class){
  *      {@literal @}Override
  *       public Integer translate(String instance) {
  *         return Integer.valueOf(instance);
  *       }})
  *   .build();
  * </code>
  * </pre>
  * @param translator
  */

打印为

new BeanTranslator.Builder()
  .translate(
    new Translator<String, Integer>(String.class, Integer.class){
      @Override
      public Integer translate(String instance) {
        return Integer.valueOf(instance);
      }})
  .build();
