我有一个包含两列的布局 - 左边的div和右边的div。

正确的div有一个灰色background-color，我需要它根据用户的浏览器窗口的高度来垂直扩展。现在，background- color在div的最后一个内容处结束。

我试过height：100％，min-height：100％;等。

有几个CSS3测量单位称为：

[Viewport-Percentage (or Viewport-Relative)

Lengths](http://www.w3.org/TR/css3-values/#viewport-relative-lengths)

What are Viewport-Percentage Lengths?

从上面链接的W3候选建议：

The viewport-percentage lengths are relative to the size of the initial
containing block. When the height or width of the initial containing block is
changed, they are scaled accordingly.

这些单位是vh（视口高度），vw（视口宽度），vmin （视口最小长度）和vmax（视口最大长度）。

How can this be used to make a divider fill the height of the browser?

对于这个问题，我们可以使用vh：1vh等于视口高度的1％。也就是说，无论元素在DOM树中的位置如何，100vh都等于浏览器窗口的高度：

HTML

<div></div>

CSS

div {
    height:100vh;
}

这是字面上所有需要的。这是一个正在使用的 JSFiddle示例 。

What browsers support these new units?

除了Opera Mini以外，所有最新的主流浏览器都支持这个功能。查看我可以使用… 进一步支持。

How can this be used with multiple columns?

在手边的问题的情况下，具有左和右分频器，这是一个JSFiddle示例，显示了包含vh和vw的两列布局。

How is 100vh different to 100%?

以这个布局为例：

<body style="height:100%">
    <div style="height:200px">
        <p style="height:100%; display:block;">Hello, world!</p>
    </div>
</body>

这里的p标记被设置为100％高度，但是因为它包含div具有200px的高度，所以200px的100％变成200px， _em>body高度的100％。使用100vh来代替

高度，意味着p标签将是body的100％高度。看看这个伴随着的JSFiddle，可以很容易地看到差异！

如果您要设置＆lt; div＆gt;或任何元素的高度，则应设置＆lt; body＆gt;的高度和＆lt; HTML＆gt;也是100％。那么你可以用100％来设置元素的高度：）

这是一个例子：

body, html {
  height: 100%;
}

#right {
  height: 100%;
}
