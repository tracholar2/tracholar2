    • 我从来没有见过 ＆lt; base＆gt;HTML标签实际上在任何地方使用。有没有陷阱的使用，这意味着我应该避免它？

    我从来没有注意到它在现代生产网站（或任何网站）上使用的事实让我感到震惊，尽管它似乎可能有用于简化我的网站上的链接的有用的应用程序。

    Edit

    使用基本标签几个星期后，我最终发现了一些使用基本标签的重要陷阱，使它比首次出现的更不理想。本质上，基本标签下对href ='＃topic'和href =''的更改与其默认行为不兼容，这种默认行为的变化很容易使第三方库以非常不可靠的方式控制在非常不可靠的状态，因为它们在逻辑上将依赖于默认的行为。通常这些变化是微妙的，并且在处理大型代码库时导致不太明显的问题。我从那以后创造了一个答案，详细说明了我所经历的问题。因此，在您承诺广泛部署＆lt; base＆gt;之前，请自行测试链接结果，这是我的新建议！

    在决定是否使用＆lt; base＆gt;标记之前，您需要了解它是如何工作的，可以使用的以及影响是什么，并最终超过其优点/缺点

    ＆lt; base＆gt;标签主要简化了在模板语言中创建相对链接，因为您无需担心每个 _链接中的当前上下文。

    你可以做例如

    <base href="${host}/${context}/${language}/">
...
<link rel="stylesheet" href="css/style.css" />
<script src="js/script.js"></script>
...
<a href="home">home</a>
<a href="faq">faq</a>
<a href="contact">contact</a>
...
<img src="img/logo.png" />

    而不是

    <link rel="stylesheet" href="/${context}/${language}/css/style.css" />
<script src="/${context}/${language}/js/script.js"></script>
...
<a href="/${context}/${language}/home">home</a>
<a href="/${context}/${language}/faq">faq</a>
<a href="/${context}/${language}/contact">contact</a>
...
<img src="/${context}/${language}/img/logo.png" />

    请注意，＆lt; base href＆gt;值以斜线结尾，否则将相对于最后一个路径解释。

    至于浏览器的兼容性，这只会导致IE中的问题。 ＆lt; base＆gt;标签在HTML中被指定为 ，而不是 具有结束标签＆lt; / base＆gt;，所以使用<代码>＆lt; base＆gt; ，不带结束标记。不过，IE6认为，＆lt; base＆gt;
    标签之后的整个内容在这种情况下被放置为＆lt; / strong>的 **HTML
    DOM树中的“base＆gt;元素。这可能会导致第一眼看不到Javascript / jQuery / CSS中的无法解释的问题，即在 html＆gt;
    body 等特定选择器中完全无法访问元素，直到您在HTML DOM检查器中发现应该有< code> base（和head）。

    一个常见的IE6修补程序正在使用IE条件注释来包含结束标记：

    <base href="http://example.com/en/"><!--[if lte IE 6]></base><![endif]-->

    如果您不关心W3 Validator，或者您已经使用了HTML5，那么您可以自行关闭它，每个webbrowser都支持它。

    <base href="http://example.com/en/" />

    关闭＆lt; base＆gt;标签也立即修复了 insanity在src中使用相对URI请求＆lt; script＆gt;无限循环。

    当在＆lt; base＆gt;标记中使用相对URI时，例如＆lt; base href =“// example.com/somefolder/” ＆gt;或＆lt; base href =“/ somefolder /”＆gt;。这将在IE6 /
    7/8中失败。然而，这不完全是浏览器的错。在＆lt; base＆gt;＆lt; / code＆gt;标记中使用相对URI是不对的。 [HTML4规范](http://www.w3.org/TR/html401/struct/links.html#h-12.4)声明，它应该是绝对URI ，因此从 http：// 或 https：// 方案开始。这已经在[ HTML5规范](http://www.w3.org/TR/html5 /document-metadata#attr-base- href)中被删除了。因此，如果您仅使用HTML5并只与HTML5兼容的浏览器定位，那么通过在＆lt; base＆gt;
    `标记中使用相对URI，您应该会很好。

    至于使用＆lt; a href =“＃anchor＆gt;＆lt; / code＆gt;命名/散列片断定位符，查询字符串定位符如＆lt; a href
    =”？foo = bar“＆gt ; ＆lt; code>＆lt; a href =“; foo = bar”＆gt;和＆lt; base＆gt;标签的路径片段锚点基本上声明 所有 相对链接， 包括 这些类型的锚。没有任何相对链接与当前的请求URI相关（如果没有＆lt; base＆gt;标签就会发生）。首先这可能会让初学者感到困惑。要正确构建这些锚点，基本上需要包含URI，

    <a href="${uri}#anchor">hash fragment</a>
<a href="${uri}?foo=bar">query string</a>
<a href="${uri};foo=bar">path fragment</a>

    $ {uri}在PHP中基本上转换为$ _ SERVER ['REQUEST_URI']，$ {pageContext.request.requestURI}在JSP中，和JSF中的＃{request.requestURI}。值得注意的是，像JSF这样的MVC框架的标签可以减少所有这些样板，并且不需要＆lt; base＆gt;。另见a.o.
    什么URL用于链接/导航到其他JSF页面。
