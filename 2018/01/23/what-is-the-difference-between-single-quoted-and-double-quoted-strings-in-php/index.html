PHP中单引号和双引号字符串有什么区别？

By zhizi

Published 2018-01-23

Contents

我不是PHP编程方面的专家，但我有点困惑，为什么我在PHP中看到一些代码，字符串放在单引号中，有时用双引号。

我只是在.NET或C语言中知道，如果是单引号，那就意味着它是一个字符，而不是一个字符串。

可以指定

PHP字符串 不只是以 两个
方式，而是以 四个 方式。

  • 单身引用的字符串 将几乎完全“按原样”显示。变量和大多数转义序列不会被解释。例外的是，要显示一个单引号，你可以用反斜杠\'转义它，并显示一个反斜杠，你可以用另一个反斜杠\\代码>（ 所以是的，即使单引号字符串解析 ）
  • Double引用字符串 将显示一系列转义字符（包括一些正则表达式），并且将评估字符串中的变量。这里重要的一点是 可以使用大括号来分隔要评估的变量的名称 。例如，让我们假设你有变量$ type，而你又是什么“$ types是”这将查找变量 $ types 。为了解决这个问题，使用 echo“{$ type}是”`您可以在左边大括号之前或之后放置大括号。看看字符串解析看看如何使用数组变量等。
  • Heredoc 字符串的语法与双引号字符串一样。它以＆lt;＆lt;开头。在这个操作符之后，提供一个标识符，然后是一个换行符。字符串本身遵循，然后再次相同的标识符来关闭报价。你不需要在这个语法中转义引号。
  • Nowdoc （自PHP 5.3.0开始）字符串语法的工作方式与单引号字符串基本相同。不同的是，甚至连单引号或反斜杠都不能逃脱。 nowdoc用与heredocs相同的`＆lt;＆lt;＆lt; / code＆gt;序列标识，但是后面的标识符用单引号括起来。 <代码>＆LT;＆LT;＆LT; ‘EOT’。 不在nowdoc中进行分析。

<强>速度结果 我不会太重视单引号比双引号快。他们在某些情况下可能会更快。这里有一篇文章[解释了单引号和双引号基本上同样快的一种方式PHP 4.3“（Useless Optimizations，底部是C）。此外，此 基准测试页也提供了单对双引号对比。大部分的比较是相同的。有一个比较，双引号比单引号慢。

事情得到双引号评估，但不是单一的：

$s = "dollars";
echo 'This costs a lot of $s.'; // This costs a lot of $s.
echo "This costs a lot of $s."; // This costs a lot of dollars.
