如何在Bash中的分隔符上分割字符串？

By zhizi

Published 2018-01-22

Contents

我把这个字符串保存在一个变量中：

IN="bla@some.com;john@home.com"

现在我想通过;分隔符来分割字符串，这样我就可以：

ADDR1="bla@some.com"
ADDR2="john@home.com"

我不一定需要ADDR1和ADDR2变量。如果它们是一个更好的数组元素。

在回答下面的建议后，我得出了以下的结论：

#!/usr/bin/env bash

IN="bla@some.com;john@home.com"

mails=$(echo $IN | tr ";" "\n")

for addr in $mails
do
    echo "> [$addr]"
done

输出：

> [bla@some.com]
> [john@home.com]

有一个解决方案，将 Internal_field_separator（IFS）设置为;。我不确定那个答案是怎么回事，你怎么把IFS重置为默认？

RE：IFS解决方案，我试过这个，它工作，我保留旧的IFS，然后恢复：

IN="bla@some.com;john@home.com"

OIFS=$IFS
IFS=';'
mails2=$IN
for x in $mails2
do
    echo "> [$x]"
done

IFS=$OIFS

顺便说一句，当我尝试

mails2=($IN)

在循环中打印时，只有第一个字符串，$ IN的括号没有括起来。

您可以设置内部字段分隔符（IFS）变量，然后让它解析数组。当这种情况发生在一个命令中，那么对IFS的赋值只发生在单个命令的环境中（到read）。然后它根据IFS变量值将输入解析成一个数组，然后迭代。

IFS=';' read -ra ADDR <<< "$IN"
for i in "${ADDR[@]}"; do
    # process "$i"
done

它将解析由;分隔的一行项目，并将其推送到一个数组中。用于处理整个$ IN的东西，每次输入由;分隔的一行输入：

while IFS=';' read -ra ADDR; do
     for i in "${ADDR[@]}"; do
         # process "$i"
     done
done <<< "$IN"

取自 Bash shell脚本拆分数组 ：

IN="bla@some.com;john@home.com"
arrIN=(${IN//;/ })

说明：

这个结构替换了IN中所有出现的''（单个空格），然后将空格分隔的字符串解释为一个数组（这是括号内的括号）。

在大括号内用'字符替换每个';'字符的语法称为参数扩展。

有一些常见的问题：

  • 如果原始字符串包含空格，则需要使用 IFS ：
    • <代码> IFS = ‘：’; arrIN =（$ IN）;未设置IFS; ` * 如果原始字符串中包含空格 和 ，则分隔符是新行，您可以将 IFS ：
    • <代码> IFS = $ ‘\ n’; arrIN =（$ IN）;未设置IFS; `
