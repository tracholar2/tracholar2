在构造函数中调用虚拟成员

By zhizi

Published 2018-01-08

Contents

我从ReSharper得到关于从我的对象构造函数调用虚拟成员的警告。为什么这是不该做的事？

当用C＃编写的对象被构​​造时，会发生什么是初始化器从大多数派生类到基类的顺序运行，然后构造函数按从基类到最派生类的顺序运行（请参阅Eric
Lippert的博客以了解详细信息至于为什么这是）。

另外在.NET中，对象不会像构造类型那样改变类型，而是从派生类型最多的方面开始，而方法表则是派生类型最多的类型。这意味着虚拟方法调用总是运行在最派生的类型上。

当你把这两个事实结合在一起的时候，你会留下一个问题：如果你在一个构造函数中调用一个虚方法，并且它不是继承层次结构中最派生的类型，那么它将被调用一个构造函数没有运行，因此可能不会处于适当的状态来调用该方法。

如果将类标记为密封，以确保它是继承层次结构中派生最多的类型，则此问题当然会得到缓解 - 在这种情况下，调用虚拟方法是非常安全的。

为了回答你的问题，考虑这个问题：当Child对象被实例化时，下面的代码会打印出什么？

class Parent
{
    public Parent()
    {
        DoSomething();
    }
    protected virtual void DoSomething() 
    {
    }
}

class Child : Parent
{
    private string foo;
    public Child() { foo = "HELLO"; }
    protected override void DoSomething()
    {
        Console.WriteLine(foo.ToLower());
    }
}

答案是，实际上NullReferenceException会被抛出，因为foo为空。一个对象的基础构造函数在它自己的构造函数之前被调用。通过在一个对象的构造函数中调用virtual，可以引入继承对象在完全初始化之前执行代码的可能性。
