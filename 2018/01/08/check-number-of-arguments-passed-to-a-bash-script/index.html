检查传递给Bash脚本的参数的数量

By zhizi

发表于 2018-01-08

文章目录
  1. 1. References:

我想我的Bash脚本打印一个错误消息，如果所需的参数不符合。

我试了下面的代码：

#!/bin/bash
echo Script name: $0
echo $# arguments 
if [$# -ne 1]; 
    then echo "illegal number of parameters"
fi

由于一些未知的原因，我有以下错误：

test: line 4: [2: command not found

我究竟做错了什么？

就像任何其他简单的命令一样，[ ... ]或test要求其参数之间有空格。

if [ "$#" -ne 1 ]; then
    echo "Illegal number of parameters"
fi

要么

if test "$#" -ne 1; then
    echo "Illegal number of parameters"
fi

在Bash中，更喜欢使用[[ ]]，因为它不会将单词拆分和路径名扩展为变量，除非它是表达式的一部分，否则引用可能不是必需的。

[[ $# -ne 1 ]]

它还具有一些其他功能，如未加引号的条件分组，模式匹配（与extglob扩展模式匹配）和正则表达式匹配。

以下示例检查参数是否有效。它允许一个或两个参数。

[[ ($# -eq 1 || ($# -eq 2 && $2 == <glob pattern>)) && $1 =~ <regex pattern> ]]

对于纯算术表达式，使用(( ))一些仍可能会更好，但它们仍然可能与它的算术运算符等[[ ]]，-eq，-ne，-lt，-le，或-gt -ge通过将表达式作为一个字符串参数：

A=1
[[ 'A + 1' -eq 2 ]] && echo true  ## Prints true.

如果您需要将其与[[ ]]的其他功能相结合，这应该是有帮助的。

References:

打击条件表达式 有条件的构造 模式匹配 分词 文件名扩展（prev。路径名扩展）

如果你正在处理数字，那么使用算术表达式可能是一个好主意。

if (( $# != 1 )); then
    echo "Illegal number of parameters"
fi
