对于大多数情况下，在单语言中，英文代码库这个回答就足够了。我怀疑是因为大多数来这里的人都属于这个类别，这是最受欢迎的答案
然而，这个答案带来了固有的问题，我们不能比较文本不区分大小写，直到我们知道两个文本是相同的文化我们知道那是什么文化这可能是一个不太受欢迎的答案，但我认为这是更正确的，这就是为什么我这样标记。

测试字符串段落是否包含字符串word（谢谢@QuarterMeister）

culture.CompareInfo.IndexOf(paragraph, word, CompareOptions.IgnoreCase) >= 0

culture是 CultureInfo 。

这个解决方案对 不区分大小写的定义是透明的，这是依赖于语言的 。例如，对于第九个字母的大写和小写版本，英文使用字符I和i，而土耳其语使用这些字符作为第十一个和第十二个字母的29个字母长的字母表。土耳其的大写字母“i”是一个陌生的字符’Ä°’。

因此，字符串tin和TIN在英语中是相同的单词 _ 。据我所知，一个意思是“灵”，另一个是象声词。
（土耳其人，请纠正我，如果我错了，或建议一个更好的例子）

总而言之，只有回答这个问题：“这两个字符串是相同的，但是在不同的情况下，如果你知道文本是以什么样的语言写成的话。如果你不知道，你将不得不平息。鉴于英文在软件霸权，你应该诉诸于[CultureInfo.InvariantCulture / a>，因为用熟悉的方法会出错。

您可以使用[ String.IndexOf Method < / a>并传递 StringComparison.OrdinalIgnoreCase.aspx)作为要使用的搜索类型：

string title = "STRING";
bool contains = title.IndexOf("string", StringComparison.OrdinalIgnoreCase) >= 0;

更好的是定义一个新的字符串扩展方法：

public static class StringExtensions
{
    public static bool Contains(this string source, string toCheck, StringComparison comp)
    {
        return source?.IndexOf(toCheck, comp) >= 0;
    }
}
