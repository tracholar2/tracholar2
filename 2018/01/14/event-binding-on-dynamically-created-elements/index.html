事件绑定动态创建的元素？

By zhizi

Published 2018-01-14

Contents

我有一些代码在循环浏览页面上的所有选择框，并绑定一个.hover事件给他们，在`鼠标上做一些小小的旋转开/关

这种情况发生在页面准备就绪，工作得很好。

我遇到的问题是，在初始循环之后，通过Ajax或DOM添加的所有选择框都不会有事件绑定。

我找到了这个插件（ jQuery Live Query Plugin ），但是在我之前用插件添加另一个5k到我的网页，我想看看是否有人知道这样做，无论是直接或通过其他选择jQuery。

从jQuery 1.7 ，你应该使用 <代码> jQuery.fn.on ：

$(staticAncestors).on(eventName, dynamicChild, function() {});

之前，推荐的方法是使用[ ` live（） / code> ](http://api.jquery.com/live)：

$(selector).live( eventName, function(){} );

但是，live（）在1.7中已被弃用，而在（）中被弃用，并且在1.9中被完全删除。 live（）`签名：

$(selector).live( eventName, function(){} );

…可以替换为以下 on（） 签名：

$(document).on( eventName, selector, function(){} );

例如，如果你的页面是动态创建类名为dosomething的元素，你可以将事件绑定到已经存在的父对象上，通常是document。 >

$(document).on('mouseover mouseout', '.dosomething', function(){
    // what you want to happen when mouseover and mouseout 
    // occurs on elements that match '.dosomething'
});

任何在事件绑定时存在的父母都可以。例如

$('.buttons').on('click', 'button', function(){
    // do something here
});

将适用于

<div class="buttons">
    <!-- <button>s that are generated dynamically and added here -->
</div>

在[ jQuery.fn.on `文档中有一个很好的解释。一>

总之：

Event handlers are bound only to the currently selected elements; they must
exist on the page at the time your code makes the call to .on().

因此，在下面的例子中，代码生成之前必须存在#dataTable tbody tr。

$("#dataTable tbody tr").on("click", function(event){
    console.log($(this).text());
});

如果将新的HTML注入到页面中，则最好使用委托事件来附加事件处理程序，如下所述。

委托事件**具有的优点是，它们可以处理来自稍后添加到文档中的后代元素的事件。例如，如果表存在，但是行使用代码动态添加，则以下内容将处理它：

$("#dataTable tbody").on("click", "tr", function(event){
    console.log($(this).text());
});

除了能够处理尚未创建的后代元素上的事件之外，委托事件的另一个优点是当必须监视许多元素时，其开销要低得多。在其tbody中有1000行的数据表中，第一个代码示例将一个处理程序附加到1,000个元素。

一个委托事件方法（第二个代码示例）只将一个事件处理程序附加到一个元素tbody上，而事件只需要冒泡一个级别（从点击的tr to tbody）。

注意： 委派的事件不适用于 SVG。< / p>
