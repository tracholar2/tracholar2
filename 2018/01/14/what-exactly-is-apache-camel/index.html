请解释骆驼是什么。

如果你有5到10分钟的时间，我通常建议人们阅读这个[与Apache Camel的集成

乔纳森Anstey。这是一个写得很好的文章，对骆驼的一些概念进行了简要的介绍和概述，并且实现了一个带有代码示例的用例。在这里，乔纳森写道：

Apache Camel is an open source Java framework that focuses on making
integration easier and more accessible to developers. It does this by
providing:

>

  • concrete implementations of all the widely used Enterprise Integration
    Patterns     (EIPs)

  • connectivity to a great variety of transports and APIs

  • easy to use Domain Specific Languages (DSLs) to wire EIPs and transports
    together

>

也有免费的一章骆驼在行动在第一章介绍骆驼。乔纳森和我一起是这本书的合着者。

我用更容易的方式来描述这个…

为了理解Apache Camel是什么，你需要了解企业集成模式是什么。

我们从大概已经知道的东西开始吧：Singleton模式，Factory模式等;他们只是组织解决问题的方法，但它们本身并不是解决方案。这些模式在“四人帮”发表之后，我们为其他人进行了分析和提炼，当时他们发表了他们的书：设计模式。他们为我们如何最好地构建我们的代码节省了很多精力。

像四人帮，Gregor Hohpe和Bobby
Woolf写的这本书是企业集成模式（EIP），他们提出记录一套新的模式和蓝图，以便我们如何设计大型的基于组件的系统，其中组件可以运行在相同的进程或不同的机器上。
__ / p>
他们基本上建议我们将系统构造成面向消息的方式，即组件之间通过消息作为输入和输出进行通信，而绝对没有其他的东西。它们向我们展示了一套完整的模式，我们可以从中选择和实现不同的组件，这些组件将组成整个系统。

那么Apache Camel是什么？**

Apache
Camel为您提供了EIP接口，基础对象，常用实现，调试工具，配置系统以及许多其他帮助程序，这将为您节省大量的时间来实施您的解决方案电子信息产品。

以MVC。
MVC在理论上相当简单，我们可以在没有任何框架帮助的情况下实现它。但是良好的MVC框架为我们提供了随时可用的结构，并且在创建大型MVC项目时已经走了更多的步伐，并考虑了所需的所有其他“副”方面的东西，这也是我们大多数时候使用它们的原因。

这正是Apache Camel用于EIP的。对于那些想要实施他们的解决方案来跟踪EIP的人来说，这是一个完整的生产就绪框架。**
