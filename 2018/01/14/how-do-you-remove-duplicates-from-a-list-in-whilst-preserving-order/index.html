如何在维护秩序的同时从列表中删除重复项？

By zhizi

Published 2018-01-14

Contents

在保存命令的同时，是否有一个内置的从Python清单中删除重复项？我知道我可以使用一组来删除重复，但是破坏了原来的顺序。我也知道我可以像这样推出自己的产品：

def uniq(input):
  output = []
  for x in input:
    if x not in output:
      output.append(x)
  return output

（感谢放宽
代码示例）。

但是，如果可能的话，我想利用一个内置的或更多的Pythonic成语。

相关问题：在Python中，什么是从列表中删除重复项的最快算法，以便所有元素都是唯一的 ，同时保持顺序
？

这里有一些选择： http://www.peterbe.com/plog/uniqifiers-
基准

最快的一个：

def f7(seq):
    seen = set()
    seen_add = seen.add
    return [x for x in seq if not (x in seen or seen_add(x))]

为什么将seen.add赋值给seen_add，而不是仅仅调用seen.add？ Python是一种动态语言，解析seen.add每次迭代比解析一个局部变量更昂贵。 seen.add在迭代之间可能会发生变化，运行时不够聪明，无法排除。为了安全起见，每次都要检查对象。

如果你打算在同一个数据集上使用这个函数，也许你最好用一个有序集合： http://code.activestate.com/recipes/528878/

（1）每个操作的插入，删除和成员检查。

修改2016

正如Raymond 指出的那样，在C ++中实现OrderedDict的python 3.5+中，列表理解方法将比 慢（除非你真的需要列表在最后 -
即使如此，只有当输入是非常短的）。所以3.5+的最佳解决方案是OrderedDict。

重要编辑2015

正如 @abarnert 所指出的， more_itertools library（pip install more_itertools）包含一个[不可读的（代码> not seen.add `） 突变 。这也是最快的解决方案：

>>> from  more_itertools import unique_everseen
>>> items = [1, 2, 0, 1, 3, 2]
>>> list(unique_everseen(items))
[1, 2, 0, 3]

只是一个简单的库导入，没有黑客。 这来自itertools配方的实现 unique_everseen看起来像：

def unique_everseen(iterable, key=None):
    "List unique elements, preserving order. Remember all elements ever seen."
    # unique_everseen('AAAABBBCCDAABBB') --> A B C D
    # unique_everseen('ABBCcAD', str.lower) --> A B C D
    seen = set()
    seen_add = seen.add
    if key is None:
        for element in filterfalse(seen.__contains__, iterable):
            seen_add(element)
            yield element
    else:
        for element in iterable:
            k = key(element)
            if k not in seen:
                seen_add(k)
                yield element

在Python 2.7 +中， 接受了常用的习惯用法 （这个工作原理并没有针对速度进行优化，我现在要用 unique_everseen ）使用 collections.OrderedDict ：

运行时间： O（N）

>>> from collections import OrderedDict
>>> items = [1, 2, 0, 1, 3, 2]
>>> list(OrderedDict.fromkeys(items))
[1, 2, 0, 3]

这看起来好多了：

seen = set()
[x for x in seq if x not in seen and not seen.add(x)]

不使用 丑陋的黑客 ：

not seen.add(x)

这依赖于set.add是一个始终返回None所以not的计算结果为` >真。

然而请注意，尽管它具有相同的运行时间复杂度O（N），但是解决方案的速度在原始速度上更快。
