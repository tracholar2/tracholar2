为什么我不能打开String？

这个功能是否会被放到更高版本的Java中？

有人可以解释为什么我不能这样做，因为Java的switch语句的技术方式是工作吗？

使用String个案切换语句已经在 Java SE 7，首次提出要求后至少16年。没有提供明确的延迟原因，但可能与性能有关。

Implementation in JDK 7

该功能现已在javac 中实现，糖化“过程;在case声明中使用String常量的干净的高级语法在编译时被扩展为遵循模式的更复杂的代码。生成的代码使用一直存在的JVM指令。

在编译过程中，带有String的开关被翻译成两个开关。第一个将每个字符串映射到一个唯一的整数 -
它在原始开关中的位置。这是通过首先打开标签的哈希码完成的。相应的情况是一个if语句测试字符串相等;如果散列发生冲突，则测试是级联if-else- if。第二个开关反映了原始源代码中的内容，但将外壳标签替换为相应的位置。这两个步骤的过程可以很容易地保留原来开关的流量控制。

Switches in the JVM

关于switch的更多技术深度，您可以参考JVM规范，其中编译switch语句。简而言之，有两种不同的JVM指令可以用于开关，具体取决于案例使用的常量的稀疏性。两者都依赖于使用整数常量为每个案件高效执行。

如果常量是密集的，它们被用作索引（在减去最小值之后）到指令指针表（tableswitch指令）中。

如果常量稀疏，则执行正确大小写的二进制搜索 - lookupswitch指令。

在String对象中删除开关时，可能会使用这两个指令。 lookupswitch适用于哈希代码上的第一个开关，以查找案例的原始位置。得到的序号自然适用于tableswitch。

这两条指令都要求在编译时分配给每个case的整数常量。在运行时，tableswitch的O（1）性能通常比的O（log（n））lookupswitch `，它需要一些分析来确定表是否足够密集，以证明时空折衷。比尔·威纳斯（Bill
Venners）写了一篇很棒的文章，详细介绍了这方面的内容，其他Java流程控制指令。

Before JDK 7

在JDK 7之前，enum可以接近一个基于字符串的开关。这使用静态每个枚举类型由编译器生成的valueOf方法。例如：

Pill p = Pill.valueOf(str);
switch(p) {
  case RED:  pop();  break;
  case BLUE: push(); break;
}

如果你的代码中有一个可以打开字符串的地方，那么重构字符串可能会更好，因为它是可以打开的可能值的枚举。当然，你可以限制字符串在枚举中的潜在价值，这可能是也可能是不希望的。

当然你的枚举可以有一个“其他”和一个fromString（String）方法的条目，那么你可以有

ValueEnum enumval = ValueEnum.fromString(myString);
switch (enumval) {
   case MILK: lap(); break;
   case WATER: sip(); break;
   case BEER: quaff(); break;
   case OTHER: 
   default: dance(); break;
}
