错误java lang OutOfMemoryError：超出GC开销限制

By zhizi

发表于 2018-01-14

文章目录

当我执行我的JUnit测试时，得到这个错误消息：

java.lang.OutOfMemoryError: GC overhead limit exceeded

我知道OutOfMemoryError是什么，但GC开销限制是什么意思？我该如何解决这个问题？

这个消息意味着由于某种原因，垃圾收集器需要花费过多的时间（默认是进程的所有CPU时间的98％），并且在每次运行中恢复的内存很少（默认为堆的2％）

这实际上意味着你的程序停止了任何进度，并且一直忙于只运行垃圾回收。

为了防止应用程序在没有完成任何事情的情况下获取CPU时间，JVM会抛出这个Error，以便您有机会诊断问题。

我见过这种情况的罕见情况是，一些代码在已经非常受内存限制的环境中创建了大量的临时对象和大量的弱引用对象。

详细了解这篇文章 （特别是这部分）

GC在垃圾收集中花费太多时间返回太少的时候抛出这个异常，例如。 CPU占用了98％的CPU时间，并且只有不到2％的堆被恢复。

此功能旨在防止应用程序在很长时间内运行，而由于堆太小，进行很少或没有进展。

您可以使用命令行选项关闭此功能 <代码> -XX：-UseGCOverheadLimit
