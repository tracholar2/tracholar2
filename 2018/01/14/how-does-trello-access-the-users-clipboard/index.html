Trello如何访问用户的剪贴板？

By zhizi

Published 2018-01-14

Contents

当您将鼠标悬停在 Trello 的卡片上，然后按Ctrl + C，此卡的网址将被复制到剪贴板。他们如何做到这一点？

据我所知，没有涉及Flash电影。我已经安装了 Flashblock，并且Firefox网络选项卡没有显示加载的Flash影片。
（这是通常的方法，例如，ZeroClipboard。）

他们如何实现这个魔法？

（在这一刻，我想我有一个顿悟：你不能选择页面上的文本，所以我假设他们有一个不可见的元素，他们创建一个通过JavaScript代码的文本选择，Ctrl+ C触发浏览器的默认行为，复制不可见节点的文本值。）

披露：
我写了Trello使用的代码;下面的代码是Trello用来完成剪贴板技巧的实际源代码。

我们实际上并没有“访问用户的剪贴板”，而是通过在按下Ctrl + C时选择一些有用的内容来帮助用户。 p>
听起来你已经知道了，我们利用这个事实，即当你想要点击Ctrl + C时，你必须首先点击Ctrl键。当按下Ctrl键时，我们弹出一个包含我们想要在剪贴板上结束的文本的textarea，并选择其中的所有文本，所以当C键被击中。 （然后当Ctrl键出现时隐藏文本区域）

特别是，Trello这样做：

TrelloClipboard = new class
  constructor: ->
    @value = ""

    $(document).keydown (e) =>
      # Only do this if there's something to be put on the clipboard, and it
      # looks like they're starting a copy shortcut
      if !@value || !(e.ctrlKey || e.metaKey)
        return

      if $(e.target).is("input:visible,textarea:visible")
        return

      # Abort if it looks like they've selected some text (maybe they're trying
      # to copy out a bit of the description or something)
      if window.getSelection?()?.toString()
        return

      if document.selection?.createRange().text
        return

      _.defer =>
        $clipboardContainer = $("#clipboard-container")
        $clipboardContainer.empty().show()
        $("<textarea id='clipboard'></textarea>")
        .val(@value)
        .appendTo($clipboardContainer)
        .focus()
        .select()

    $(document).keyup (e) ->
      if $(e.target).is("#clipboard")
        $("#clipboard-container").empty().hide()

  set: (@value) ->

在DOM中我们有

<div id="clipboard-container"><textarea id="clipboard"></textarea></div>

用于剪贴板的CSS：

#clipboard-container {
  position: fixed;
  left: 0px;
  top: 0px;
  width: 0px;
  height: 0px;
  z-index: 100;
  display: none;
  opacity: 0;
}
#clipboard {
  width: 1px;
  height: 1px;       
  padding: 0px;
}

…和CSS使得它实际上看不到textarea当它弹出…但它是“可见的”，足以从中复制。

当你将鼠标悬停在卡上时，它会调用

TrelloClipboard.set(cardUrl)

…那么当按下Ctrl键时，剪贴板帮手知道要选择什么。

实际上，我制作了 Chrome扩展程序， ，并为所有的网页。源代码在GitHub上。

我发现Trello的方法有三个bug，我知道是因为我自己面对过他们。）

副本在这些情况下不起作用：

  • 如果您已经按Ctrl，然后悬停链接并点击C，则副本不起作用。
  • 如果您的光标位于页面的其他文本字段中，则复制不起作用。
  • 如果您的光标在地址栏中，则副本不起作用。

我通过始终隐藏范围解决了＃1问题，而不是在用户点击Ctrl / Cmd时创建一个。

通过暂时清除零长度选择，保存插入位置，复制和恢复插入位置，我解决了＃2问题。

我还没找到＃3的解决方法:)（有关信息，请查看我的GitHub项目中的开放问题）。
