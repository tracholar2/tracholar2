自我的目的是什么？

By zhizi

Published 2018-01-14

Contents

Python中self单词的用途是什么？我知道它是指从该类创建的特定对象，但我不明白为什么它明确需要作为参数添加到每个函数。为了说明，在Ruby中我可以这样做：

class myClass
    def myFunc(name)
        @name = name
    end
end

我很明白，这很容易。然而在Python中，我需要包含self：

class myClass:
    def myFunc(self, name):
        self.name = name

任何人都可以通过这个说话吗？这不是我在我的（当然是有限的）经验中遇到的。

您需要使用self。是因为Python不使用@语法来引用实例属性。
Python决定以一种方式来执行方法，使方法所属的实例自动被自动传递，而不是自动接收_：方法的第一个参数是方法的实例被调用。这使得方法和函数完全一样，并且留下实际的名字给你使用（尽管self是约定，当你使用别的东西的时候，人们通常会对你皱眉）。self对代码来说并不特别，它只是另一个对象。

Python本来可以做一些其他的东西来区分正常的名字和属性 - 像Ruby这样的特殊语法，或者像C ++和Java这样的声明，或者可能更不同的东西 -
但事实并非如此。
Python所有的东西都是明确的，明确了什么是什么，尽管它并不是完全在做，但它确实是为了实例。这就是为什么分配给实例属性需要知道分配给哪个实例的原因，这就是为什么它需要self。。

让我们来看一个简单的向量类：

class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y

我们想要一个计算长度的方法。如果我们想在课堂上定义它，它会是什么样子？

def length(self):
    return math.sqrt(self.x ** 2 + self.y ** 2)

当我们将其定义为全局方法/函数时，应该是什么样子？

def length_global(vector):
    return math.sqrt(vector.x ** 2 + vector.y ** 2)

所以整个结构保持不变。我怎么能利用这个？如果我们暂时假设我们没有为我们的Vector类写一个length方法，我们可以这样做：

Vector.length_new = length_global
v = Vector(3, 4)
print(v.length_new()) # 5.0

这是可行的，因为length_global的第一个参数可以被重新用作length_new中的self参数。如果没有明确的self，这是不可能的。

理解明确的self的另一种方法是看Python在哪里添加一些语法糖。当你记住的时候，基本上就是像

这样的电话

v_instance.length()

内部转换为

Vector.length(v_instance)

很容易看到self适合在哪里。您实际上并没有在Python中编写实例方法。你写的是必须把实例作为第一个参数的类方法。因此，您必须将实例参数放在明确的地方。
