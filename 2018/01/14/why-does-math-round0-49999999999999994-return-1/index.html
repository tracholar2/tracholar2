为什么Math round（0 49999999999999994）返回1？

By zhizi

Published 2018-01-14

Contents

在下面的程序中，你可以看到每个小于.5的值被向下取整，除了0.5。

for (int i = 10; i >= 0; i--) {
    long l = Double.doubleToLongBits(i + 0.5);
    double x;
    do {
        x = Double.longBitsToDouble(l);
        System.out.println(x + " rounded is " + Math.round(x));
        l--;
    } while (Math.round(x) > i);
}

我正在使用Java 6 update 31。

<强>摘要

round（x）在Java 6中（大概是早些时候），round（x）被实现为` floor（x + 0.5）。一个规范错误，正是这个病态的情况。
2 Java 7不再要求这个破坏的实现。 3

问题

0.5 + 0.49999999999999994正好是1的双精度：

static void print(double d) {
    System.out.printf("%016x\n", Double.doubleToLongBits(d));
}

public static void main(String args[]) {
    double a = 0.5;
    double b = 0.49999999999999994;

    print(a);      // 3fe0000000000000
    print(b);      // 3fdfffffffffffff
    print(a+b);    // 3ff0000000000000
    print(1.0);    // 3ff0000000000000
}

这是因为0.49999999999999994的指数小于0.5，所以当它们被添加时，它的尾数是移位的，并且ULP变大了。

解决方案

从Java 7开始，OpenJDK（例如）就是这样实现的： 4

public static long round(double a) {
    if (a != 0x1.fffffffffffffp-2) // greatest double value less than 0.5
        return (long)floor(a + 0.5d);
    else
        return 0;
}
