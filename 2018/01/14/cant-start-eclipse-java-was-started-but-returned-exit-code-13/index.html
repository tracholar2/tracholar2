无法启动Eclipse - Java已启动，但返回退出代码= 13

By zhizi

Published 2018-01-14

Contents

我尝试着使用Eclipse来初次体验Android开发。试图运行Eclipse时遇到了这个问题，仅在几分钟前安装了4.2版本。

在第一次尝试启动没有任何参数指定Java
VM的Eclipse之后，我收到一条错误消息，说它在Eclipse文件夹中找不到一个名为javaw.exe的Java
VM，所以我找到了Java安装和指定的位置该位置作为快捷方式目标中的参数。现在我得到一个不同的错误，“Java已经启动，但返回退出代码= 13”。

类似的问题似乎表明，这是一个32位/ 64位的冲突，但我99％的积极，我下载了Eclipse和Java（RC
7u5）的64位版本，我选择，因为我有64位的Windows 7.

OS，JDK和eclipse
bitness的工作组合。在我的情况下，我在64位操作系统中使用64位JDK和32位eclipse。将JDK降级到32位eclipse后开始工作。
请使用以下1种组合。

  • 32位操作系统，32位JDK，32位Eclipse（仅限32位）
  • 64位操作系统，32位JDK，32位Eclipse
  • 64位操作系统，64位JDK，64位Eclipse（仅限64位）

希望这可以帮助你。

您的Eclipse版本是基于路径和文件名的64位版本。 然而，它所采用的Java版本是32位的，就像它来自哪里，就是这条线所表示的：

-vm C:\Program Files (x86)\Java\jre7\bin\javaw.exe

Program Files（x86）`是64位Windows放置 32位 程序的文件夹。

Program Files `是64位Windows放置 64位 程序的文件夹。

当系统安装了多个JVM时，可能会发生这种情况，在Windows
64位上经常出现这种情况（例如，JRE下载页面使用浏览器的位宽来确定将哪些位下载提供给你，许多人使用（d）32位浏览器，即使他们运行的是64位Windows）。

假如你的系统上有64位的JRE或JDK，最好的解决办法就是在eclipse.ini中指定你想使用哪个JVM。这些指令在 Eclipse
wiki页面中详细介绍，但基本上你必须指定- vm选项 -
确保仔细阅读wiki页面，因为格式非常特殊。

强烈建议在eclipse.ini中指定JVM路径，因为这样做会将Eclipse与某些程序安装程序可能对系统执行的任何潜在更改（PATH
`跟你说话，甲骨文！）。

另一个选择是下载和使用32位Eclipse而不是64位，但是强烈建议在eclipse.ini中指定JVM的路径。

要检查您的Java版本，请运行

java -version

在控制台（命令提示符）中。在具有64位Java 6的Windows 7上，我得到：

java version "1.6.0_27"
Java(TM) SE Runtime Environment (build 1.6.0_27-b07)
Java HotSpot(TM) 64-Bit Server VM (build 20.2-b06, mixed mode)

注意第三行，这表明这是一个64位版本。

在32位版本中，您将得到如下所示的内容：

Java HotSpot(TM) Client VM (build 20.1-b02, mixed mode, sharing)
