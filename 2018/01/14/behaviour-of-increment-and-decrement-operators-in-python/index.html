我注意到一个预先增加/减少的操作符可以应用于一个变量（如++ count）。它编译，但它并没有真正改变变量的值！

Python中预增/减运算符（++ / - ）的行为是什么？

为什么Python偏离了在C / C ++中看到的这些运算符的行为？

++不是一个运算符。它是两个+运算符。 +运算符是 标识 运算符，它不做任何事情。 （澄清：+和-一元运算符只能用于数字，但我认为你不会期望一个假设的++运算符在字符串上工作。）

++count

解析为

+(+count)

转换为

count

您必须使用略长的+ =运算符来执行您想要的操作：

count += 1

当你想增加或减少，你通常想要在一个整数上做到这一点。像这样：

b++

但在Python中，整数是 不可变的 。那是你不能改变他们。这是因为整数对象可以使用几个名字。试试这个：

>>> b = 5
>>> a = 5
>>> id(a)
162334512
>>> id(b)
162334512
>>> a is b
True

上面的a和b实际上是同一个对象。如果你增加a，你也会增加b。这不是你想要的。所以你必须分配。像这样：

b = b + 1

或者更简单：

b += 1

将 **重新分配给 b + 1 。这不是一个增量运算符，因为它不会增加 b `，它会重新分配它。
