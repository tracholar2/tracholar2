context：annotation-config vs context：component-scan之间的区别

By zhizi

发表于 2018-01-14

文章目录

我正在学习Spring 3，而且似乎并没有掌握＆lt; context：annotation-config＆gt;和＆lt; context ：component-scan＆gt;
从我读过的，他们似乎处理不同的注释（@Required，@Autowired等与@Component，@Repository，@Service等），但也从我读了他们注册相同的豆后处理器类。

更令我困惑的是，＆lt; context：component-scan＆gt;上有annotation-config属性。
有人可以看到这些标签？什么是相似的，什么是不同的，是一个被另一个取代的，他们完成了对方，我需要其中的一个吗？

＆lt; context：annotation-config＆gt;用于激活已在应用程序上下文中注册的bean中的注释（不管它们是使用XML还是通过包扫描来定义的） ＆lt; context：component- scan＆gt;也可以执行＆lt; context：annotation-config＆gt;的操作，＆lt; context ：component-scan＆gt;也扫描软件包以在应用程序上下文中查找和注册bean。

我会用一些例子来说明差异/相似之处。

让我们从A，B和C的三个bean的基本设置开始，其中B和C被注入A。

package com.xxx;
public class B {
  public B() {
    System.out.println("creating bean B: " + this);
  }
}

package com.xxx;
public class C {
  public C() {
    System.out.println("creating bean C: " + this);
  }
}

package com.yyy;
import com.xxx.B;
import com.xxx.C;
public class A { 
  private B bbb;
  private C ccc;
  public A() {
    System.out.println("creating bean A: " + this);
  }
  public void setBbb(B bbb) {
    System.out.println("setting A.bbb with " + bbb);
    this.bbb = bbb;
  }
  public void setCcc(C ccc) {
    System.out.println("setting A.ccc with " + ccc);
    this.ccc = ccc; 
  }
}

使用以下XML配置：

<bean id="bBean" class="com.xxx.B" />
<bean id="cBean" class="com.xxx.C" />
<bean id="aBean" class="com.yyy.A">
  <property name="bbb" ref="bBean" />
  <property name="ccc" ref="cBean" />
</bean>

加载上下文产生以下输出：

creating bean B: com.xxx.B@c2ff5
creating bean C: com.xxx.C@1e8a1f6
creating bean A: com.yyy.A@1e152c5
setting A.bbb with com.xxx.B@c2ff5
setting A.ccc with com.xxx.C@1e8a1f6

好的，这是预期的输出。但这是“旧式”的春天。现在我们有了注释，所以可以使用这些注释来简化XML。

首先，让我们在bean A上自动装载bbb和ccc属性，如下所示：

package com.yyy;
import org.springframework.beans.factory.annotation.Autowired;
import com.xxx.B;
import com.xxx.C;
public class A { 
  private B bbb;
  private C ccc;
  public A() {
    System.out.println("creating bean A: " + this);
  }
  @Autowired
  public void setBbb(B bbb) {
    System.out.println("setting A.bbb with " + bbb);
    this.bbb = bbb;
  }
  @Autowired
  public void setCcc(C ccc) {
    System.out.println("setting A.ccc with " + ccc);
    this.ccc = ccc;
  }
}

这使我可以从XML中删除以下行：

<property name="bbb" ref="bBean" />
<property name="ccc" ref="cBean" />

我的XML现在简化为：

<bean id="bBean" class="com.xxx.B" />
<bean id="cBean" class="com.xxx.C" />
<bean id="aBean" class="com.yyy.A" />

当我加载上下文时，我得到以下输出：

creating bean B: com.xxx.B@5e5a50
creating bean C: com.xxx.C@54a328
creating bean A: com.yyy.A@a3d4cf

好的，这是错误的！发生了什么？为什么不是我的属性自动装配？

好吧，注释是一个很好的功能，但它们本身什么也不做。他们只是注释的东西。您需要一个处理工具来查找注释并与他们做一些事情。

＆lt; context：annotation-config＆gt;来拯救。这激活了它在自己定义的相同应用程序上下文中定义的bean上找到的注释的动作。

如果我将XML更改为：

<context:annotation-config />
<bean id="bBean" class="com.xxx.B" />
<bean id="cBean" class="com.xxx.C" />
<bean id="aBean" class="com.yyy.A" />

当我加载应用程序上下文时，我得到了正确的结果：

creating bean B: com.xxx.B@15663a2
creating bean C: com.xxx.C@cd5f8b
creating bean A: com.yyy.A@157aa53
setting A.bbb with com.xxx.B@15663a2
setting A.ccc with com.xxx.C@cd5f8b

好吧，这很好，但是我从XML中删除了两行并添加了一行。这不是一个很大的区别。注释的想法是它应该删除XML。

那么让我们删除XML定义，并用注释替换它们：

package com.xxx;
import org.springframework.stereotype.Component;
@Component
public class B {
  public B() {
    System.out.println("creating bean B: " + this);
  }
}

package com.xxx;
import org.springframework.stereotype.Component;
@Component
public class C {
  public C() {
    System.out.println("creating bean C: " + this);
  }
}

package com.yyy;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Component;
import com.xxx.B;
import com.xxx.C;
@Component
public class A { 
  private B bbb;
  private C ccc;
  public A() {
    System.out.println("creating bean A: " + this);
  }
  @Autowired
  public void setBbb(B bbb) {
    System.out.println("setting A.bbb with " + bbb);
    this.bbb = bbb;
  }
  @Autowired
  public void setCcc(C ccc) {
    System.out.println("setting A.ccc with " + ccc);
    this.ccc = ccc;
  }
}

在XML中我们只保留这个：

<context:annotation-config />

我们加载上下文，结果是…没有。没有bean被创建，没有bean被自动装配。没事！

这是因为，正如我在第一段中所说的那样，＆lt; context：annotation-config /＆gt;仅适用于在应用程序上下文中注册的bean。因为我删除了三个bean的XML配置，所以没有创建bean，并且＆lt; context ：annotation-config /＆gt;没有可用的“目标”。

但是这对＆lt; context：component-scan＆gt;来说不是问题，它可以扫描一个包来处理“目标”。我们将XML配置的内容更改为以下条目：

<context:component-scan base-package="com.xxx" />

当我加载上下文时，我得到以下输出：

creating bean B: com.xxx.B@1be0f0a
creating bean C: com.xxx.C@80d1ff

嗯…有东西丢失了。为什么？

如果你仔细看看这些类，class A有package com.yyy，但是我已经在＆lt; context：component- scan＆gt;使用包com.xxx，所以这完全错过了我的A类，只拾取了B和C代码>位于com.xxx包中。

为了解决这个问题，我还添加了这个包：

<context:component-scan base-package="com.xxx,com.yyy" />

现在我们得到了预期的结果：

creating bean B: com.xxx.B@cd5f8b
creating bean C: com.xxx.C@15ac3c9
creating bean A: com.yyy.A@ec4a87
setting A.bbb with com.xxx.B@cd5f8b
setting A.ccc with com.xxx.C@15ac3c9

就是这样！现在你再也没有XML定义了，你有了注释。

作为最后一个例子，保留注解的类A，B和C，并将以下内容添加到XML中，加载上下文后？

<context:component-scan base-package="com.xxx" />
<bean id="aBean" class="com.yyy.A" />

我们仍然得到正确的结果：

creating bean B: com.xxx.B@157aa53
creating bean C: com.xxx.C@ec4a87
creating bean A: com.yyy.A@1d64c37
setting A.bbb with com.xxx.B@157aa53
setting A.ccc with com.xxx.C@ec4a87

即使没有通过扫描获得类A的bean，处理工具仍然通过＆lt; context：component-scan＆gt;在所有注册的bean上应用
在应用程序上下文中，即使是手动在XML中注册的A。

但是如果我们有下面的XML，那么我们是否会得到重复的bean，因为我们已经指定了＆lt; context：annotation-config /＆gt;和`＆lt; context：component-扫描＆GT;

<context:annotation-config />
<context:component-scan base-package="com.xxx" />
<bean id="aBean" class="com.yyy.A" />

不，没有重复，我们再次得到预期的结果：

creating bean B: com.xxx.B@157aa53
creating bean C: com.xxx.C@ec4a87
creating bean A: com.yyy.A@1d64c37
setting A.bbb with com.xxx.B@157aa53
setting A.ccc with com.xxx.C@ec4a87

这是因为这两个标签注册了相同的处理工具（＆lt; context：annotation-config /＆gt;），如果＆lt; context ：component-scan＆gt;是指定），但Spring只负责运行一次。

即使你自己多次注册加工工具，Spring仍然会确保他们只做一次魔法;这个XML：

<context:annotation-config />
<context:component-scan base-package="com.xxx" />
<bean id="aBean" class="com.yyy.A" />
<bean id="bla" class="org.springframework.beans.factory.annotation.AutowiredAnnotationBeanPostProcessor" />
<bean id="bla1" class="org.springframework.beans.factory.annotation.AutowiredAnnotationBeanPostProcessor" />
<bean id="bla2" class="org.springframework.beans.factory.annotation.AutowiredAnnotationBeanPostProcessor" />
<bean id="bla3" class="org.springframework.beans.factory.annotation.AutowiredAnnotationBeanPostProcessor" />

仍然会产生以下结果：

creating bean B: com.xxx.B@157aa53
creating bean C: com.xxx.C@ec4a87
creating bean A: com.yyy.A@25d2b2
setting A.bbb with com.xxx.B@157aa53
setting A.ccc with com.xxx.C@ec4a87

好吧，关于它的说唱。

我希望这个信息以及来自@Tomasz Nurkiewicz和@Sean Patrick Floyd的回复是你需要了解的 ＆lt; context ：annotation-config＆gt;和＆lt; context：component-scan＆gt;工作。

我发现这个不错的[摘要＆lt; context：component-scan /＆gt; 识别由＆lt; context：annotation-
config /＆gt; `识别的注释的超集，即：

@Component，@Service，@Repository，@Controller，`代码>

@Configuration，@Bean，@Lazy ， @Scope ， @Order 代码> @Primary，@Profile，@DependsOn，@Import，@ImportResource代码>

正如您所看到＆lt; context：component-scan /＆gt;在逻辑上 扩展 ＆lt; context ：annotation-config /＆gt;使用CLASSPATH组件扫描和Java @Configuration功能。
