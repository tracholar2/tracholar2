如何使用@link标记链接到某个方法？

我想改变

/**
 * Returns the Baz object owned by the Bar object owned by Foo owned by this.
 * A convenience method, equivalent to getFoo().getBar().getBaz()
 * @return baz
 */
public Baz fooBarBaz()

到

/**
 * Returns the Baz object owned by the Bar object owned by Foo owned by this.
 * A convenience method, equivalent to {@link getFoo()}.{@link getBar()}.{@link getBaz()}
 * @return baz
 */
public Baz fooBarBaz()

但我不知道如何正确格式化@link标签。

您可以在JavaDoc中找到许多有关JavaDoc的信息工具参考页面，包括关于

一般格式，从 @link
javadoc文档的一部分是：

Examples

同一课程中的方法：

/** See also {@link #myMethod(String)}. */
void foo() { ... }

** 在同一个包或导入的 不同类中的方法_ ：

/** See also {@link MyOtherClass#myMethod(String)}. */
void foo() { ... }

__ ** 中的方法 

/** See also {@link com.mypackage.YetAnotherClass#myMethod(String)}. */
void foo() { ... }

标签链接到方法，纯文本**，而不是代码字体：

/** See also this {@linkplain #myMethod(String) implementation}. */
void foo() { ... }

方法调用链 ，就像你的问题一样。我们必须为这个类外的方法指定标签，或者我们得到getFoo（）。Foo.getBar（）。Bar.getBaz（）。但是这些标签可能是脆弱的;请参阅下面的“标签”。

/**
 * A convenience method, equivalent to 
 * {@link #getFoo()}.{@link Foo#getBar() getBar()}.{@link Bar#getBaz() getBaz()}.
 * @return baz
 */
public Baz fooBarBaz()

Labels

自动重构可能不会影响标签。 这包括重命名方法，类或包;并更改方法签名。

因此，如果您需要不同于默认文本的文本，请仅提供 __ 。**

例如，您可能会从人类语言链接到代码：

/** You can also {@linkplain #getFoo() get the current foo}. */
void setFoo( Foo foo ) { ... }

或者，您也可以从代码示例中链接不同于默认的文本，如上面的“方法调用链”所示。但是，当API正在发展时，这可能是脆弱的。

Type erasure and #member

如果方法签名包含参数化类型，请使用javadoc @link中的这些类型的擦除。例如：

int bar( Collection<Integer> receiver ) { ... }

/** See also {@link #bar(Collection)}. */
void foo() { ... }
