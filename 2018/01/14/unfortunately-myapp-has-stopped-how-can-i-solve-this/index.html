我正在开发一个应用程序，每次运行它时，都会收到消息：

Unfortunately, MyApp has stopped.

我能做些什么来解决这个问题？

_关于这个问题 - 显然是受到启发的[

这个答案描述了检索堆栈跟踪的过程。已经有堆栈跟踪？请阅读“什么是堆栈跟踪，以及如何使用它来调试我的应用程序错误？”

The Problem

您的应用程序因为未被捕获的RuntimeException被抛出而退出 其中最常见的是 ` NullPointerException < /代码>。

How to solve it?

每当Android应用程序崩溃时（或者任何Java应用程序），一个Stack trace被写入控制台（在本例中为logcat）。这个堆栈跟踪包含了解决您的问题的重要信息。

Android Studio

在窗口的底部栏中，点击Android按钮。或者，您可以按alt + 6。确保在Devices面板中选择了您的仿真器或设备。接下来，尝试找到红色的堆栈轨迹。可能有很多东西登录到logcat，所以你可能需要滚动一下。找到堆栈跟踪的简单方法是清除logcat（使用右侧的回收站），让应用程序再次崩溃。

Eclipse

点击右上角的DDMS按钮。如果不存在，可能需要先使用Java按钮左侧的Open Perspective按钮添加它。您将在底部找到logcat窗格。首先，请确保您的设备已在“devices”面板中选中。接下来，尝试找到红色的堆栈轨迹。同样，可能有很多东西登录到logcat，所以你可能需要滚动一下。在这里找到堆栈跟踪的简单方法是清除logcat（使用右上角的clear log按钮），并让应用程序再次崩溃。您还应该点击您的应用程序的包名称，如果它尚未被选中。这只会过滤掉您的应用程序所做的日志消息。

I have found the stack trace, now what?

耶！你解决问题的一半 您只需要通过分析堆栈跟踪来查明究竟是什么导致了应用程序崩溃。

请阅读“什么是堆栈跟踪，以及如何使用它来调试我的应用程序错误？”

I still can’t solve my problem!

如果你发现你的Exception和它发生的那一行，但是仍然不知道如何解决它，不要犹豫在StackOverflow上提出一个问题。

尝试尽可能简洁：发布堆栈跟踪和 相关代码（例如，几行直到抛出Exception的行）。 / p>

您可以使用 Google的ADB工具来获取Logcat文件来分析问题。

adb logcat > logcat.txt

打开logcat.txt文件并搜索您的应用程序名称。应该有关于为什么失败的信息，行号，类名等。
