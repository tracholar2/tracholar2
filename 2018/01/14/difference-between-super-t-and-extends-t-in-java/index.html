    extends

    List <？>的通配符声明扩展数字＆gt; foo3表示这些都是合法的分配：

    List<? extends Number> foo3 = new ArrayList<Number>();  // Number "extends" Number (in this context)
List<? extends Number> foo3 = new ArrayList<Integer>(); // Integer extends Number
List<? extends Number> foo3 = new ArrayList<Double>();  // Double extends Number

    * 您可以阅读 **` Number ` ** ，因为可以分配给` foo3 `的任何列表包含` Number `或` Number `的一个子类。
* 您无法读取` Integer `，因为` foo3 `可能指向` List＆lt; Double＆gt; `。 
* 您无法读取` Double `，因为` foo3 `可能指向` List＆lt; Integer＆gt; `。

    写作 - 给定上述可能的赋值，可以将List foo3添加到 all 上述可能的ArrayList赋值：

    * 您无法添加` Integer `，因为` foo3 `可能指向` List＆lt; Double＆gt; `。 
* 您无法添加` Double `，因为` foo3 `可能指向` List＆lt; Integer＆gt; `。 
* 您无法添加` Number `，因为` foo3 `可能指向` List＆lt; Integer＆gt; `。

    您无法将任何对象添加到List <？因为你不能保证它真的指向了什么样的 List ，所以你不能保证该对象被允许在 List >。唯一的“保证”是你只能从中读取，你将得到 T 或 T `的子类。

    super

    现在考虑List <？> super T＆gt;。

    List <？>的通配符声明超整型＆gt; foo3表示这些都是合法的分配：

    List<? super Integer> foo3 = new ArrayList<Integer>();  // Integer is a "superclass" of Integer (in this context)
List<? super Integer> foo3 = new ArrayList<Number>();   // Number is a superclass of Integer
List<? super Integer> foo3 = new ArrayList<Object>();   // Object is a superclass of Integer

    阅读** - 给定上述可能的赋值，当你从List foo3读取时，你保证接收什么类型的对象：

    * 由于` foo3 `可能指向` List ＆gt; `或` List＆lt; Object＆gt;，所以不能保证` Integer `    * 由于` foo3 `可能指向` List＆lt; Object＆gt; `，所以不能保证` Number ` ****only** 保证是您将获得一个 **` Object ` ** 或` Object `的子类的实例你不知道是什么子类）

    写作 - 给定上述可能的赋值，可以将List foo3添加到 all 上述可能的ArrayList赋值：

    * 您可以添加一个` Integer `，因为在上面的任何列表中都允许` Integer `。
* 您可以添加` Integer `子类的实例，因为` Integer `的子类的实例在以上任何列表中都是允许的。
* 您无法添加` Double `，因为` foo3 `可能指向` ArrayList＆lt; Integer＆gt; `。 
* 您无法添加` Number `，因为` foo3 `可能指向` ArrayList＆lt; Integer＆gt; `。 
* 因为` foo3 `可能指向` ArrayList＆lt; Integer＆gt; `，所以不能添加` Object `
