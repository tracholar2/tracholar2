如何检查一个对象是否有JavaScript属性？

考虑：

x = {'key': 1};
if ( x.hasOwnProperty('key') ) {
    //Do this
}

这是最好的方法吗？

对于已经给出的答案，我感到非常困惑 - 其中大部分都是完全错误的。当然，你可以拥有未定义，空值或假值的对象属性。所以简单地减少属性检查到typeof这个[property]，更糟的是，x.key会给你带来完全误导的结果。

这取决于你在找什么。如果你想知道一个对象物理上是否包含一个属性（并且它不是来自原型链上的某个地方），那么object.hasOwnProperty就是要走的路。所有现代浏览器都支持它。 （在较旧版本的Safari - 2.0.1和更旧的版本中缺少 - 但浏览器的这些版本很少再使用。）

如果你正在寻找的是如果一个对象有一个可迭代的属性（当你遍历对象的属性，它会出现），然后执行：prop在对象会给你想要的效果。

由于使用hasOwnProperty可能是您想要的，并且考虑到您可能需要一个回退方法，我向您提供以下解决方案：

var obj = {
    a: undefined,
    b: null,
    c: false
};

// a, b, c all found
for ( var prop in obj ) {
    document.writeln( "Object1: " + prop );
}

function Class(){
    this.a = undefined;
    this.b = null;
    this.c = false;
}

Class.prototype = {
    a: undefined,
    b: true,
    c: true,
    d: true,
    e: true
};

var obj2 = new Class();

// a, b, c, d, e found
for ( var prop in obj2 ) {
    document.writeln( "Object2: " + prop );
}

function hasOwnProperty(obj, prop) {
    var proto = obj.__proto__ || obj.constructor.prototype;
    return (prop in obj) &&
        (!(prop in proto) || proto[prop] !== obj[prop]);
}

if ( Object.prototype.hasOwnProperty ) {
    var hasOwnProperty = function(obj, prop) {
        return obj.hasOwnProperty(prop);
    }
}

// a, b, c found in modern browsers
// b, c found in Safari 2.0.1 and older
for ( var prop in obj2 ) {
    if ( hasOwnProperty(obj2, prop) ) {
        document.writeln( "Object2 w/ hasOwn: " + prop );
    }
}

以上是针对hasOwnProperty的一个工作的，跨浏览器的解决方案，但有一点需要注意：无法区分原型和实例上的属性相同的情况 -
只是假定它来自原型。根据你的情况，你可以把它变得更宽松或者更严格，但是至少这应该会更有帮助。

使用 Underscore.js 或（更好） lodash ：

_.has(x, 'key');

调用Object.prototype.hasOwnProperty，但是（a）键入的更短，并且（b）使用“hasOwnProperty
的安全引用”即使 hasOwnProperty `被覆盖）。

特别是，lodash将_。has定义为：

function has(object, key) {
   return object ? hasOwnProperty.call(object, key) : false;
}
// hasOwnProperty = Object.prototype.hasOwnProperty
