wait（）和sleep（）之间的区别

By zhizi

Published 2018-01-14

Contents

线程中的wait（）和sleep（）有什么区别？

我的理解是，一个wait（） -ing线程仍处于运行模式并使用CPU周期，但是sleep（）`不消耗任何CPU周期正确的吗？

为什么我们都有_ wait（）和sleep（）：它们的实现是如何在较低级别上变化的？

等待可以被另一个调用notify在正在等待的监视器上，而sleep 不能。在监视器对象的block synchronized中必须发生wait（和notify），而sleep不会：

Object mon = ...;
synchronized (mon) {
    mon.wait();
}

此时正在执行的线程正在等待 并释放监视器 。另一个线程可能会做

synchronized (mon) { mon.notify(); }

（在同一个mon对象上），第一个线程（假设它是在显示器上等待的唯一线程）将会唤醒。

您也可以致电 notifyAll如果在监视器上有多个线程正在等待

  • 这会将所有线程唤醒_。但是，只有一个线程能够获取监视器（请记住wait位于synchronized块中）并继续运行 -
    其他的将被阻塞，直到他们可以获得显示器的锁定。

另一点是你可以在 wait =“noreferrer”> Object `本身（即等待对象的监视器），而您在sleep oracle.com/javase/8/docs/api/java/lang/Thread.html“rel =”noreferrer“> Thread ` 。

还有一点是，你可以从wait中得到 虚假唤醒 （即正在等待的线程没有明显的原因恢复）。您应该 始终wait，同时按照以下条件转动
：

synchronized {
    while (!condition) { mon.wait(); }
}

还没有提到的一个关键区别是，在休眠线程时， 不会 释放它保存的锁，而在等待释放时，wait（）的对象上的锁是呼吁。

synchronized(LOCK) {
    Thread.sleep(1000); // LOCK is held
}


synchronized(LOCK) {
    LOCK.wait(); // LOCK is not held
}
