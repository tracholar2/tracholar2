jQuery的jquery-1.10.2.min.map触发404（Not Found）

    • 我看到有关文件的错误消息，min.map，找不到：

    GET jQuery’s jquery-1.10.2.min.map is triggering a 404 (Not Found)

    Screenshot

    这是从哪里来的？

    如果Chrome DevTools报告.map文件（可能是jquery-1.10.2.min.map，jquery.min.map或jquery-2.0.3.min.map，但可以与任何东西发生）首先要知道的是，这只是使用DevTools时要求。 您的用户不会遇到这个404。

    现在你可以修复这个问题或者禁用sourcemap功能。

    Fix: get the files

    接下来，这是一个简单的修复。前往 http://jquery.com/download/，然后点击 下载地图文件 链接版本，而且还需要下载未压缩的文件。

    有了地图文件，你可以通过原始的源代码来调试你的jQuery，如果你不喜欢处理像a这样的变量名，那么这将节省大量的时间和沮丧。 <代码> C 。

    关于源代码的更多信息，请访问：JavaScript源代码简介

    Dodge: disable sourcemaps

    取而代之的是，您可以在您的设置中完全禁用JavaScript源地图。如果您从未打算在此页面上调试JavaScript，那么这是一个不错的选择。
    使用DevTools右下方的cog图标打开设置，然后：

    您可以通过移除行

    来移除404

    //@ sourceMappingURL=jquery-1.10.2.min.map

    来自您的jQuery文件的顶部。

    jQuery文件的顶部将如下所示。

    /*! jQuery v1.10.2 | (c) 2005, 2013 jQuery Foundation, Inc. | jquery.org/license
//@ sourceMappingURL=jquery-1.10.2.min.map
*/

    只要将其改为

    /*! jQuery v1.10.2 | (c) 2005, 2013 jQuery Foundation, Inc. | jquery.org/license */

    Purpose of a source map

    基本上，这是一种将组合/缩小文件映射回未建立状态的方法。当您为生产而构建时，同时缩小和组合您的JavaScript文件，您将生成一个包含原始文件信息的源地图。当您在生成的JavaScript中查询某一行和列号时，您可以在返回原始位置的源地图中进行查找。开发人员工具（目前WebKit每晚构建，Google
    Chrome或Firefox 23+）可以自动解析源映射，并使其看起来好像运行的是未记录的和未组合的文件。
