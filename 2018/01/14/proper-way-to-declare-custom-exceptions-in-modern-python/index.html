在现代Python中声明自定义异常的正确方法是什么？

By zhizi

Published 2018-01-14

Contents

在现代Python中声明自定义异常类的正确方法是什么？我的主要目标是跟随任何标准的其他异常类，以便（例如）任何额外的字符串，我包括在例外是打印出来的任何工具抓住例外。

“现代Python”是指在Python 2.5中运行的东西，但对于Python 2.6和Python
3。通过“自定义”，我的意思是一个Exception对象，可以包含有关错误原因的额外数据：一个字符串，也许还有一些其他与异常相关的任意对象。

我被Python 2.6.2中的下面的弃用警告绊倒了：

>>> class MyError(Exception):
...     def __init__(self, message):
...         self.message = message
... 
>>> MyError("foo")
_sandbox.py:3: DeprecationWarning: BaseException.message has been deprecated as of Python 2.6

BaseException 对于名为 message `的属性有特殊的含义。我从 PEP-352中收集到的那个属性在2.5中有特殊的含义我试图贬低，所以我猜这个名字（现在只有一个人）现在被禁止？啊。

我也模糊地意识到Exception有一些神奇的参数args，但我从来不知道如何使用它。我也不确定这是做事的正确方法。我在网上发现的很多讨论表明他们试图在Python 3中取消参数。

更新：两个答案建议覆盖__ init __和__ str __ / __ unicode __ / __ repr __。这似乎很多打字，是否有必要？

也许我错过了这个问题，但为什么不呢：

class MyException(Exception):
    pass

编辑：**重写某些内容（或传递额外的参数），执行以下操作：

class ValidationError(Exception):
    def __init__(self, message, errors):

        # Call the base class constructor with the parameters it needs
        super(ValidationError, self).__init__(message)

        # Now for your custom code...
        self.errors = errors

通过这种方式，你可以将错误消息的字典传递给第二个参数，稍后用e.errors
使用现代Python异常，您不需要滥用.message，或重写.__ str __（）或.__ repr __（）或者其中的任何一个。如果所有你想要的是一个信息性的消息，当你的异常提出，做到这一点：

class MyException(Exception):
    pass

raise MyException("My hovercraft is full of eels")

这会给出一个以MyException结尾的回溯：我的气垫船充满了鳗鱼。

如果你想从异常中获得更多的灵活性，你可以传递一个字典作为参数：

raise MyException({"message":"My hovercraft is full of animals", "animal":"eels"})

然而，要想在除了块以外获得这些细节有点复杂，它们存储在args属性中，该属性是一个列表。你需要做这样的事情：

try:
    raise MyException({"message":"My hovercraft is full of animals", "animal":"eels"})
except MyException as e:
    details = e.args[0]
    print(details["animal"])

仍然有可能将多个项目传入异常，但将来会被弃用。如果您确实需要多个信息，那么您应该考虑完全继承Exception。
