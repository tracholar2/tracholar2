为什么在Python中使用'=='或'is'比较字符串有时会产生不同的结果？

By zhizi

Published 2018-01-14

Contents

我有一个Python程序，其中两个变量被设置为值'public'。在条件表达式中，我有比较var1是var2，但是如果将其更改为var1 == var2，则返回True / p> 现在，如果我打开我的Python解释器，并做相同的“是”比较，它会成功。

>>> s1 = 'public'
>>> s2 = 'public'
>>> s2 is s1
True

我在这里错过了什么？

是是身份测试，==是平等测试。在你的代码中会发生什么，就像这样在解释器中模拟：

>>> a = 'pub'
>>> b = ''.join(['p', 'u', 'b'])
>>> a == b
True
>>> a is b
False

所以，难怪他们不一样，对吧？

换句话说：是是id（a）== id（b）
SilentGhost和其他在这里是正确的。 is用于 identity 比较，而==用于 equal 比较。

交互式工作的原因是（大多数）字符串默认是实习。来自维基百科：

Interned strings speed up string comparisons, which are sometimes a
performance bottleneck in applications (such as compilers and dynamic
programming language runtimes) that rely heavily on hash tables with string
keys. Without interning, checking that two different strings are equal
involves examining every character of both strings. This is slow for several
reasons: it is inherently O(n) in the length of the strings; it typically
requires reads from several regions of memory, which take time; and the reads
fills up the processor cache, meaning there is less cache available for other
needs. With interned strings, a simple object identity test suffices after the
original intern operation; this is typically implemented as a pointer equality
test, normally just a single machine instruction with no memory reference at
all.

所以，当你的程序中有两个字符串文字（字面类型到程序源代码中，用引号括起来的话）中，它们具有相同的值时，Python编译器将自动实习字符串，使它们都被存储在相同的内存位置。
（请注意，这不会总是发生，而且发生这种情况的规则相当复杂，所以请不要在生产代码中依赖这种行为！）

因为在交互式会话中，两个字符串实际上都存储在同一个内存位置，它们具有相同的 标识
，所以是运算符按预期工作。但是，如果通过其他方法构造字符串（即使该字符串恰好包含相同的字符），那么该字符串可以是 相等 ，但不是 相同的字符串
- 也就是说，它具有不同的 标识 ，因为它存储在内存中的不同位置。
