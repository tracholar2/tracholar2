这是有效的，并在JavaScript中返回字符串“10”（更多示例）：

++[[]][+[]]+[+[]]

为什么呢？这里发生了什么？

如果我们把它分开，混乱就等于：

++[[]][+[]]
+

在JavaScript中，+ [] === 0是真的。 +将某些东西转换成数字，在这种情况下，它会回到+“”或0 / p>
因此，我们可以简化它（++优先于+）：

++[[]][0]
+
[0]

因为[[]] [0]表示：从[[]]获取第一个元素，确实如此：

  • [[]] [0]返回内部数组（[]）。由于引用了[[]] [0] === []，所以我们调用内部数组A来避免错误的表示法。 >
  • ++ [[]] [0] == A + 1，因为++表示“按1递增”。
  • ++ [[]] [0] === +（A + 1）;换句话说，它总是一个数字（+1不一定返回一个数字，而++总是这样做） - 感谢Tim Down指出这一点）。

再次，我们可以简化这个混乱成更清晰的东西。让我们用[]替换A：

+([] + 1)
+
[0]

在JavaScript中，也是如此：[] + 1 ===“1”，因为[] ==“”（加入一个空数组） ，所以：

  • +（[] + 1）=== +（“”+ 1）和
  • +（“”+ 1）=== +（“1”）和
  • +（“1”）=== 1
    让我们更简化一下：
1
+
[0]

另外，这在JavaScript中也是如此：[0] ==“0”，因为它是用一个元素连接一个数组。连接将连接由，分隔的元素。用一个元素，你可以推断出这个逻辑将会产生第一个元素本身。

所以，最后我们得到（number + string = string）：

1
+
"0"

=== "10" // Yay!
