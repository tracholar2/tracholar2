在Python中创建多行注释的方法？

By zhizi

发表于 2018-01-14

文章目录

我最近开始学习 Python，但是我找不到实行多线评论。大多数语言都有块注释符号，如

/* 

*/

我在Python中试过这个，但是它会抛出一个错误，所以这可能不是正确的方法。 Python实际上是否有多行注释功能？

你可以使用三重引号的字符串。当它们不是一个文档字符串（类/函数/模块中的第一件事）时，它们被忽略。

'''
This is a multiline
comment.
'''

（确保正确缩进前面的'''以避免IndentationError。）

Guido van
Rossum（Python的创建者）啾啾这个“专家提示”。<
/ p>
然而，Python的风格指南，PEP8，主张使用连续的单向链接，在线评论，这也是你会在许多项目中找到的。编辑们通常会有一个简单的方法去做这件事。

Python的确有一个多行字符串/注释语法，除非用作文档字符串，否则多行字符串
不生成字节码 - 就像＃注释。实际上，它的行为完全像一个评论。

另一方面，如果你说这个行为必须在官方记录 docs是一个真正的评论语法，那么是的，你会说是对的 保证作为语言规范的一部分。

在任何情况下，你的编辑也应该能够轻松地注释一个选定的 区域（通过在每行的前面分别放置一个＃）。如果不是，请切换到 一个编辑器。

在Python中编程没有一定的文本编辑功能可能是一件痛苦的事情 经验。找到正确的编辑（并知道如何使用它）可以做一个大的
在Python编程体验的感知方面有所不同。

编辑不仅能够评论选定的地区，它应该 也能够轻易地将代码块左右移动，并且应该 按下时自动将光标放置在当前缩进级别 输入。代码折叠也是有用的。
