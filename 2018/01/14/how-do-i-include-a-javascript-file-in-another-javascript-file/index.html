怎样在js中包含一个javascript文件

    • JavaScript中是否存在类似于@import的JavaScript，允许您在另一个JavaScript文件中包含JavaScript文件？

    旧版本的JavaScript没有导入，包含或需要，所以已经开发了许多不同的方法来解决这个问题。

    但是最新版本的JavaScript有像
    ES6模块    这样的标准导入模块，尽管这不是大多数浏览器都支持。许多使用模块的浏览器应用程序使用构建和/或
    transpilation     工具，以便使用具有模块特性的新语法。

    ES6 Modules

    请注意，目前，浏览器对ES6模块的支持并不是特别好，但它正在发展中。根据这个StackOverflow答案，它们在Chrome
    60，Firefox 54和MS Edge 15中被支持（但在标志后面），只有Safari 10.1提供无标志支持。

    因此，您现在仍然需要使用构建和/或转换工具来运行有效的JavaScript，而不需要用户使用这些浏览器版本或启用任何标志。

    一旦ES6模块很常见，下面是你如何使用它们：

    // module.js
export function hello() {
  return "Hello";
}



// main.js
import {hello} from 'module'; // or './module'
let val = hello(); // val is "Hello";

    Node.js require

    Node.js目前正在使用 module.exports / require 系统。如果您需要import语法，则可以使用babel进行转码。

    // mymodule.js
module.exports = {
   hello: function() {
      return "Hello";
   }
}



// server.js
const myModule = require('./mymodule');
let val = myModule.hello(); // val is "Hello"

    还有其他一些方法可以让JavaScript在浏览器中包含外部JavaScript内容，而不需要预处理。

    AJAX Loading

    您可以使用AJAX调用加载其他脚本，然后使用eval来运行它。这是最直接的方式，但是由于JavaScript沙盒安全模型，它仅限于您的域。使用eval也为漏洞，黑客和安全问题打开了大门。

    jQuery Loading

    jQuery 库提供了加载功能在一行：

    $.getScript("my_lovely_script.js", function() {
   alert("Script loaded but not necessarily executed.");
});

    Dynamic Script Loading

    您可以将脚本网址添加到HTML中。为了避免jQuery的开销，这是一个理想的解决方案。

    该脚本甚至可以驻留在不同的服务器上。此外，浏览器评估代码。 ＆lt; script＆gt;标签可以被注入到网页＆lt; head＆gt;中，或者在关闭＆lt; / body＆gt;标记。

    这是一个如何工作的例子：

    function dynamicallyLoadScript(url) {
    var script = document.createElement("script"); // Make a script DOM node
    script.src = url; // Set it's src to the provided URL

    document.head.appendChild(script); // Add it to the end of the head section of the page (could change 'head' to 'body' to add it to the end of the body section instead)
}

    这个函数会在页面的头部结尾添加一个新的＆lt; script＆gt;标签，其中src属性被设置为给定的URL作为第一个参数的函数。

    这两个解决方案在 JavaScript疯狂：动态脚本加载中进行了讨论和说明。

    Detecting when the script has been executed

    现在，你必须知道一个大问题。这样做意味着您远程加载代码_。现代Web浏览器将加载文件并继续执行当前脚本，因为它们会异步加载所有内容以提高性能。
    （这适用于jQuery方法和手动动态脚本加载方法。）

    这意味着，如果直接使用这些技巧，那么在您要求加载后，您将无法在下一行使用新加载的代码，因为它仍然会加载。

    例如：my_lovely_script.js包含MySuperObject：

    var js = document.createElement("script");

js.type = "text/javascript";
js.src = jsFilePath;

document.body.appendChild(js);

var s = new MySuperObject();

Error : MySuperObject is undefined

    然后你重新加载页面F5。它的工作原理！混乱…

    那该怎么办？**

    那么，你可以使用我给你的链接中的作者提出的黑客攻击。总之，对于匆忙的人来说，当脚本被加载时，他使用事件来运行回调函数。所以你可以把所有使用远程库的代码放在回调函数中。例如：

    function loadScript(url, callback)
{
    // Adding the script tag to the head as suggested before
    var head = document.getElementsByTagName('head')[0];
    var script = document.createElement('script');
    script.type = 'text/javascript';
    script.src = url;

    // Then bind the event to the callback function.
    // There are several events for cross browser compatibility.
    script.onreadystatechange = callback;
    script.onload = callback;

    // Fire the loading
    head.appendChild(script);
}

    然后，您在
    lambda函数    中加载脚本后，编写要使用的代码：

    var myPrettyCode = function() {
   // Here, do whatever you want
};

    然后你运行所有这些：

    loadScript("my_lovely_script.js", myPrettyCode);

    请注意，脚本可能会在DOM加载之后或之前执行，具体取决于浏览器以及是否包含行script.async = false;。有一个关于Javascript加载的一般文章，讨论这个。 / p>

    Source Code Merge/Preprocessing

    正如在这个答案的顶部所提到的，许多开发人员现在在他们的项目中使用像WebPack，Babel或者Gulp这样的构建/转换工具，允许他们更好地使用新的语法和支持模块，合并文件，缩小，等

    如果有人正在寻找更高级的内容，请尝试 RequireJS 。您将获得额外的好处，例如依赖关系管理，更好的并发性和避免重复（即多次检索脚本）。

    你可以在“模块”中编写你的JavaScript文件，然后把它们作为依赖关系引用到其他脚本中。或者你可以使用RequireJS作为一个简单的“去获得这个脚本”的解决方案。

    示例：

    定义依赖关系为模块：

    <强>一些-dependency.js

    define(['lib/dependency1', 'lib/dependency2'], function (d1, d2) {

     //Your actual script goes here.
     //The dependent scripts will be fetched if necessary.

     return libraryObject;  //For example, jQuery object
});

    implementation.js 是您的“主要”JavaScript文件，它依赖 some-dependency.js

    require(['some-dependency'], function(dependency) {

    //Your script goes here
    //some-dependency.js is fetched.
    //Then your script is executed
});
