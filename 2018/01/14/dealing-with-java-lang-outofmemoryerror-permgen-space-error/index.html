处理“java lang OutOfMemoryError：PermGen空间”错误

By zhizi

Published 2018-01-14

Contents

最近我在我的web应用程序中遇到了这个错误：

java.lang.OutOfMemoryError: PermGen space

这是一个在Tomcat 6和JDK 1.6上运行的典型的Hibernate / JPA + IceFaces / JSF应用程序。
显然这可能会在重新部署应用程序几次之后发生。

是什么原因造成的？为了避免这种情况，可以做些什么？ 我该如何解决这个问题？

解决方法是在Tomcat启动时将这些标志添加到JVM命令行：

-XX:+CMSClassUnloadingEnabled -XX:+CMSPermGenSweepingEnabled

您可以通过关闭tomcat服务，然后进入Tomcat /
bin目录并运行tomcat6w.exe。在“Java”选项卡下，将参数添加到“Java选项”框中。点击“确定”，然后重新启动服务。

如果您收到错误 ，指定的服务不作为已安装的服务存在 ，您应该运行：

tomcat6w //ES//servicename

servicename 是在services.msc中查看的服务器的名称

来源：orx对埃里克的敏捷答案的评论。

您最好试试 **-XX：MaxPermSize = 128M ，而不是-XX：MaxPermGen = 128M。

我无法确切地说明这个内存池的使用情况，但是它与加载到JVM中的类的数量有关。
（因此，为tomcat启用类卸载可以解决问题。）如果您的应用程序在运行时生成并编译类，则更可能需要比默认更大的内存池。
