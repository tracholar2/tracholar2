type（）和isinstance（）之间有什么区别？

By zhizi

Published 2018-01-14

Contents

这两个代码段有什么区别？ 使用type（）：

import types

if type(a) is types.DictType:
    do_something()
if type(b) in types.StringTypes:
    do_something_else()

使用isinstance（）：

if isinstance(a, dict):
    do_something()
if isinstance(b, str) or isinstance(b, unicode):
    do_something_else()

为了总结其他（已经很好！）答案的内容，isinstance迎合了继承（派生类的一个实例也是基类的一个实例） ），而检查type是否相等（它要求类型的同一性，并拒绝子类的实例，AKA子类）。

通常，在Python中，你希望你的代码支持继承，当然（因为继承是如此的方便，用你的代码来停止代码是不好的），所以isinstance是比检查type的身份差，因为它无缝地支持继承。

以下是isinstance比type更好的原因：

class Vehicle:
    pass

class Truck(Vehicle):
    pass

在这种情况下，卡车对象是一辆车，但你会得到这个：

isinstance(Vehicle(), Vehicle)  # returns True
type(Vehicle()) == Vehicle      # returns True
isinstance(Truck(), Vehicle)    # returns True
type(Truck()) == Vehicle        # returns False, and this probably won't be what you want.

换句话说，isinstance也适用于子类。
