用Java创建内存泄漏

By zhizi

Published 2018-01-14

Contents

我刚刚接受了一次采访，并被要求用Java创建内存泄漏。 毋庸置疑，我对于如何开始创建一个产品毫无头绪

一个例子是什么？

在纯Java中创建一个真正的内存泄漏（运行代码无法访问的对象，但仍然保存在内存中）的一个好方法是：

  • 应用程序创建一个长时间运行的线程（或使用线程池更快地泄漏）。
  • 线程通过一个（可选的）ClassLoader加载一个类。
  • 类分配一大块内存（例如new byte [1000000]），在静态字段中存储对它的强引用，然后将引用存储到ThreadLocal中。分配额外的内存是可选的（泄露Class实例就足够了），但它会使泄漏工作更快。
  • 线程清除对自定义类或从其加载的ClassLoader的所有引用。
  • 重复。

这是因为ThreadLocal保持对该对象的引用，该对象保持对其Class的引用，该Class继而保持对其ClassLoader的引用。
ClassLoader反过来保留了它加载的所有类的引用。

（在Java 7之前，尤其在Java
7之前，这种情况更糟，因为Classes和ClassLoader是直接分配给permgen的，而且从来都不是GC’d，但是不管JVM如何处理类的卸载，ThreadLocal都会仍然阻止一个Class对象被回收。）

这种模式的一个变种是应用程序容器（如Tomcat）如果经常以任何方式重新部署恰巧使用ThreadLocals的应用程序，就会像筛子一样泄漏内存。
（由于应用程序容器使用所描述的线程，并且每次重新部署应用程序时都会使用新的ClassLoader。）

静态字段保存对象引用[esp final field]

class MemorableClass {
    static final ArrayList list = new ArrayList(100);
}

上

String str=readString(); // read lengthy string any source db,textbox/jsp etc..
// This will place the string in memory pool from which you cant remove
str.intern();
