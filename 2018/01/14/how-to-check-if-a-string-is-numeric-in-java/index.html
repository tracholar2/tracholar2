如何在Java中检查字符串是否为数字

By zhizi

Published 2018-01-14

Contents

在解析它之前，如何检查String是否是一个数字？

使用 Apache Commons Lang 3.5及以上版本： NumberUtils。 isCreatable 或 StringUtils.isNumeric。

使用 Apache Commons Lang 3.4及以下版本： NumberUtils。 isNumber 或 StringUtils.isNumeric。

您也可以使用[ StringUtils.isNumericSpace ](“http://commons.apache.org/proper
/commons-lang/javadocs/api-release/org/apache/commons/lang3/StringUtils.html

#isNumericSpace-java”。)，它为空字符串返回true，并忽略字符串中的内部空格。
（链接的javadoc包含每个方法的详细示例。）

通常这是通过一个简单的用户自定义函数完成的（即Roll-your-own“isNumeric”函数）。

类似于：

public static boolean isNumeric(String str)  
{  
  try  
  {  
    double d = Double.parseDouble(str);  
  }  
  catch(NumberFormatException nfe)  
  {  
    return false;  
  }  
  return true;  
}

然而，如果你正在调用这个函数，并且你期望许多检查失败，因为不是一个数字，那么这个机制的性能就不会很好，因为你依靠抛出异常每次失败，这是一个相当昂贵的操作。

另一种方法可能是使用正则表达式来检查数字的有效性：

public static boolean isNumeric(String str)
{
  return str.matches("-?\\d+(\\.\\d+)?");  //match a number with optional '-' and decimal.
}

请注意上面的RegEx机制，因为如果你使用的是非阿拉伯数字（即0到9以外的数字），它会失败。这是因为正则表达式的“\
d”部分只会匹配[0-9]，并且在国际上不具有数字意义。 （感谢OregonGhost指出这一点！）

甚至另一种选择是使用Java内置的java.text.NumberFormat对象来查看解析器的位置是否在字符串的末尾。如果是这样，我们可以假设整个字符串是数字：

public static boolean isNumeric(String str)
{
  NumberFormat formatter = NumberFormat.getInstance();
  ParsePosition pos = new ParsePosition(0);
  formatter.parse(str, pos);
  return str.length() == pos.getIndex();
}
