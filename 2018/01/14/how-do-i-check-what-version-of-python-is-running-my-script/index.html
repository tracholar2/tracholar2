如何检查Python解释器的版本是否解释我的脚本？

这些信息可在 sys.version字符串中找到在 sys模块中：

>>> import sys

人类可读：

>>> print (sys.version) #parentheses necessary in python 3.       
2.5.2 (r252:60911, Jul 31 2008, 17:28:52) 
[GCC 4.2.3 (Ubuntu 4.2.3-2ubuntu7)]

进一步处理：

>>> sys.version_info
(2, 5, 2, 'final', 0)
# or
>>> sys.hexversion
34014192

为了确保脚本以Python解释器的最低版本要求运行，请将其添加到您的代码中：

assert sys.version_info >= (2,5)

这比较了主版本和次版本信息。添加微（= 0，1等），甚至releaselevel（=
'alpha'，'final'等）到你喜欢的元组。但是，请注意，如果某个功能在那里，那么“鸭子”检查几乎总是会更好，如果没有，则解决方法（或退出）。有时候功能会在新版本中消失，被其他功能取代。

从命令行（注意大写’V’）：

python -V

这在“man python”中有记载。
