有什么区别HashMap和Hashtable在Java？

中

对于非线程应用程序更有效率

Java在HashMap和Hashtable之间有几个区别：

  • Hashtable 是同步，而 HashMap 不是。这使得 HashMap 对于非线程应用程序更好，因为未同步的对象通常比同步的对象执行得更好。

  • Hashtable 不允许null键或值。 HashMap 允许一个null键和任何数量的null值。

  • 其中一个HashMap的子类是 LinkedHashMap ，所以如果您希望可预测的迭代顺序（默认为插入顺序），您可以轻松地将 HashMap 。 sun.com/javase/7/docs/api/java/util/LinkedHashMap.html“rel =”noreferrer“> LinkedHashMap 。如果您使用的是 ` >哈希表 。

既然同步对你来说不是问题，我推荐 <代码> HashMap中。如果同步成为问题，您也可以查看<代码>的ConcurrentHashMap。

注意，很多答案都表示Hashtable是同步的。 在实践中，这种购买很少。 在accessor /
mutator方法上的同步将停止两个线程同时添加或从地图中移除，但在现实世界中，您经常需要额外的同步。 p> 一个非常常见的习惯用法是“检查然后放入” -
即在地图中查找一个条目，如果它尚不存在，则添加它。无论使用Hashtable还是HashMap，这都不是原子操作。

等价的同步HashMap可以通过以下方式获得：

Collections.synchronizedMap(myMap);

但为了正确地实现这个逻辑，你需要 附加同步 形式：

synchronized(myMap) {
    if (!myMap.containsKey("tomato"))
        myMap.put("tomato", "red");
}

甚至迭代Hashtable的条目（或由Collections.synchronizedMap获取的HashMap）也不是线程安全的，除非通过附加同步来防止映射被修改。

ConcurrentMap的实现界面（例如ConcurrentHashMap）通过包含
线程安全的check-then-act语义 来解决这个问题，例如：

ConcurrentMap.putIfAbsent(key, value);
