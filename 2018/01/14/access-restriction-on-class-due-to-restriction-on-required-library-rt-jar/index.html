由于对所需库rt jar的限制而对类进行访问限制？

By zhizi

发表于 2018-01-14

文章目录

我试图在Java5上编译由IBM的WSDL2Java创建的Java
1.4代码，而不重新创建存根，并在Eclipse中看到这个错误。我假设创建的存根应该只要运行时罐子可用（它们就是）就可以编译。

访问限制：QName类型由于对所需库的限制而不可访问：C：\ Program Files \ Java \ jdk1.5.0_16 \ jre \ lib
\ rt.jar 完整的类名是 javax.xml.namespace.QName `
这到底是怎么回事？

这是我试图从香肠重构猪的情况吗？我最好重新创建存根吗？

还有另一个解决方案，也是可行的。我在此论坛上找到它：

  • 转至项目属性中的 Build Path 设置。
  • 移除 JRE系统库
  • 将其添加回来;选择 “添加库” 并选择 JRE系统库 。默认为我工作。

这是可行的，因为你在不同的jar文件中有多个类。删除并重新添加JRE库将使正确的类成为第一个。
如果您想要一个基本的解决方案，请确保您排除具有相同类的jar文件。

对于我来说：javax.xml.soap.SOAPPart有三个不同的jar：axis-saaj-1.4.jar，saaj- api-1.3 .jar和rt.jar
http://www.digizol.com/ 2008/09 / eclipse-access-restriction-on-library.html对我最合适。

＆GT;弃用和限制的API - ＆gt;禁止参考（访问规则）： - ＆gt;改为警告
