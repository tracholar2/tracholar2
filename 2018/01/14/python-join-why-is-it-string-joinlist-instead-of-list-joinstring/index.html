Python连接：为什么它是string.join（list）而不是list.join（string）？

By zhizi

Published 2018-01-14

Contents

这一直困扰着我。这似乎是更好的：

my_list = ["Hello", "world"]
print my_list.join("-")
# Produce: "Hello-world"

比这个：

my_list = ["Hello", "world"]
print "-".join(my_list)
# Produce: "Hello-world"

有没有一个特定的原因是这样的？

这是因为任何迭代都可以被连接，不仅仅是列表，而且结果和“连接器”总是字符串。

E.G：

import urllib2
print '\n############\n'.join(
    urllib2.urlopen('http://data.stackexchange.com/users/7095'))

因为join（）方法是在字符串类中，而不是列表类？

我同意这看起来很有趣。

请参阅 http://www.faqs.org/docs/diveintopython/odbchelper_join.html：
