感叹号在函数之前做了什么？

By zhizi

Published 2018-01-14

Contents

!function () {}();

JavaScript语法101.下面是 函数声明 ：

function foo() {}

请注意，没有分号：这只是一个 声明 的函数。您需要调用foo（）才能真正运行该函数。

现在，当我们添加看起来无害的感叹号时：！function foo（）{}将它变成一个 表达式 。现在是 函数表达式 。

当然，单独使用！并不会调用这个函数，但现在我们可以把（）放在最后：！function foo（）{ }（），它的优先级高于！，并立即调用该函数。

所以作者正在做的是每个函数表达式保存一个字节;一个更可读的写作方式是这样的：

(function(){})();

最后，！使表达式返回true。这是因为默认情况下，所有的IIFE都返回undefined，这留下了！undefined，它是true。不是特别有用。

功能：

function () {}

没有返回（或未定义）。

有时我们想在创建它的时候调用一个函数。你可能会试试这个：

function () {}()

但是会导致SyntaxError。

在函数之前使用！运算符将其视为表达式，所以我们可以调用它：

!function () {}()

因为！undefined是true，所以这也将返回函数返回值的布尔值，在这种情况下是true。如果你想让实际返回值成为调用的结果，那就试试这样做：

(function () {})()
