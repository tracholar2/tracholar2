为什么打印“B”比打印“＃”慢得多？

By zhizi

Published 2018-01-14

Contents
  1. 1. Eclipse:
  2. 2. Netbeans:
  3. 3. Summary:

我生成了两个1000 x 1000的矩阵：

First Matrix：O和＃ 第二矩阵：O和B。

使用以下代码，第一个矩阵需要8.52秒才能完成：

Random r = new Random();
for (int i = 0; i < 1000; i++) {
    for (int j = 0; j < 1000; j++) {
        if(r.nextInt(4) == 0) {
            System.out.print("O");
        } else {
            System.out.print("#");
        }
    }

   System.out.println("");
 }

用这个代码，第二个矩阵需要259.152秒才能完成：

Random r = new Random();
for (int i = 0; i < 1000; i++) {
    for (int j = 0; j < 1000; j++) {
        if(r.nextInt(4) == 0) {
            System.out.print("O");
        } else {
            System.out.print("B"); //only line changed
        }
    }

    System.out.println("");
}

运行时间显着不同的原因是什么？

正如在注释中所建议的那样，只打印System.out.print（“＃”）;需要7.8871秒的时间，而System.out.print “B”）;使仍在打印...。

正如其他人指出，它正常工作，我尝试了 Ideone.com ，例如，两件代码以相同的速度执行。

测试条件：

  • 我从 Netbeans 7.2 运行这个测试，输出到控制台
  • 我使用System.nanoTime（）进行度量

纯粹的推测是你正在使用一个试图做单词的终端包装而不是字符包装，并将B作为单词字符，而将＃作为非单词字符。所以，当它到达一行的末尾，并搜索一个地方来打破这一行，它几乎立即看到一个＃，并愉快地在那里打破;而使用B时，它必须继续搜索更长的时间，并且可能有更多的文本要打包（在某些终端上这可能是昂贵的，例如输出退格，然后输出空格来覆盖被包装的字母）。

但是，这是纯粹的猜测。

我对Eclipse和Netbeans 8.0.2进行了测试，都是Java版本1.8; 我使用System.nanoTime（）进行测量。

Eclipse:

我在两种情况下均获得 ** - 1.564秒** 。

Netbeans:

  • 使用“＃”： 1.536秒
  • 使用“B”： 44.164秒

所以，看起来Netbeans在打印控制台方面表现不佳。

经过更多的研究，我意识到这个问题是Netbeans最大缓冲区的换行
（它不限于System.out.println命令），这个代码演示：

for (int i = 0; i < 1000; i++) {
    long t1 = System.nanoTime();
    System.out.print("BBB......BBB"); \\<-contain 1000 "B"
    long t2 = System.nanoTime();
    System.out.println(t2-t1);
    System.out.println("");
}

时间结果每次迭代小于1毫秒，除了 每五次迭代 ，时间结果大约是225毫秒。类似于（纳秒）：

Summary:

Eclipse与“B”完美结合

  • Netbeans有一个可以解决的换行问题（因为这个问题在eclipse中不会发生）（在B（“B”）之后没有增加空格）
