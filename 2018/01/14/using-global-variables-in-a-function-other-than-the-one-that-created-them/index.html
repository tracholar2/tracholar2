在创建它们的函数中使用全局变量

By zhizi

Published 2018-01-14

Contents

如果我在一个函数中创建一个全局变量，我怎样才能在另一个函数中使用这个变量呢？ 我是否需要将全局变量存储在需要访问的函数的局部变量中？

你可以在其他函数中使用一个全局变量，在每个赋值给它的函数中声明它为global：

globvar = 0

def set_globvar_to_one():
    global globvar    # Needed to modify global copy of globvar
    globvar = 1

def print_globvar():
    print(globvar)     # No need for global declaration to read value of globvar

set_globvar_to_one()
print_globvar()       # Prints 1

我想这是因为全局变量非常危险，所以Python希望通过明确地要求全局关键字来确保你确实知道你在玩什么。 / p>

如果您想跨模块共享全局变量，请参阅其他答案。

如果我正确地理解了你的情况，你看到的是Python处理本地（函数）和全局（模块）命名空间的结果。

假设你有一个这样的模块：

# sample.py
myGlobal = 5

def func1():
    myGlobal = 42

def func2():
    print myGlobal

func1()
func2()

您可能希望打印42，但打印出来的结果是5.如前所述，如果向func1（）添加一个global声明，那么func2（）将打印42。

def func1():
    global myGlobal
    myGlobal = 42

这里发生的事情是，Python假定在函数中的任何地方被分配给的任何名字都是该函数的本地名字，除非另有明确的说明。如果只是从名称中读取，并且名称不在本地，则它将尝试在任何包含的作用域（例如模块的全局作用域）中查找该名称。

因此，当将名称分配给名称myGlobal时，Python会创建一个局部变量，该局部变量会遮蔽同名全局变量。该地区超出了范围，并在垃圾收集
func1（）返回;同时，func2（）除了（未修改的）全局名称外永远不会看到任何东西。请注意，这个命名空间决定发生在编译时，而不是在运行时 -
如果您在分配给func1（）之前阅读 myGlobal 的值， d得到一个 UnboundLocalError
，因为Python已经决定它必须是一个局部变量，但是它还没有任何相关的值。但是通过使用' global
`’语句，你告诉Python它应该在别处寻找名字，而不是在本地分配。

（我相信这种行为主要是通过优化本地命名空间而产生的 -
没有这种行为，Python的虚拟机将需要至少执行三次名称查找，每次在函数内部分配一个新名字（以确保名称在模块/内置级别上不存在），这将显着减慢一个非常常见的操作。）
