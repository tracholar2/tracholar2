Java中的静态类

By zhizi

Published 2018-01-14

Contents

在java中有没有像static class的东西？

这类课程的意义是什么？静态类的所有方法都需要是static吗？

是否需要相反的方法，如果一个类包含所有的静态方法，那么这个类是否也是静态的？

什么是静态类适合？

Java有静态嵌套类，但是这听起来像是在寻找一个顶级的静态类。 Java没有办法使顶层类成为静态的，但是你可以像这样模拟静态类：

  • 声明您的类final - 防止扩展类，因为扩展静态类没有意义
  • 构造函数private - 防止客户端代码实例化，因为实例化一个静态类是没有意义的
  • 使 所有 类的成员和函数static - 由于类不能被实例化，所以没有实例方法可以被调用或实例字段被访问
  • 请注意，编译器不会阻止你声明一个实例（非静态）成员。只有在您尝试调用实例成员
    时，问题才会显示出来

上面每个建议的简单示例：

public class TestMyStaticClass {
     public static void main(String []args){
        MyStaticClass.setMyStaticMember(5);
        System.out.println("Static value: " + MyStaticClass.getMyStaticMember());
        System.out.println("Value squared: " + MyStaticClass.squareMyStaticMember());
        // MyStaticClass x = new MyStaticClass(); // results in compile time error
     }
}

// A top-level Java class mimicking static class behavior
public final class MyStaticClass {
    private MyStaticClass () { // private constructor
        myStaticMember = 1;
    }
    private static int myStaticMember;
    public static void setMyStaticMember(int val) {
        myStaticMember = val;
    }
    public static int getMyStaticMember() {
        return myStaticMember;
    }
    public static int squareMyStaticMember() {
        return myStaticMember * myStaticMember;
    }
}

静态类有什么好处？静态类的一个很好的使用是定义一次性，实用程序和/或库类，其中实例化是没有意义的。一个很好的例子是数学类包含一些数学常数，如PI和E，只是提供数学计算。在这种情况下要求实例化将是没有必要和混乱的。
查看Java的数学课程。注意它是最终的，它的所有成员都是静态的。如果Java允许顶级类声明为静态，那么Math类确实是静态的。

那么，Java有“静态嵌套类”，但是它们与C＃的静态类并不一样，如果这就是你来自的地方。静态嵌套类只是一个不隐式地引用外部类的实例。

静态嵌套类可以有实例方法和静态方法。

在Java中没有这样的顶级静态类。
