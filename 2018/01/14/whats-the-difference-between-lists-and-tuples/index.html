列表和元组有什么区别？

By zhizi

发表于 2018-01-14

文章目录

有什么区别？

元组/列表有哪些优点/缺点？

除了元组是不可改变的，还有一个应该指导它们用法的语义区别。元组是异构的数据结构（即它们的条目有不同的含义），而列表是同类的序列。
元组有结构，列表有顺序。

使用这种区别使得代码更加明确和易于理解。

一个例子是在页面中引用页面和行号对，例如：

my_location = (42, 11)  # page number, line number

然后，您可以将其作为字典中的键来存储位置的注释。另一方面，列表可以用来存储多个位置。当然，人们可能想要添加或删除列表中的位置，所以列表是可变的。另一方面，从现有位置添加或删除项目是没有意义的

  • 因此元组是不可变的。

在某些情况下，您想要更改现有位置元组中的项目，例如在遍历页面行时。但是元组的不可变性迫使你为每个新值创建一个新的位置元组。这看起来似乎很不方便，但是像这样使用不可变的数据是价值类型和函数式编程技术的基石，它们可以有很大的优势。

在这个问题上有一些有趣的文章，例如“Python元组不是恒定列表或”了解Python中的元组和列表“。正式的Python文档[也提到了这个](http://docs.python.org/2/tutorial/datastructures.html

#tuples-and-sequences)（ “元组是不可变的，通常包含一个异构的序列…” ）。

在像Haskell这样的静态类型语言中，元组中的值通常具有不同的类型，并且元组的长度必须是固定的。在一个列表中，值都是相同的类型，长度不固定。所以差别非常明显。

最后是Python中的 namedtuple，这使得因为一个元组已经被认为是有结构的。这强调了元组是类和实例的轻量级替代的想法。

列表和元组之间的区别

  • <强>立即

    someTuple =（1,2）

    someList = [1,2]

  • <强>尺寸

    a =元组（范围（1000））

    b =列表（范围（1000））

    a . sizeof （）＃8024
    b . sizeof （）＃9088

由于元组操作的规模较小，所以它变得更快一些，但没有太多要提及，直到你有大量元素。

  • 允许的操作

    b = [1,2]

    b [0] = 3＃[3,2]

    a =（1,2）
    a [0] = 3＃错误

这也意味着你不能删除一个元素或排序一个元组。 然而，你可以添加新的元素到列表和元组，唯一的区别是你将通过添加元素来改变元组的id

      a =（1,2）
b = [1,2]

ID（一）＃140230916716520
编号（b）＃748527696

a + =（3，）＃（1，2，3）
b + = [3]＃[1，2，3]

编号（a）＃140230916878160
编号（b）＃748527696
  • <强>用法

因为列表是可变的，所以它不能用作字典中的键，而可以使用元组。

      a =（1,2）
b = [1,2]

c = {a：1}＃确定
c = {b：1}＃错误
