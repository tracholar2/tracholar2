Python中的@staticmethod和@classmethod有什么区别？

By zhizi

Published 2018-01-14

Contents

用 @staticmethod装饰的功能有什么区别？，并使用 @classmethod进行装饰？

也许有一些示例代码会有所帮助：注意foo，class_foo和static_foo的调用签名的区别：

class A(object):
    def foo(self,x):
        print "executing foo(%s,%s)"%(self,x)

    @classmethod
    def class_foo(cls,x):
        print "executing class_foo(%s,%s)"%(cls,x)

    @staticmethod
    def static_foo(x):
        print "executing static_foo(%s)"%x    

a=A()

下面是对象实例调用方法的常用方法。对象实例a隐式作为第一个参数传递。

a.foo(1)
# executing foo(<__main__.A object at 0xb7dbef0c>,1)

使用classmethods **，对象实例的类作为第一个参数而不是self隐式传递。

a.class_foo(1)
# executing class_foo(<class '__main__.A'>,1)

您也可以使用类来调用class_foo。事实上，如果你定义了一些东西 一个类方法，可能是因为你打算从类而不是从类实例中调用它。
A.foo（1）会引发TypeError，但是 A.class_foo（1）`工作得很好：

A.class_foo(1)
# executing class_foo(<class '__main__.A'>,1)

一个人使用的方法是创建可继承的替代构造函数。

使用静态方法**，既不将self（对象实例）和cls（该类）隐式地作为第一个参数传递。除了可以从实例或类中调用它们外，它们的行为与普通函数类似：

a.static_foo(1)
# executing static_foo(1)

A.static_foo('hi')
# executing static_foo(hi)

静态方法用于将与类有逻辑联系的函数分组到类中。

foo 只是一个函数，但是当你调用 a.foo 的时候，你不只是得到这个函数， 你得到一个函数的“部分应用”版本，绑定对象实例 a
作为函数的第一个参数。 foo 需要2个参数，而 a.foo `只需要1个参数。

a绑定到foo。以下是“绑定”一词的含义：

print(a.foo)
# <bound method A.foo of <__main__.A object at 0xb7d52f0c>>

使用a.class_foo，a不会绑定到class_foo，而是类A绑定到<代码> class_foo

print(a.class_foo)
# <bound method type.class_foo of <class '__main__.A'>>

在这里，使用静态方法，即使它是一个方法，a.static_foo只是返回 一个好的’ole函数没有参数绑定。 static_foo需要1个参数 a.static_foo也需要1个参数。

print(a.static_foo)
# <function static_foo at 0xb7d479cc>

当然，当你用类A调用static_foo时，情况也是如此。

print(A.static_foo)
# <function static_foo at 0xb7d479cc>

静态方法是一种对所调用的类或实例一无所知的方法。它只是获得通过的参数，没有隐含的第一个参数。在Python中基本没用 - 你可以使用模块函数而不是静态方法。

另一方面，类方法是一个方法，它通过被调用的类或者被调用的实例的类作为第一个参数。当你希望这个方法成为这个类的一个工厂的时候，这是非常有用的：因为它获得了作为第一个参数被调用的实际类，所以你可以总是实例化正确的类，即使在涉及子类时也是如此。例如，观察一下类方法dict.fromkeys（）是如何在子类上调用时返回子类的实例的：

>>> class DictSubclass(dict):
...     def __repr__(self):
...         return "DictSubclass"
... 
>>> dict.fromkeys("abc")
{'a': None, 'c': None, 'b': None}
>>> DictSubclass.fromkeys("abc")
DictSubclass
>>>
