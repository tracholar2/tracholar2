在Python中检查类型的规范方法是什么？

By zhizi

Published 2018-01-14

Contents

检查给定对象是否为给定类型的最佳方法是什么？如何检查对象是否从给定类型继承？

假设我有一个对象o。我如何检查它是否是str？

检查o的类型是否正好是str：

type(o) is str

要检查o是否是str或str的任何子类的实例（这将是“规范”的方式）：< / p>

isinstance(o, str)

以下内容也适用，在某些情况下可能有用：

issubclass(type(o), str)
type(o) in ([str] + str.__subclasses__())

有关相关信息，请参阅Python库参考中的内置函数。

还有一点需要注意：在这种情况下，您可能真的想要使用：

isinstance(o, basestring)

因为这也会捕获Unicode字符串（unicode不是str的子类; str和unicode是basestring）的子类。

或者，isinstance接受一个类的元组。如果x是（str，unicode）中任何一个子类的一个实例，这将返回True：

isinstance(o, (str, unicode))

大多数 检查对象类型的Pythonic方法是…不检查它。

由于Python鼓励 Duck Typing，因此您应该尽量按照您想要的方式使用对象的方法使用它们。所以如果你的函数正在寻找一个可写的文件对象，那么不要检查它是file的子类，只要使用它的` .write（） / code>方法！

当然，有时候这些很好的抽象会被破坏，而isinstance（obj，cls）就是你所需要的。但谨慎使用。
