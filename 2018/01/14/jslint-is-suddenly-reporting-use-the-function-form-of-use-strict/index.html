JSLint突然报告：使用“严格使用”的函数格式

By zhizi

Published 2018-01-14

Contents

包括声明：

"use strict";

在我的大部分Javascript文件的开头。

JSLint从未对此提出过警告。但现在是这样说：

Use the function form of “use strict”.

有人知道“功能表”是什么吗？

在包装函数中包含'use strict';作为第一个语句，所以它只影响该函数。这可以防止连接不严格的脚本时出现问题。

看到道格拉斯·克罗克福德的最新博客严格模式来到城镇。

来自该帖子的示例：

(function () {
   'use strict';
   // this function is strict...
}());

(function () {
   // but this function is sloppy...
}());

<强>更新 如果您不想立即换行（例如，它是一个节点模块），那么您可以禁用该警告。

对于 JSLint （根据 Zhami ）：

/*jslint node: true */

JSHint ：

/*jshint strict:false */

或（每 Laith Shadeed ）

/* jshint -W097 */

要禁用任何来自JSHint的任意警告，请在JSHint源代码中检查地图代码（详情请参阅文档）。

更新2： JSHint** 支持node：boolean选项。请参阅 .jshintrc在github上

/* jshint node: true */

如果你正在为NodeJS编写模块，它们已经被封装了。告诉JSLint，通过在文件顶部包含节点来获取节点：

/*jslint node: true */
