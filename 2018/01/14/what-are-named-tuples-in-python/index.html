什么是Python中的“命名元组”？

By zhizi

Published 2018-01-14

Contents

我从来没有听说过命名元组，我认为元素可以通过数字（如元组和列表）或键（如在字典中）进行索引。我从来没有想到他们可以被两种方式索引。

因此，我的问题是：

  • 什么是命名元组？
  • 如何使用它们？
  • 为什么/何时应该使用命名元组而不是普通元组？
  • 为什么/何时应该使用普通元组而不是命名元组呢？
  • 是否有任何一种“命名列表”（指定元组的可变版本）？

命名元组基本上是易于创建的轻量级对象类型。命名的元组实例可以使用类似于对象的变量解引用或标准元组语法来引用。它们可以与struct或其他常见记录类型类似地使用，只是它们是不可变的。它们是在Python 2.6和Python 3.0中添加的，尽管在Python
2.4中有一个实现配方

例如，将一个点表示为一个元组（x，y）是很常见的。这导致如下代码：

pt1 = (1.0, 5.0)
pt2 = (2.5, 1.5)

from math import sqrt
line_length = sqrt((pt1[0]-pt2[0])**2 + (pt1[1]-pt2[1])**2)

使用一个命名的元组变得更可读：

from collections import namedtuple
Point = namedtuple('Point', 'x y')
pt1 = Point(1.0, 5.0)
pt2 = Point(2.5, 1.5)

from math import sqrt
line_length = sqrt((pt1.x-pt2.x)**2 + (pt1.y-pt2.y)**2)

但是，名称元组仍然向后兼容正常的元组，所以以下内容仍然可以工作：

Point = namedtuple('Point', 'x y')
pt1 = Point(1.0, 5.0)
pt2 = Point(2.5, 1.5)

from math import sqrt
# use index referencing
line_length = sqrt((pt1[0]-pt2[0])**2 + (pt1[1]-pt2[1])**2)
 # use tuple unpacking
x1, y1 = pt1

因此， 你应该使用命名的元组而不是元组，而你认为​​对象符号会使你的代码变得更加pythonic和更易于阅读
。我个人已经开始使用它们来表示非常简单的值类型，特别是当它们作为参数传递给函数时。它使得这些函数更具可读性，而无需查看元组打包的上下​​文。

另外， 也可以替换普通的 不可变的 类，它们没有函数，只有字段和它们一起使用。你甚至可以使用你命名的元组类型作为基类：

class Point(namedtuple('Point', 'x y')):
    [...]

然而，与元组一样，命名元组中的属性是不可变的：

>>> Point = namedtuple('Point', 'x y')
>>> pt1 = Point(1.0, 5.0)
>>> pt1.x = 2.0
AttributeError: can't set attribute

如果您想要更改值，则需要另一种类型。
可变记录类型有一个方便的配方，它允许你设置新的属性值。 p>

>>> from rcdtype import *
>>> Point = recordtype('Point', 'x y')
>>> pt1 = Point(1.0, 5.0)
>>> pt1 = Point(1.0, 5.0)
>>> pt1.x = 2.0
>>> print(pt1[0])
    2.0

我不知道任何形式的“命名列表”，可以让你添加新的领域，但是。你可能只想在这种情况下使用字典。命名元组可以使用返回{'x'：1.0，'y'：5.0}的pt1._asdict（）转换为字典，并且可以用所有通常的字典功能。

namedtuple是
工厂功能 ，用于制作元组类。通过这个类，我们可以创建可以按名称调用的元组。

import collections

#Create a namedtuple class with names "a" "b" "c"
Row = collections.namedtuple("Row", ["a", "b", "c"], verbose=False, rename=False)   

row = Row(a=1,b=2,c=3) #Make a namedtuple from the Row class we created

print row    #Prints: Row(a=1, b=2, c=3)
print row.a  #Prints: 1
print row[0] #Prints: 1

row = Row._make([2, 3, 4]) #Make a namedtuple from a list of values

print row   #Prints: Row(a=2, b=3, c=4)
