Python @property与getters和setter

By zhizi

Published 2018-01-14

Contents

这是一个纯Python特有的设计问题：

class MyClass(object):
    ...
    def get_my_attr(self):
        ...

    def set_my_attr(self, value):
        ...

和

class MyClass(object):
    ...        
    @property
    def my_attr(self):
        ...

    @my_attr.setter
    def my_attr(self, value):
        ...

Python让我们可以做到这一点。如果你要设计一个Python程序，你会使用哪种方法，为什么？

首选属性 。这是他们在那里。

原因是所有的属性在Python中都是公共的。用下划线或两个名字开始名称只是一个警告，即给定的属性是一个实现细节，可能在未来的代码版本中保持不变。这并不妨碍你实际获取或设置该属性。因此，标准属性访问是正常的，Pythonic访问属性的方式。

属性的优点在于，它们在语法上与属性访问完全相同，因此您可以在不改变客户端代码的情况下从一个更改为另一个。你甚至可以拥有一个使用属性的类的版本（比如按代码或者调试）和一个不用于生产的类，而不需要改变使用它的代码。同时，你不必为所有东西写getter和setter，以防万一以后需要更好地控制访问。

在Python中，你不使用getter或者setter或者只是为了它的乐趣。你首先使用属性，然后再使用，最后只需要，最终迁移到一个属性，而不必使用你的类来改变代码。

确实有很多扩展名为.py的代码，它们使用getters和setter以及继承和无意义的类。一个简单的元组就可以做，但它是使用Python编写C
++或Java的人的代码。

这不是Python代码。
