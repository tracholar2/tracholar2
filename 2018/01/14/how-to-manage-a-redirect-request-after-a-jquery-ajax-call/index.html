如何在jQuery Ajax调用之后管理重定向请求

By zhizi

Published 2018-01-14

Contents

我使用$。post（）来调用使用Ajax的servlet，然后使用生成的HTML片段替换用户当前页面中的div元素。但是，如果会话超时，服务器将发送重定向指令以将用户发送到登录页面。在这种情况下，jQuery正在用登录页面的内容替换div元素，迫使用户的眼睛看到一个罕见的场景。

如何使用jQuery 1.2.6从Ajax调用中管理重定向指令？

我读了这个问题，实现了将响应状态码设置为278的方法，以避免浏览器透明地处理重定向。即使这个工作，我有点不满意，因为它是一个黑客。“

经过更多的挖掘，我放弃了这种方法，并使用 JSON。在这种情况下，所有对ajax请求的响应都具有状态码200，并且响应正文包含在服务器上构建的JSON对象。客户端上的JavaScript可以使用JSON对象来决定它需要做什么。

我有一个类似的问题，你的。我执行一个ajax请求，它有两个可能的响应：一个将浏览器重定向到一个新页面，一个用当前页面替换当前页面上的现有HTML表单。这样做的jQuery代码看起来像这样：

$.ajax({
    type: "POST",
    url: reqUrl,
    data: reqBody,
    dataType: "json",
    success: function(data, textStatus) {
        if (data.redirect) {
            // data.redirect contains the string URL to redirect to
            window.location.href = data.redirect;
        }
        else {
            // data.form contains the HTML for the replacement form
            $("#myform").replaceWith(data.form);
        }
    }
});

JSON对象“data”在服务器上被构造为有两个成员：data.redirect和data.form。我发现这种方法要好得多。

我解决了这个问题：

  • 在响应中添加自定义标头：

      public ActionResult Index（）{
如果（！HttpContext.User.Identity.IsAuthenticated）
{
    HttpContext.Response.AddHeader（ “REQUIRES_AUTH”， “1”）;
}
返回View（）;

    }

  • 将JavaScript函数绑定到 ajaxSuccess 事件，检查是否存在标题：

$（document）.ajaxSuccess（function（event，request，settings）{
if（request.getResponseHeader（’REQUIRES_AUTH’）===’1’）{ window.location
=’/‘; } }）;
