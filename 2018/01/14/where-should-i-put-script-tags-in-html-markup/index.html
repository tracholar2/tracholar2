    • 将JavaScript嵌入到HTML文档中时，放置＆lt; script＆gt;标签和包含JavaScript的适当位置在哪里？我似乎记得，你不应该把它们放在＆lt; head＆gt;部分，但是放在＆lt; body＆gt;部分的开头也是不好的，因为JavaScript必须在页面完全呈现（或类似的东西）之前解析。这似乎将＆lt; body＆gt;部分的 end
    作为＆lt; script＆gt;标记的逻辑位置。

    那么， __ 是放置＆lt; script＆gt;标记的正确位置？

    （这个问题引用了[这个问题< / a>，其中建议JavaScript函数调用应该从＆lt; a＆gt;标记移动到＆lt; script＆gt;标记。但更一般的答案也是适当的。）

    以下是当浏览器加载带有＆lt; script＆gt;标记的网站时发生的情况：

    • 获取HTML页面（例如index.html）
    • 开始解析HTML
    • 解析器遇到引用外部脚本文件的＆lt; script＆gt;标记。
    • 浏览器请求脚本文件。同时，解析器会阻止并解析页面上的其他HTML。
    • 经过一段时间脚本被下载并随后执行。
    • 解析器继续解析HTML文档的其余部分。

    第4步导致不良的用户体验。你的网站基本上停止加载，直到你下载了所有的脚本。如果有一件事，用户恨它正在等待一个网站加载。

    Why does this even happen?

    任何脚本都可以通过document.write（）或其他DOM操作插入自己的HTML。这意味着解析器必须等待，直到脚本被下载＆amp;在它可以安全地解析文档的其余部分之前执行。毕竟，脚本
    可以 在文档中插入自己的HTML。

    然而，大多数JavaScript开发人员不再在文档加载时操作DOM _。而是在修改文档之前等待文档的加载。例如：

    <!-- index.html -->
<html>
    <head>
        <title>My Page</title>
        <script type="text/javascript" src="my-script.js"></script>
    </head>
    <body>
        <div id="user-greeting">Welcome back, user</div>
    </body>
</html>

    使用Javascript：

    // my-script.js
document.addEventListener("DOMContentLoaded", function() { 
    // this function runs when the DOM is ready, i.e. when the document has been parsed
    document.getElementById("user-greeting").textContent = "Welcome back, Bart";
});

    因为你的浏览器不知道my-script.js是不会修改文档的，直到它被下载完毕。执行，解析器停止解析。

    Antiquated recommendation

    解决这个问题的旧方法是将＆lt; script＆gt;标签放在＆lt; body＆gt;的底部，因为这确保了解析器不是阻止直到最后。

    这种方法有其自身的问题：浏览器不能开始下载脚本，直到整个文档被解析。对于大脚本的大型网站，样式表，能够尽快下载脚本对于性能非常重要。如果你的网站在2秒内没有加载，人们会去另一个网站。

    在最佳解决方案中，浏览器将尽快下载脚本，同时解析文档的其余部分。

    The modern approach

    今天，浏览器支持脚本中的async和defer属性。这些属性告诉浏览器在下载脚本时继续解析是安全的。

    async

    <script type="text/javascript" src="path/to/script1.js" async></script>
<script type="text/javascript" src="path/to/script2.js" async></script>

    具有异步属性的脚本是异步执行的。这意味着脚本一下载就被执行，同时不会阻塞浏览器 这意味着脚本2有可能被下载＆amp;在脚本1之前执行。

    根据 http://caniuse.com/#feat=script-async ，所有浏览器中有90％支持这一功能。

    defer

    <script type="text/javascript" src="path/to/script1.js" defer></script>
<script type="text/javascript" src="path/to/script2.js" defer></script>

    具有defer属性的脚本按顺序执行（即第一个脚本1，然后是脚本2）。这也不会阻止浏览器。

    与异步脚本不同，延迟脚本只能在整个文档被加载后执行。

    根据 http://caniuse.com/#feat=script-defer ， 90％的浏览器都支持这一点。 92％至少部分支持。

    浏览器兼容性的重要注意事项：在某些情况下，IE <= 9可能会乱序执行延期脚本。如果您需要支持这些浏览器，请首先阅读！

    Conclusion

    当前最先进的技术是将脚本放在＆lt; head＆gt;标记中，并使用async或defer属性。这可以让您的脚本尽快下载，而不会阻塞您的浏览器。

    好消息是你的网站仍然可以正确加载不支持这些属性的20％的浏览器，同时加速其他80％。

    Put Scripts at the Bottom

    The problem caused by scripts is that they block parallel downloads. The
    HTTP/1.1 specification suggests that browsers download no more than two
    components in parallel per hostname. If you serve your images from multiple
    hostnames, you can get more than two downloads to occur in parallel. While a
    script is downloading, however, the browser won’t start any other downloads,
