检查文件目录是否被写入的最优雅的方法是什么？如果没有，使用Python创建目录？这是我试过的：

import os

file_path = "/my/directory/filename.txt"
directory = os.path.dirname(file_path)

try:
    os.stat(directory)
except:
    os.mkdir(directory)       

f = file(filename)

不知何故，我错过了os.path.exists（感谢kanja，Blair和Douglas）。这是我现在拥有的：

def ensure_dir(file_path):
    directory = os.path.dirname(file_path)
    if not os.path.exists(directory):
        os.makedirs(directory)

有没有一个标志为“开放”，这使得自动发生？

我看到两个质量好的答案，每个答案都有一个小小的缺陷，所以我会把它拿下来：

尝试 ` os.path.exists < / code>，并考虑 `os.makedirs < / code>进行创建。

if not os.path.exists(directory):
    os.makedirs(directory)

正如注释和其他地方所指出的那样，竞争条件 - 如果在os.path.exists和os.makedirs调用之间创建目录，则<代码>
os.makedirs 将失败，并带有 OSError 。不幸的是，一蹴可几的 OSError
`继续是不是万无一失的，因为它会忽略由于其他因素，如权限不足，完整的磁盘等造成的目录失败。

一个选项是捕获OSError并检查嵌入的错误代码（参见[

import os, errno

try:
    os.makedirs(directory)
except OSError as e:
    if e.errno != errno.EEXIST:
        raise

或者，可能会有第二个os.path.exists，但假设另一个在第一次检查之后创建目录，然后在第二个之前删除它 - 我们仍然可能被愚弄。

根据应用程序的不同，并发操作的危险可能大于或小于文件权限等其他因素造成的危险。在选择实现之前，开发人员必须更多地了解正在开发的特定应用程序及其预期的环境。

Python 3.5+:

import pathlib
pathlib.Path('/my/directory').mkdir(parents=True, exist_ok=True)

这个递归的创建目录，如果目录已经存在，不会引发异常。

Python 3.2+:

import os
os.makedirs(path, exist_ok=True)

如果使用Python 3.2+，可选的 < code> exist_ok `参数可用，默认值为False。它在Python 2.x中不存在，最高为2.7。因此，可以简单地在Python 3.2+中指定exists_ok = True以避免在目录已经存在的情况下引发异常。因此，与Python 2.7一样，不需要手动异常处理。

Python 2.7+:

使用pathlib：

如果可以，请安装名为 pathlib2的当前 pathlib ` backport。不要安装名为 pathlib的较旧的未维护的backport。接下来，请参阅上面的Python
3.5+部分，并使用它。

使用os：

import os
try: 
    os.makedirs(path)
except OSError:
    if not os.path.isdir(path):
        raise

虽然一个天真的解决方案可能会首先使用 os.path.isdir后跟[注意，捕获异常并使用errno是有用的，因为OSError：[Errno 17]文件存在，即errno.EEXIST，针对文件和目录都会引发。简单地检查目录是否存在更为可靠。

Alternative:

mkpath创建嵌套的目录，如果目录已经存在，则不执行任何操作。这适用于Python
2和Python 3。

import distutils.dir_util
distutils.dir_util.mkpath(path)

根据错误10948，这个替代方法的一个严重的限制是，它只对每个python进程给定的路径。换句话说，如果使用它来创建一个目录，那么从Python内部或外部删除目录，然后再次使用mkpath重新创建相同的目录，mkpath将会简单地无声地使用其先前创建目录的无效缓存信息，而不会再次创建目录。相比之下，os.makedirs不依赖于任何这样的缓存。对于某些应用程序，这种限制可能没有问题。
