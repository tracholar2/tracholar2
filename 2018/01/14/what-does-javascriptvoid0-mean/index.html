这里提供了一个解释： void运算符。

你想用链接的href来做这件事的原因通常是一个javascript：URL将浏览器重定向到纯文本版本评估JavaScript的结果。但是，如果结果是undefined，那么浏览器保持在同一页面上。 void（0）只是一个短而简单的脚本，它的计算结果为undefined。

除了技术上的答案之外，javascript：void表示作者正在做错了。

使用javascript：伪网址（*）没有什么好的理由。在实践中，如果有人尝试诸如“书签链接”，“在新标签中打开链接”等，就会造成混淆或错误。这种情况发生了很多，现在人们已经习惯了中间点击换新标签：它看起来像一个链接，你想要在一个新的标签中阅读它，但事实证明，这根本不是一个真正的链接，并在中间点击时给出不想要的结果，如空白页面或JS错误。

＆lt; a href =“＃”＆gt;是一个常见的替代方案，可以说不会那么糟糕。不过，您必须记得从您的onclick事件处理程序中返回false，以防止链接被跟踪并滚动到页面顶部。

在某些情况下，可能会有实际有用的地方指向链接。例如，如果您有一个控件，您可以点击打开一个以前隐藏的＆lt; div id =“foo”＆gt;，使用＆lt; a href = “#foo”＆gt;链接到它。或者，如果有一个非JavaScript的方法来做同样的事情（例如，“this.pdf.fp？show =
foo”，设置foo开始），您可以链接到。

否则，如果一个链接只指向某个脚本，那么它不是一个真正的链接，不应该被标记为这样。通常的做法是将onclick添加到＆lt; span＆gt;，＆lt; div＆gt;或＆lt; a＆gt; ;没有一个href，并以某种方式设置它的样式，以清楚你可以点击它。这就是StackOverflow [在撰写本文时所做的;现在它使用href =“＃”]。

这样做的缺点是你失去了键盘控制，因为你不能在span / div / bare-
a上标签或者用空格来激活它。这是否实际上是一个劣势取决于元素打算采取什么样的行动。您可以尝试通过向元素添加一个tabIndex来模仿键盘的可交互性，然后侦听一个Space键。但是，它绝不会100％重现真实的浏览器行为，这不仅仅是因为不同的浏览器可以对键盘作出不同的响应（更不用说非视觉浏览器）。

如果你真的想要一个不是链接的元素，但可以通过鼠标或键盘正常激活，你想要的是一个＆lt; button type =“button”＆gt;（或＆lt; input type =“button”＆gt;对于简单的文本内容来说是一样的）。如果你愿意的话，你总是可以使用CSS来重新定义它，所以它看起来更像一个链接而不是一个按钮。但是因为它的行为像一个按钮，所以你应该如何标记它。

（*：在网站创作中，无论如何，显然它们对bookmarklets非常有用。javascript：伪网址是一种概念上的奇怪现象：一个定位器并不指向一个位置，当前位置内的活动代码，它们对浏览器和webapps都造成了巨大的安全问题，而且不应该被Netscape所发明。）
