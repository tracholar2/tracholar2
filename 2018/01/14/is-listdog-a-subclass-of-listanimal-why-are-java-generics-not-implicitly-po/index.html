List Dog是List Animal的一个子类吗？为什么Java泛型不是隐含多态的？

By zhizi

Published 2018-01-14

Contents

我对Java泛型如何处理继承/多态性有些困惑。

假设有以下层次结构 -

动物 （家长）

狗 - 猫** （儿童）

假设我有一个方法doSomething（List＆lt; Animal＆gt; animals）。根据所有继承和多态的规则，我假定List＆lt; Dog＆gt; 是List＆lt; Animal＆gt;列表＆lt; Cat＆gt; ` 是List＆lt; Animal＆gt;- 所以任何一个都可以传递给这个方法。不是这样。如果我想要实现这个行为，我必须明确地告诉方法通过说 doSomething（List <？extends
Animal＆gt; animals `）接受动物子集的列表。

我明白这是Java的行为。我的问题是 为什么 ？为什么多态一般是隐含的，但是当涉及泛型时，必须指定它？

不，＆lt; Dog＆gt; 不是_ a List＆lt; Animal＆gt; 。考虑一下你可以用 List＆lt; Animal＆gt;
`做什么 - 你可以添加任何 _动物给它……包括一只猫。现在，你可以从逻辑上将一只猫添加到一窝小狗吗？绝对不是。

// Illegal code - because otherwise life would be Bad
List<Dog> dogs = new ArrayList<Dog>(); // ArrayList implements List
List<Animal> animals = dogs; // Awooga awooga
animals.add(new Cat());
Dog dog = dogs.get(0); // This should be safe, right?

突然间，你有一只非常迷惑的猫。

现在，您不能将Cat添加到List <？>中。由于您不知道它是 List＆lt; Cat＆gt; ，所以扩展了Animal＆gt;。你可以检索一个值，并知道它是一个Animal，但是你不能添加任意的动物。对于List＆lt; super Animal＆gt; -
在这种情况下，您可以安全地添加一个Animal，但是您不知道可能从中检索的内容，因为它可能是`列表与LT;对象＆gt;

你要找的是 协变类型参数 。问题是它们在一般情况下不是类型安全的，特别是对于可变列表。假设你有一个List＆lt; Dog＆gt;，它可以用作List＆lt; Animal＆gt;。当您尝试将Cat添加到List＆lt; Animal＆gt;，这实际上是一个List＆lt; Dog＆gt;时会发生什么？自动允许类型参数是协变的，因此打破了类型系统。

添加语法来允许将类型参数指定为协变，这会避免？在方法声明中扩展Foo，但确实增加了额外的复杂性。
