何时以及如何使用ThreadLocal变量？

By zhizi

发表于 2018-01-14

文章目录

什么时候应该使用ThreadLocal变量？

它是如何使用的？

一个可能的（也是常见的）用法是当你有一些不是线程安全的对象，但是你想避免同步访问该对象（我在看你，SimpleDateFormat）。相反，给每个线程自己的对象实例。

例如：

public class Foo
{
    // SimpleDateFormat is not thread-safe, so give one to each thread
    private static final ThreadLocal<SimpleDateFormat> formatter = new ThreadLocal<SimpleDateFormat>(){
        @Override
        protected SimpleDateFormat initialValue()
        {
            return new SimpleDateFormat("yyyyMMdd HHmm");
        }
    };

    public String formatIt(Date date)
    {
        return formatter.get().format(date);
    }
}

文档。

由于ThreadLocal是给定Thread中的数据的引用，因此在应用程序中使用ThreadLocal时，最终可能会加载类泄漏使用线程池的服务器。您需要非常小心地使用ThreadLocal清理任何ThreadLocal的get（）或代码>的remove（）方法。

如果在完成后没有清理，那么对于作为已部署webapp的一部分加载的类的任何引用都将保留在永久堆，永远不会收集垃圾。由于Thread不是您的webapp拥有的，所以重新部署/取消部署webapp不会清理每个Thread对您的webapp的类的引用。每个连续的部署都将创建一个新的类实例，永远不会被垃圾收集。

由于java.lang.OutOfMemoryError：PermGen space的存在，会导致内存不足，而且在使用一些googling之后，可能会增加-XX：MaxPermSize修复这个bug。

如果你最终遇到了这些问题，你可以通过使用Eclipse’s来确定哪个线程和类保留了这些引用内存分析器和/或遵循 Frank
Kieviet的指南和跟进。
