Note on self-closing tags and id= versus name=

本文的早期版本建议使用＆lt; a id ='tith'/＆gt;，使用XHTML的自关闭语法并使用id属性而不是name。

XHTML允许任何标签是“空的”和“自封闭的”。也就是说，＆lt; tag /＆gt;对于＆lt; tag＆gt;＆lt; / tag＆gt;＆lt; / code＆gt;来说是简短的。大多数浏览器都会接受XHTML，但有些则不会。为了避免跨浏览器问题，请按照上面的建议，使用＆lt; tag＆gt;＆lt;
/ tag＆gt; `明确关闭标签。

最后，属性name =在XHTML中被弃用，所以我最初使用的是每个人都能识别的id =。不过，使用id =时，HTML5现在会在JavaScript中创建一个全局变量，这可能不一定是您想要的。因此，使用name =现在可能更友好。

（感谢 Slipp Douglas 向我解释XHTML，nailer ，指出HTML5的副作用 - 请参阅评论和 打钉器
的汇总以获取更多详细信息。name =似乎可以在任何地方使用，但在XHTML中已被弃用。）

使用名称。在HTML 5中使用id不是必需的，并且会在JavaScript中创建全局变量 。

请参阅HTML 5规范，[ 5.9.8导航到片段标识符](http://dev.w3.org/html5/spec/Overview.html

#scroll-to-fragid) - 使用id和name。

知道[大多数浏览器仍将ID变成全局变量< / A>。这是快速测试。使用name避免了创建全局和可能导致的任何冲突。

使用名称的示例：

Take me to [pookie](#pookie)

和目标主播：
