在Android编程中，什么是Context类，它用于什么？

我在开发者网站上阅读了这篇文章，但是我无法理解。

简单地说：

顾名思义，这是应用程序/对象的当前状态的上下文。它可以让新创建的对象了解到底发生了什么。通常，您可以调用它来获取有关程序另一部分（活动和包/应用程序）的信息。

通过调用getApplicationContext（），getContext（），getBaseContext（）或>（当在从Context `扩展的类中时，比如Application，Activity，Service和IntentService类）。

上下文的典型用法：

  • 创建新对象 ： 创建新的视图，适配器，监听器：

    TextView tv = new TextView（getContext（））;

    ListAdapter adapter = new SimpleCursorAdapter（getApplicationContext（），…）;

  • 访问标准公共资源 ： 诸如LAYOUT_INFLATER_SERVICE，SharedPreferences之类的服务：

<预> <代码> context.getSystemService（LAYOUT_INFLATER_SERVICE）

    getApplicationContext（）。getContentResolver（）。query（uri，...）;

Definition of Context::

  • 上下文表示环境数据
  • 它提供对诸如数据库之类的东西的访问

Simpler terms ::

假设Person-X是一家创业软件公司的首席执行官。

公司里有一位首席架构师，这位首席架构师 做公司里所有涉及到数据库，用户界面等工作 等

  • 这是建筑师谁告诉新雇的责任 基于新人的技能，他是否会 在数据库或UI等工作。

Simpler terms ::

  • 就像android活动访问应用程序资源一样。

  • 这与您访问酒店时相似，您需要早餐，午餐和晚餐。 在合适的时间晚餐，对吧？

  • 在逗留期间，您还喜欢其他许多东西。你怎么 得到这些东西？

  • 您请客房服务人员为您提供这些东西。

  • 这里的房间服务人员是考虑到你是上下文的 单人活动和酒店是你的应用程序，最后是早餐，午餐和饮料。 晚餐必须是资源。

涉及上下文的事情是：**

  • 加载资源
  • 启动新的活动。
  • 创建视图。
  • 获取系统服务。
