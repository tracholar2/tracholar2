Android错误：无法在设备上安装* apk *：超时

By zhizi

Published 2018-01-11

Contents

我不时得到这个错误，不知道是什么原因造成的： 当试图运行/调试一个真正的设备上的Android应用程序（在我的情况下银河三星S）我在控制台中收到以下错误：

Failed to install .apk on device :

>

timeout Launch canceled!

这是所有的控制台告诉我。 LogCat不提供任何信息。 Eclipse Problems视图没有显示任何问题。

我尝试了以下步骤，但没有成功： 1.清洁项目（项目 - >清洁） 2.重启设备，Eclipse，笔记本电脑，以上所有…
3.将项目移动到一个没有空格的位置，按照在设备’emulator-5554’上安装apk失败：超时

该应用程序已在该设备上过去多次调试（应用程序在市场上），但这个问题每隔一段时间都会发生，而且非常麻烦。

任何帮助将不胜感激！谢谢。

尝试更改ADB连接超时。我认为它默认为5000毫秒，我把我改为10000毫秒，以摆脱这个问题。

如果你在Eclipse中，可以通过浏览来完成

窗口 - >首选项 - > Android - > DDMS - > ADB连接超时（ms）

当我得到这个错误时，我通常会做的是通过输入cmd来重新启动adb服务器：

adb kill-server

adb start-server

编辑：有一些从不版本的平台工具，你可以从DDMS Perspective菜单（靠近捕捉按钮）Devices Tab，点击Reset adb。

编辑2：另外我发现，最好在你的电脑后面使用USB port，因为大部分前端USB ports都是低功耗的，而且在你的设备上上传apks的速度真的很慢。
