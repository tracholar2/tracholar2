CSS - 展开浮动子DIV高度父母的高度

By zhizi

Published 2018-01-29

Contents

我有页面结构：

<div class="parent">
    <div class="child-left floatLeft">
    </div>

    <div class="child-right floatLeft">
    </div>
</div>

现在， child-left DIV将会有更多的内容，所以 parent ` DIV的高度会随着子DIV的增加而增加。

但问题是child-right高度没有增加。我怎么能把它的高度和它的父母一样？

为父元素添加以下属性

.parent {
    overflow: hidden;
    position: relative;
    width: 100%;
}

然后为.child-right这些：

.child-right {
    background:green;
    height: 100%;
    width: 50%;
    position: absolute;
    right: 0;
    top: 0;
}

这个问题的一个常见的解决方案是使用绝对定位或裁剪的浮点数，但是这些都很棘手，因为如果列的数量和大小发生变化，需要进行大量的调整，并且需要确保“main”列总是最长。相反，我建议你使用三个更强大的解决方案之一：

  • display：flex：到目前为止 最简单的＆amp;最好的解决方案 ，而且非常灵活 - 但不支持IE9及以上版本。
  • table或display：table：非常简单，非常兼容（几乎每个浏览器都有），相当灵活。
  • display：inline-block;宽度：50％带有负边距黑客行为：非常简单，但是列底边框有点棘手。

1. display:flex

这很简单，很容易适应更复杂或更详细的布局 - 但flexbox只支持IE10或更高版本（除了其他现代浏览器）。

_ http：//输出。 jsbin.com/hetunujuma/1

相关html： 

<div class="parent"><div>column 1</div><div>column 2</div></div>

相关css： 

.parent { display: -ms-flex; display: -webkit-flex; display: flex; }
.parent>div { flex:1; }

Flexbox支持更多的选项，但只要有任意数量的列就足够了！

2.<table> or display: table

简单＆amp;非常兼容的方法是使用表 - 如果您需要旧IE支持，我建议您先尝试一下 。你正在处理专栏。 divs +
float并不是最好的方法（更不用提多个级别的嵌套div来绕开css的限制，而不仅仅是使用一个简单的表）。如果您不希望使用table元素，请考虑使用display：table **（不受IE7及更高版本的支持）。

示例： http://jsfiddle.net / emn13 / 7FFp3 /

_ ** （但是考虑使用普通的＆lt; table＆gt;来代替）

<div class="parent"><div>column 1</div><div>column 2</div></div>

相关css： 

.parent { display: table; }
.parent > div {display: table-cell; width:50%; }
/*omit width:50% for auto-scaled column widths*/

这个方法比使用浮动的overflow：hidden更健壮。您可以添加几乎任何数量的列;如果你想的话，你可以让它们自动缩放_。并保持与古代浏览器的兼容性。与float解决方案不同的是，你也不需要事先知道哪一列是最长的;高度很好。

KISS：**不要使用浮动黑客，除非你特别需要。如果IE7是一个问题，我仍然会选择一个带有语义列的普通表格，而不是在任何一天难以维护
，灵活性较低的trick-CSS解决方案。

顺便说一句，如果你需要你的布局是响应式的（例如在小型手机上没有列），你可以使用@media查询返回到小屏幕宽度的普通块布局 - 无论您使用＆lt; table＆gt;或其他任何display：table元素，这都是有效的。

3. display:inline block with a negative margin hack.

另一种方法是使用display：inline block。

_ ： http：// jsbin。 COM / ovuqes / 2
/编辑

_ **（div标签之间没有空格是非常重要的！）

<div class="parent"><div><div>column 1</div></div><div><div>column 2</div></div></div>

相关css： 

.parent { 
    position: relative; width: 100%; white-space: nowrap; overflow: hidden; 
}

.parent>div { 
    display:inline-block; width:50%; white-space:normal; vertical-align:top; 
}

.parent>div>div {
    padding-bottom: 32768px; margin-bottom: -32768px; 
}

这有点棘手，负边界意味着列的“真实”底部被遮蔽。这反过来意味着你不能定位任何相对于这些列的底部的东西，因为它被overflow：hidden所截断。请注意，除了内联块，您还可以使用浮动功能获得类似效果。

TL; DR ：如果您可以忽略IE9及更早版本，请使用 flexbox**
;否则请尝试一个（css）表。如果这两个选项都不适合你，那么就有负面的边缘黑客攻击，但是这些可能会导致奇怪的显示问题，在开发过程中很容易被忽略，而且还有布局限制需要注意。
