这本身不是C＃，但是我还没有看到任何真正使用System.IO.Path.Combine（）的人。实际上，整个Path类是非常有用的，但
没有人使用它！

我敢打赌，每个生产应用程序都有以下代码，即使它不应该：

string path = dir + "\\" + fileName;

lambdas和类型推断 被低估了。 Lambda可以具有多个语句 ，并且它们自动地（作为兼容的委托对象）加倍（只要确保签名匹配）如下：

Console.CancelKeyPress +=
    (sender, e) => {
        Console.WriteLine("CTRL+C detected!\n");
        e.Cancel = true;
    };

请注意，我没有new CancellationEventHandler，也不需要指定sender和e的类型，从事件。这就是为什么编写整个委托（blah blah）时不那么麻烦，这也需要你指定参数的类型。

**

而且，顺便说一下，在函数式编程的意义上，你总是可以返回使Lambdas 的 Lambdas。例如，下面是一个使lambda处理Button.Click事件的lambda：

Func<int, int, EventHandler> makeHandler =
    (dx, dy) => (sender, e) => {
        var btn = (Button) sender;
        btn.Top += dy;
        btn.Left += dx;
    };

btnUp.Click += makeHandler(0, -1);
btnDown.Click += makeHandler(0, 1);
btnLeft.Click += makeHandler(-1, 0);
btnRight.Click += makeHandler(1, 0);

注意链接：（dx，dy）=＆gt; （sender，e）=＆gt;
所以我很高兴参加了函数式编程课程： - ）

除了C中的指针，我认为这是你应该学习的另一个基本的东西： - ）
