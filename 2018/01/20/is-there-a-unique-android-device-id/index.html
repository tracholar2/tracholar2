对这个问题有很多答案，其中大部分只会在某些时候起作用，不幸的是这还不够好。

基于我对设备的测试（所有手机，至少其中一个未被激活）：

  • 所有测试的设备都为TelephonyManager.getDeviceId（）返回值
  • 所有GSM设备（全部使用SIM卡测试）为TelephonyManager.getSimSerialNumber（）返回值
  • 所有的CDMA设备都返回null getSimSerialNumber（）（如预期的那样）
  • 添加Google帐户的所有设备都为ANDROID_ID返回了一个值
  • 只要 ANDROID_ID 和 TelephonyManager.getDeviceId（）`，所有的CDMA设备都返回相同的值（或者派生相同的值） / em>在安装期间添加了Google帐户。
  • 我还没有机会测试没有SIM卡的GSM设备，没有添加Google帐户的GSM设备，或者飞行模式下的任何设备。

因此，如果你想要设备本身是唯一的，TM.getDeviceId（）_就足够了。很显然，有些用户比其他用户更偏执，所以散列一个或多个这些标识符可能是有用的，这样字符串对于设备来说实际上是唯一的，但是并没有明确标识用户的实际设备。例如，使用String.hashCode（）`，并结合一个UUID：

final TelephonyManager tm = (TelephonyManager) getBaseContext().getSystemService(Context.TELEPHONY_SERVICE);

final String tmDevice, tmSerial, androidId;
tmDevice = "" + tm.getDeviceId();
tmSerial = "" + tm.getSimSerialNumber();
androidId = "" + android.provider.Settings.Secure.getString(getContentResolver(), android.provider.Settings.Secure.ANDROID_ID);

UUID deviceUuid = new UUID(androidId.hashCode(), ((long)tmDevice.hashCode() << 32) | tmSerial.hashCode());
String deviceId = deviceUuid.toString();

可能会导致类似以下情况：00000000-54b3-e7c7-0000-000046bffd97
这对我来说足够好了。
