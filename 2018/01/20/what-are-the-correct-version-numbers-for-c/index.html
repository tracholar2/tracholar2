什么是C＃的正确版本号？

By zhizi

Published 2018-01-20

Contents

C＃的正确版本号是什么？什么时候出来？为什么我找不到有关 C＃3.5 的答案？

这个问题主要是为了帮助使用不正确的版本号寻找答案的人， C＃3.5 。希望是任何人找不到错误版本号的答案都会找到 这个
问题，然后用正确的版本号再次搜索。

以下是在撰写本文时已知的C＃版本：

在.NET 1.0和VS2002（2002年1月）发布的 C＃1.0

  • C＃1.2 （非常奇怪）; .NET 1.1和VS2003（2003年4月）发布。在IDisposable实现的IEnumerator上调用Dispose的第一个版本。其他一些小功能。
    .NET 2.0和VS2005（2005年11月）发布的 C＃2.0 。主要新功能：泛型，匿名方法，可空类型，迭代器块

.NET 3.5和VS2008（2007年11月）发布的 C＃3.0 。主要新功能：lambda表达式，扩展方法，表达式树，匿名类型，隐式类型（var），查询表达式

.NET 4和VS2010（2010年4月）发布的 C＃4.0 。主要新功能：后期绑定（dynamic），委托和接口通用方差，更多COM支持，命名参数，元组数据类型和可选参数

使用.NET 4.5和VS2012（2012年8月）发布的 C＃5.0 。
主要特点：异步编程，来电者信息属性。突破性变化：循环变量关闭。

使用.NET 4.6和VS2015（2015年7月）发布的 C＃6.0 。由 Roslyn实施。功能：自动实现属性的初始化器，使用指令导入静态成员，异常过滤器，元素初始化器，catch和finally中的await <代码>在集合初始化器中添加方法。

主要新特性：元组，ref本地和ref返回，模式匹配（包括基于模式的switch语句），内联** out参数声明，本地函数，二进制文字，数字分隔符和任意异步返回。

在VS2017 v15.3（2017年8月发布）中发布了 C＃7.1 次要新功能： async main，元组成员名称推理，默认表情，与泛型匹配的模式。

C＃7.2 随VS2017
v15.5发布（2017年11月）次要新功能：私人保护访问修饰符，span＆lt; T＆gt ;,又名内部指针， aka stackonly struct，其他。

C＃3.5没有这样的事情 - 这里引起混淆的是.NET 3.5中存在C＃3.0。然而，语言和框架是独立版本的，就像CLR 2.0版本是.NET
2.0到3.5，.NET 4引入了CLR 4.0，尽管如此。 .NET 4.5中的CLR有不同的改进，但版本还不清楚：在某些地方，它可能被称为CLR
4.5（这个MSDN页面就是用来引用它的，例如），但是 Environment.Version属性仍然报告4.0.xxx。

有关语言，运行时和框架版本之间关系的更详细信息，请参见C＃在深度网站。这包括有关定位.NET
2.0时可以使用的C＃3.0的哪些功能的信息。 （如果有人想把所有的内容带到这个wiki的答案，他们欢迎。）

截至2017年5月3日，C＃语言团队在他们的github仓库中创建了C＃版本和功能的历史记录：在C＃语言版本中添加的功能

处理C＃的版本号时遇到的最大问题是它与.NET Framework的版本无关，这似乎是由于Visual Studio和.NET
Framework之间的同步版本所致。 p> C＃的版本实际上绑定到编译器，而不是框架。例如，在Visual Studio
2008中，您可以编写C＃3.0并以.NET Framework 2.0,3.0和3.5为目标。 C＃3.0命名法以与 ANSI
C89，C90，C99相同的方式描述了代码语法的版本和支持的功能描述了C的代码语法/功能。

看一下 Mono ，你会看到Mono 2.0（主要实现了2.0版的来自ECMA规范的.NET
Framework）支持C＃3.0语法和功能。
