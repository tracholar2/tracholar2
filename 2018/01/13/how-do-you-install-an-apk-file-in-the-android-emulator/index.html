如何在Android模拟器中安装APK文件？

By zhizi

发表于 2018-01-13

文章目录

我终于设法混淆了我的Android应用程序，现在我想通过安装APK文件并在模拟器上运行它来测试它。

如何在Android模拟器上安装APK文件？

视窗：

执行模拟器（SDK Manager.exe-> Tools-> Manage AVDs-> New然后开始） 启动控制台（Windows XP），运行 -

输入cmd，然后移到SDK目录的platform-tools文件夹。 将APK文件粘贴到“android-sdk \ tools”或“platform-
tools”文件夹中。 然后键入以下命令。 adb install [.apk路径] 例： adb安装C：\ Users \ Name \
MyProject \ build \ Jorgesys.apk

Linux的：

将apk文件复制到platform-tools文件夹中的android-sdk linux。 打开终端并导航到android-
sdk中的platform-tools文件夹。 然后执行这个命令 - ./adb安装FileName.apk
如果操作成功（结果显示在屏幕上），则可以在模拟器的启动器中找到您的文件。

苹果电脑：

PATH=$PATH:~/Library/Android/sdk/platform-tools

Example : PATH=$PATH:/users/jorgesys/eclipse/android-sdk-mac_64/tools

然后运行adb。

苹果电脑：

1.运行仿真器，

2.然后复制你的.apk文件并粘贴到/Users/your_system_username/Library/Android/sdk/platform- tools中， 如果在Mac系统中无法找到sdk路径，请执行以下步骤：打开finder->select在顶层菜单中选择Go选项 - >选择Go to
Folder选项 - >它会弹出一个带有文本框的窗口：/Users/your_system_username/Library/Android/sdk/
- > now open platform -tools文件夹并粘贴您复制的.apk文件，

现在打开终端并键入以下内容： cd Library/Android/sdk/platform-tools 在您的终端执行以下操作：./adb install yourapkfilename.apk如果您收到以下错误消息：error: no devices found - waiting for device，请按照步骤5。 从Android Studio运行你的模拟器，一旦仿真器激活，然后重复步骤4，你将在终端上看到成功消息。

如果你创建了多个模拟器，或者你有一个Android设备插入，adb会抱怨

error: more than one device and emulator

adb help是不是非常清楚该怎么做：

-d                        - directs command to the only connected USB device...
-e                        - directs command to the only running emulator...
-s <serial number>        ...
-p <product name or path> ...

您决定使用的标志必须位于实际的adb命令之前：

adb -e install path/to/app.apk
