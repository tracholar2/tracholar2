calling-an-external-command-in-python

By zhizi

Published 2018-01-13

Contents

如何从Python脚本中调用外部命令（就像我在Unix shell或Windows命令提示符下键入的那样）？

查看标准库中的子流程模块：

from subprocess import call
call(["ls", "-l"])

子流程 vs 系统 的优点是它更加灵活（可以获得stdout，stderr，“真实”状态码，更好的错误处理等…）。

官方文档推荐 子流程
模块替代os.system（）：

The subprocess module provides more powerful facilities for spawning new
processes and retrieving their results; using that module is preferable to
using this function
[os.system()].

“[替换旧版 子流程 文档中的子流程模块](https://docs.python.org/2/library/subprocess.html

#replacing-older-functions-with-the-subprocess-module)部分的功能可能有一些有用的配方。

子流程 模块的官方文档：

  • Python 2 - 子流程
  • Python 3 - 子流程

以下是调用外部程序的方法以及每种方法的优缺点：

os.system（“some_command with
args”）`将命令和参数传递给你的系统的shell。这很好，因为你可以用这种方式一次运行多个命令，并设置管道和输入/输出重定向。例如：

os.system（“some_command＆lt; input_file | another_command＆gt; output_file”）

然而，虽然这很方便，但是你必须手动处理诸如空格之类的shell字符的转义。另一方面，这也可以让你运行简单的命令而不是外部程序。请参阅文档。

stream = os.popen（“some_command with args”）将执行与os.system相同的功能，只不过它为您提供了一个类似文件对象，您可以使用该对象访问该进程的标准输入/输出。还有另外3种不同的popen，它们的处理方式略有不同。如果你把所有东西都作为一个字符串传递，那么你的命令就被传递给shell;如果你把它们作为一个列表，那么你不需要担心逃脱任何事情。请参阅文档。

subprocess 模块的 Popen 类。这是作为 os.popen `的替代品，但由于其如此全面而略微复杂一些。例如，你会说：

打印subprocess.Popen（“回声你好世界”，壳=真，stdout = subprocess.PIPE）.stdout.read（）

而不是：

print os.popen（“echo Hello World”）。read（）

但是，在一个统一的班级里有所有的选项，而不是4个不同的功能。请参阅[文档](https://docs.python.org/2/library/subprocess.html

#popen-constructor)。

来自 subprocess 模块的调用函数。这基本上就像 Popen
`类一样，并且采用了所有相同的参数，但是它只是等到命令完成并返回代码。例如：

return_code = subprocess.call（“echo Hello World”，shell = True）

请参阅文档。

  • 如果您使用Python 3.5或更高版本，则可以使用新的 subprocess.run 函数，它与上面类似，但更灵活，并返回一个 CompletedProcess 对象。

os模块也具有C程序中所有的fork / exec / spawn函数，但我不建议直接使用它们。

subprocess `模块可能应该是你使用的。

最后请注意，对于所有将shell以字符串的形式传递给最终命令的方法，您有责任将其转义。
如果您传递的字符串的任何部分不能完全信任，则存在严重的安全隐患
。例如，如果用户正在输入字符串的某个/任何部分。如果您不确定，只能使用这些常量的方法。给你一个暗示的提示考虑这个代码：

print subprocess.Popen("echo %s " % user_input, stdout=PIPE).stdout.read()

假设用户输入“我的妈妈没有爱我＆amp;＆rm -rf /”。
