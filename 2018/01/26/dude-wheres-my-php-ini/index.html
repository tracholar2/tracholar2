伙计，我的php ini在哪里？

By zhizi

Published 2018-01-26

Contents

几年前，我在我维护的Linux服务器上安装了Apache 2.2x和PHP 5.3.1。我使用了.tar.gz，并按照指示构建它们
（而不是rpms和what-have-you）。一切都很好。

今天，我需要安装这个，这看起来像一个PHP库。我完成了所有安装步骤，并在$ PHP_HOME / lib / extensions / somecomplicatedname / ibm_db2.so中找到了ibm_db2.so

最大的困难是最后一步是配置php.ini，但是我的系统中没有php.ini。恐怖的恐怖。 PHP的工作正常，当然除了这个新的ibm_db2
thingamagic，我想要使用，所以有人可以使用GUI修补DB2。 （我尝试了一个小的PHP脚本，失败并表示ibm_db2函数不可用）。

我必须每隔几年处理一次PHP，所以请在启发我的基础知识的基础上，说明如何启用基于Web的GUI访问DB2。

找到这个最好的方法是： 创建一个PHP文件并添加下面的代码：

<?php phpinfo(); ?>

然后在浏览器中打开，它会显示正在被读取的文件！

通过OP进行更新：

  • 以前接受的答案可能会更快更多方便你，但并不总是正确的。请参阅有关该答案的评论。
  • 请注意更方便的替代＆lt;？php echo php_ini_loaded_file（）; [这个答案](https://stackoverflow.com/questions/8684609/dude-wheres-my-php-ini/24342463#24342463)中提到<？>＆gt; LI>

在命令行执行：

php --ini

你会得到像这样的东西：

Configuration File (php.ini) Path: /etc/php5/cli
Loaded Configuration File:         /etc/php5/cli/php.ini
Scan for additional .ini files in: /etc/php5/cli/conf.d
Additional .ini files parsed:      /etc/php5/cli/conf.d/curl.ini,
/etc/php5/cli/conf.d/pdo.ini,
/etc/php5/cli/conf.d/pdo_sqlite.ini,
/etc/php5/cli/conf.d/sqlite.ini,
/etc/php5/cli/conf.d/sqlite3.ini,
/etc/php5/cli/conf.d/xdebug.ini,
/etc/php5/cli/conf.d/xsl.ini

这是从我的本地开发机器。但是，第二行是有趣的一个。如果没有提到什么，看看第一个。这是PHP查找php.ini的路径。

您可以在脚本中使用phpinfo（）来grep相同的信息，并用浏览器调用它。它在输出的第一部分提到。 php -i对于命令行也是一样的，但是相当不舒服。
