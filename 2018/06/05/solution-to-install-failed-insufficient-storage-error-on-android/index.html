解决Android上的INSTALL_FAILED_INSUFFICIENT_STORAGE错误[关闭]

By zhizi

Published 2018-06-05

Contents
  1. 1. Explanation
  2. 2. Solution
  3. 3. Why other “solutions” worked

INSTALL_FAILED_INSUFFICIENT_STORAGE错误是每个Android开发者生活的祸根。无论应用程序大小，还是有多少存储空间，都会发生这种情况。重新启动目标设备可以简化问题，但很快就会恢复。有几百条（如果不是数千条）留言板帖子为什么会出现这个问题，但是Google的这个人却对这个问题沮丧地保持沉默。

有一个简单的解决方法。如果您的测试设备运行的是Android 2.2或更高版本，请将 android：installLocation
属性添加到应用程序的清单文件中，值为“ preferExternal ”。这将迫使应用程序安装在设备的外部存储设备上，例如手机的SD卡。

例如：

<manifest xmlns:android="http://schemas.android.com/apk/res/android"
  package="com.andrewsmith.android.darkness"
  android:installLocation="preferExternal"

这更像是一个创可贴，而不是一个修补程序，如果你想把你的应用程序安装在设备的内部存储器上，这可能并不理想。但至少会使开发过程不那么令人沮丧。

这只是一个 临时解决方法 ，而不是真正的解决方法。

在发生这种事情之后，我并不满意目前的回应，于是我去工作 试图从AOSP来源中找出答案。我找到了 REAL 解决方案。

Explanation

首先，介绍如何安装 Android（简体）和更新

The first time an app is installed:

>

  • The APK file is
    saved as

>

/data/app/-1.apk (1.apk)

>

>

>

When the app is to be updated:

>

  • The updated APK file is saved as:

>

/data/app/-2.apk (2.apk)

>

  • The first version (1.apk) gets deleted.

>

>

>

On our next update(s):

>

  • The new APK is saved as (1.apk) and (2.apk) is deleted (Repeat forever).

>

当应用程序更新时，我们大多数人都会遇到这个问题，但删除旧的APK失败。这本身并没有导致更新失败，但确实会导致/ data / app中有两个APK文件。

下次尝试更新应用程序时，系统无法移动临时文件，因为（1.apk）和（2.apk）都不为空。由于File＃renameTo（File）不会引发异常，而是返回一个布尔值PackageManager，所以它没有任何方法可以告诉它为什么返回INSTALL_FAILED_INSUFFICIENT_STORAGE，即使这个失败与可用空间量无关。
p>

Solution

执行命令

adb shell "pm uninstall <full.packge.name>"
adb shell "rm -rf /data/app/<full.package.name>-*"

**或

卸载应用

使用您喜欢的方法删除 两者 ：

/data/app/-1.apk

>

/data/app/-2.apk

确保没有其他东西以类似的方式阻止未来的安装。在我的情况下，我有一个/data/app-lib/<full.package.name>-1目录徘徊！在这种情况下，安装到SD卡工作，并随后移动到内存。 （创建/data/app- lib/<full.package.name>，不用-1结尾）。

Why other “solutions” worked

  • 安装到外部存储器的代码显着不同，不存在相同的问题

  • 卸载应用程序只会删除/ data / app中的一个APK文件版本。这就是为什么你可以重新安装一次，但不能更新。

  • 在发生此错误时，模拟器中的可用空间量并不真正相关

您需要增加Android模拟器的内存容量。有两种方法：

右键单击你的Android项目的根目录，进入“Run As”，然后进入“Run Configurations …”。在左边的树中找到“Android
Application”节点，然后选择你的项目，并进入窗口右侧的“目标”标签，查看“Additional Emulator Command Line
Options”字段将需要使窗口更大），最后粘贴“分区大小1024”在那里。点击应用，然后运行以使用您的模拟器。

  • 转到 Eclipse的首选项，然后选择“Launch” €在“默认模拟器选项”字段中添加“分区大小1024”。点击“Apply”，照常使用模拟器。
