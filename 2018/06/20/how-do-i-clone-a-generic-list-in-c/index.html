如何克隆C＃中的通用列表？

By zhizi

Published 2018-06-20

Contents

我有一个C＃中的对象的通用列表，并希望克隆列表。列表中的项目是可复制的，但似乎没有做list.Clone（）的选项。

有没有简单的方法呢？

您可以使用扩展方法。

static class Extensions
{
    public static IList<T> Clone<T>(this IList<T> listToClone) where T: ICloneable
    {
        return listToClone.Select(item => (T)item.Clone()).ToList();
    }
}

如果您的元素是值类型，那么您可以执行：

List<YourType> newList = new List<YourType>(oldList);

但是，如果它们是引用类型，并且需要深度复制（假设元素正确地实现ICloneable），则可以这样做：

List<ICloneable> oldList = new List<ICloneable>();
List<ICloneable> newList = new List<ICloneable>(oldList.Count);

oldList.ForEach((item) =>
    {
        newList.Add((ICloneable)item.Clone());
    });

很显然，在上面的泛型中替换ICloneable，并且使用实现ICloneable的任何元素类型进行强制类型转换。

如果你的元素类型不支持ICloneable，但是有一个copy-constructor，你可以这样做：

List<YourType> oldList = new List<YourType>();
List<YourType> newList = new List<YourType>(oldList.Count);

oldList.ForEach((item)=>
    {
        newList.Add(new YourType(item));
    });

就个人而言，我会避免ICloneable，因为需要保证所有成员的深层副本。相反，我会建议复制构造函数或像YourType.CopyFrom（YourType itemToCopy）这样的返回YourType的新实例的工厂方法。

这些选项中的任何一个都可以用方法（扩展名或其他）来包装。
