如何在Bash中的分隔符上分割字符串？

By zhizi

Published 2017-10-20

Contents

我有这个字符串存储在一个变量：

IN="bla@some.com;john@home.com"

现在我想分割;分隔符的字符串，以便我有：

ADDR1="bla@some.com"
ADDR2="john@home.com"

我不一定需要ADDR1和ADDR2变量。如果他们是一个更好的数组元素。

经过下面答案的建议后，我得到了以下的结果：

#!/usr/bin/env bash

IN="bla@some.com;john@home.com"

mails=$(echo $IN | tr ";" "\n")

for addr in $mails
do
    echo "> [$addr]"
done

输出：

> [bla@some.com]
> [john@home.com]

有一个解决方案将Internal_field_separator（IFS）设置为;。我不知道这个答案发生了什么，你怎么重置IFS回到默认？

RE：IFS解决方案，我试过这个，它的工作原理，我保留旧IFS，然后恢复它：

IN="bla@some.com;john@home.com"

OIFS=$IFS
IFS=';'
mails2=$IN
for x in $mails2
do
    echo "> [$x]"
done

IFS=$OIFS

顺便说一句，当我尝试

mails2=($IN)

我只有第一个字符串在循环打印时，没有括号$IN它的工作原理。

您可以设置内部字段分隔符（IFS）变量，然后让它解析成一个数组。当这种情况发生在一个命令中，那么对IFS的赋值只发生在那个单一命令的环境中（到read）。然后根据IFS变量值将输入解析成一个数组，然后我们可以迭代。

IFS=';' read -ra ADDR <<< "$IN"
for i in "${ADDR[@]}"; do
    # process "$i"
done

它将解析由;分隔的一行项目，并将其推送到一个数组中。用于处理整个$IN的东西，每次输入由;隔开的一行输入：

while IFS=';' read -ra ADDR; do
     for i in "${ADDR[@]}"; do
         # process "$i"
     done
done <<< "$IN"

采取从Bash shell脚本拆分数组：

IN="bla@some.com;john@home.com"
arrIN=(${IN//;/ })

说明：

这个构造用';'（一个空格）替换//（初始的IN表示全局替换）中的所有匹配项，然后将空格分隔的字符串解释为一个数组（这是括号内的括号）。

大括号内使用的语法用' '字符替换每个';'字符称为参数扩展。

有一些常见的问题：

如果原始字符串有空格，则需要使用IFS： ' ' 如果原始字符串有空格，分隔符是新行，则可以使用以下命令设置IFS： IFS=':'; arrIN=($IN); unset IFS;
