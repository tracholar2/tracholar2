如何检查一个变量是否在Bash中设置？

By zhizi

Published 2017-12-30

Contents
  1. 1. The right way
    1. 1.1. Quotes Digression
  2. 2. The wrong way

我如何知道是否在Bash中设置了一个变量？

例如，如何检查用户是否给函数第一个参数？

function a {
    # if $1 is set ?
}

The right way

if [ -z ${var+x} ]; then echo "var is unset"; else echo "var is set to '$var'"; fi

其中${var+x}是一个参数扩展，如果var未被设置，则其值为空，否则替换为字符串x。

Quotes Digression

引号可以省略（所以我们可以说${var+x}，而不是"${var+x}"），因为这个语法和用法保证这只会扩展到不需要引号（因为它扩展到x（它不包含任何分词，所以它不需要引号）
，或者什么也不做（这导致了[ -z ]，它可以方便地计算出与[ -z "" ]相同的值（true））。

然而，虽然引号可以被安全地省略，但并不是很明显（对于这个引用解释的第一作者来说，谁也是一个主要的Bash编码器甚至不是很明显），有时候最好写一个解决方案报价为[ -z "${var+x}" ]，在O（1）速度罚款的可能成本非常小。第一位作者还在代码旁边添加了这个注释，使用这个解决方案给出了这个答案的URL，现在还包括解释为什么可以安全地省略引号。

The wrong way

if [ -z "$var" ]; then echo "var is unset"; else echo "var is set to '$var'"; fi

这是因为它不区分未设置的变量和设置为空字符串的变量。也就是说，如果var=''，那么上面的解决方案会错误的输出那个var是未设置的。

但是这种区分在用户必须指定扩展名或附加属性列表的情况下是非常重要的，而不是指定它们默认为非空值，而指定空字符串应该使脚本使用空的扩展名或列表的附加属性。

要检查非空/非零字符串变量，即如果设置，请使用

if [ -n "$1" ]

这与-z相反。我发现自己使用-n多于-z。
