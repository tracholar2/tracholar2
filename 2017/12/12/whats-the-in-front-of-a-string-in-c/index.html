C＃中字符串前面的@是什么？

By zhizi

发表于 2017-12-12

文章目录

这是C＃（或可能VB.net）.NET的问题，但我想弄清楚以下声明之间的区别：

string hello = "hello";

与

string hello_alias = @"hello";

在控制台上打印没有区别，长度属性是相同的。

它将字符串标记为逐字字符串文本 - 字符串中通常被解释为转义序列的任何内容都被忽略。

所以"C:\\Users\\Rich"和@"C:\Users\Rich"

有一个例外：双引号需要转义序列。为了避免双引号，你需要在一行中放两个双引号。例如，@""""的计算结果为"。

这是一个逐字字符串文字。这意味着逃避不适用。例如：

string verbatim = @"foo\bar";
string regular = "foo\\bar";

这里verbatim和regular具有相同的内容。

它也允许多行内容 - 对于SQL来说非常方便：

string select = @"
SELECT Foo
FROM Bar
WHERE Name='Baz'";

逐字字符串文字所需要的一点转义就是得到一个双引号（“），你把它加倍：

string verbatim = @"He said, ""Would you like some coffee?"" and left.";
string regular = "He said, \"Would you like some coffee?\" and left.";
