如何在PHP中重定向？

是否有可能通过使用PHP将用户重定向到不同的页面？

说用户去www.example.com/page.php，我想重定向到www.example.com/index.php，我怎么会这样做，而不使用元刷新？可能？

这甚至可以保护我的网页免受未经授权的使用

现有答案总结加我自己的两分钱：

1. Basic answer

你可以使用header（）函数来发送一个新的HTTP头，但是这个头必须在任何HTML或者文本之前被发送到浏览器（比如在<!DOCTYPE ...>声明之前）。

header('Location: '.$newURL);

2. Important details

死（）或退出（）

header("Location: http://example.com/myOtherPage.php");
die();

为什么你应该使用die()或exit()：每日跆拳道

绝对网址

该网址必须是绝对的。请参阅RFC 2616.但在大多数情况下，也会接受相对URL。

状态码

PHP的“位置”标题仍然使用HTTP 302重定向代码，但这不是你应该使用的。你应该考虑301（永久重定向）或303（其他）。

注意：W3C提到303-header与“许多pre-HTTP / 1.1用户代理不兼容”，目前使用的浏览器都是HTTP /
1.1用户代理，许多其他用户代理（如蜘蛛和机器人）则不适用。

3. Documentation

HTTP头和PHP中的header（）函数

什么PHP手册说 什么维基百科说 W3C说什么

4. Alternatives

你可以使用http_redirect（$ url）的替代方法;需要安装PECL包pecl。

5. Helper Functions

此功能不包含303状态码：

function Redirect($url, $permanent = false)
{
    header('Location: ' . $url, true, $permanent ? 301 : 302);

    exit();
}

Redirect('http://example.com/', false);

这更灵活：

function redirect($url, $statusCode = 303)
{
   header('Location: ' . $url, true, $statusCode);
   die();
}

6. Workaround

如前所述，只有在写出任何内容之前，header（）才会重定向。如果在HTML输出中被调用，他们通常会失败。那么你可以使用HTML头解决方法（不是很专业！），如：

<meta http-equiv="Location" content="http://example.com/">

或甚至一个Javascript重定向。

window.location.replace("http://example.com/");



function Redirect($url, $permanent = false)
{
    if (headers_sent() === false)
    {
        header('Location: ' . $url, true, ($permanent === true) ? 301 : 302);
    }

    exit();
}

Redirect('http://www.google.com/', false);

不要忘了死（）/退出（）！
