>

There are 3 possible solutions: using INSERT IGNORE, REPLACE, or INSERT a¦
ON DUPLICATE KEY UPDATE.

>

Imagine we have a table:

data from Ensembl, and that due to various reasons the pipeline might be
broken at any step of execution. Thus, we need to ensure two things: 1)
repeated executions of the pipeline will not destroy our database, and 2)
repeated executions will not die due to aduplicate primary keya errors.

There is one more option: to use INSERT a¦ ON DUPLICATE KEY UPDATE syntax,
