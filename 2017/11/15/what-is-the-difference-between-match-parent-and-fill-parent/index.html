match_parent和fill_parent有什么区别？

By zhizi

发表于 2017-11-15

文章目录

我有点困惑了两个XML属性：match_parent和fill_parent。看来，两者都是一样的。他们之间有什么区别？

他们是相同的东西（在API级别8+）。使用match_parent。

FILL_PARENT (renamed MATCH_PARENT in API Level 8 and higher), which means
that the view wants to be as big as its parent (minus padding)

>

…

>

fill_parent: The view should be as big as its parent (minus padding). This
constant is deprecated starting from API Level 8 and is replaced by
match_parent.

http://developer.android.com/reference/android/view/ViewGroup.LayoutParams.html

谷歌改变了名字，以避免混淆。

旧名称fill parent的问题在于，它意味着它影响父级的维度，而match parent更好地描述了结果的行为 - 将维度与父级匹配。

这两个常量最终解析为-1，并在应用程序中导致相同的行为。具有讽刺意味的是，这个名称变更澄清了事情似乎增加了混淆，而不是消除它。
